Manage usage point life cycles

You can add new usage point life cycles and edit or remove existing ones. You can determine what usage point life cycle is default for newly created usage points. For each usage point life cycle you are able to see and manage the associated states and transitions. Usage point life cycles are managed in Administration.

View usage point life cycles

You can view a list of all usage point life cycles that can be associated with usage points in Connexo Insight. One of the shown usage point life cycles is marked as the default one for newly created usage points. By viewing the usage point life cycles you understand what actions are needed to go through all of the different states.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

The available life cycles are shown in a table sorted by name. Scroll to see all usage point life cycles. Details of the selected usage point life cycle are shown in the details pane below.

2.   Click a usage point life cycle’s name to open its landing page with full details.

On the Overview page, you can see the usage point life cycle’s name, default status and a summary of its states and transitions.

3.   Click States to see the usage point life cycle’s states.

4.   Click Transitions to see the usage point life cycle’s transitions.

Out-of-the-box usage point life cycles in Connexo Insight

There is one out-of-the-box usage point life cycle in Connexo Insight which is typical for most usage points. You are free to create additional usage point life cycles if they are needed. See “Add a usage point life cycle”.

The out-of-the-box usage point life cycle is called Standard usage point life cycle. After the Connexo installation, this usage point life cycle is marked as the default one.

The Standard usage point life cycle has four states and six transitions. In a pre-operational stage (Under construction) the usage point can be installed as an active or inactive usage point (Install active, Install inactive). While being suspended, the usage point can be activated and from the operational stage it can be deactivated. From the operational state, the usage point can be parked in a post-operational stage (Demolish).

Figure: Standard usage point life cycle.

 

Out-of-box states in Standard usage point life cycle. Also see “Manage states on a usage point life cycle”

State

Description

Under construction

The usage point is created but not yet configured. For example, a building has been erected and a cable is available but it is still being configured, certified or it hasn’t got any meters linked with it yet. So, there isn’t any consumption.

From this state, two transitions are possible:

  • Install active.

  • Install inactive.

Active

A usage point will spend most of its lifetime in this state. It is constructed, installed and fully functional. All communications are in place and meters are installed. It is now possible to collect data and to validate, estimate, edit and export this data.

From this state, two transitions are possible:

  • Demolish.

  • Deactivate.

Inactive

A usage point can be deactivated temporarily due to a customer relocation, a meter replacement process or another cause.

From this state, two transitions are possible:

  • Demolish.

  • Activate.

Demolished

This is the final state for a usage point. The usage point doesn’t exist anymore, for example, as a result of the destruction of the building in which it was installed. There aren’t any further transitions from this state.

Out-of-the-box transitions in Standard usage point life cycle including associated pretransition checks and auto actions. Also see “Manage transitions on a usage point life cycle”.

Transition

Description

Install active

The usage point moves from a pre-operational stage to an operational stage where it is configured and ready to collect data.

 

The following pretransition checks are done to enable this transition:

  • Meters must be specified for all the roles.

  • The metrology configuration must be defined.

 

Additional pretransition checks are configurable:

  • All data validated.

  • All data valid.

  • No running service calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

The following auto actions are done if the pretransition checks are successful:

  • Set connection state: Connection state is changed into Connected. This action is needed because a usage point doesn’t have a connection state right after creation in its pre-operational stage. This transition allows the automatic update of the connection state during a usage point life cycle.

 

Additional auto actions are configurable:

  • Close all issues.

  • Reset validation results.

  • Cancel all service calls.

  • Force validation and estimation.

  • Remove from static usage point groups.

Install inactive

The usage point moves from a pre-operational stage to a suspended stage where it is configured and but not yet ready to collect data. It is on hold.

 

The following pretransition checks are done to enable this transition:

  • Meters must be specified for all the roles.

  • The metrology configuration must be defined.

 

Additional pretransition checks are configurable:

  • All data validated.

  • All data valid.

  • No running serivce calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

 

The following auto actions are done if the pretransition checks are successful:

  • Set connection state: Connection state is changed into Connected. This action is needed because a usage point doesn’t have a connection state right after creation in its pre-operational stage. This transition allows the automatic update of the connection state during a usage point life cycle.

 

Additional auto actions are configurable:

  • Close all issues.

  • Reset validation results.

  • Cancel all service calls.

  • Force validation and estimation.

  • Remove from static usage point groups.

Deactivate

The usage point becomes temporarily inactive, for example due to a customer relocation or technical issues such as a meter replacement procedure.

 

There aren’t any pretransition checks for this transition. The following pretransition checks can be configured:

  • Meters must be specified for all the roles.

  • The metrology configuration must be defined.

  • All data validated.

  • All data valid.

  • No running service calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

 

There aren’t any auto actions for this transition. The following auto actions can be configured:

  • Set connection state.

  • Close all issues.

  • Reset validation results.

  • Cancel all service calls.

  • Force validation and estimation.

  • Remove from static usage point groups.

Activate

The usage point becomes active again after resolution of the issues that prevented its use.

 

There aren’t any pretransition checks for this transition. The following pretransition checks can be configured:

  • Meters must be specified for all the roles.

  • The metrology configuration must be defined.

  • All data validated.

  • All data valid.

  • No running service calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

 

There aren’t any auto actions for this transition. The following auto actions can be configured:

  • Set connection state.

  • Close all issues.

  • Reset validation results.

  • Cancel all service calls.

  • Force validation and estimation.

  • Remove from static usage point groups.

Demolish from active

The usage point becomes permanently unusable, for example after destruction of the building in which it was installed. The usage point was still active before being demolished.

 

There aren’t any pretransition checks for this transition. The following pretransition checks can be configured:

  • Meters must be specified for all the roles.

  • The metrology configuration must be defined.

  • All data validated.

  • All data valid.

  • No running service calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

  • Meters are specified for all the roles.

  • Metrology configuration is defined.

 

There aren’t any auto actions for this transition. The following auto actions can be selected optionally:

  • Reset validation results.

  • Set connection state.

  • Close all issues.

  • Cancel all service calls.

  • Force validation and estimation.

  • Remove from static usage point groups.

Demolish from inactive

The usage point becomes permanently unusable, for example after destruction of the building in which it was installed. The usage point was already temporarily inactive before being demolished.

 

There aren’t any pretransition checks for this transition. The following pretransition checks can be configured:

  • The metrology configuration must be defined.

  • All data validated.

  • All data valid.

  • No running service calls.

  • No active processes.

  • Linked meters are activated.

  • Linked meters have initial read.

  • All the meters are decommissioned.

Add a usage point life cycle

You can add a usage point life cycle to support usage points with a life cycle that is different than the configuration of existing usage point life cycles in Connexo Insight.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click Add usage point life cycle.

The Add usage point life cycle page appears.

3.   Enter a name for the new life cycle.

The chosen name must be unique.

By default, the new usage point life cycle has the same states and transitions as the default usage point life cycle.

4.   Click Add.

The new usage point life cycle is now listed in the overview table.

5.   Make the needed changes to the states and transitions of your new usage point life cycle.

Clone a usage point life cycle

You can clone an existing usage point life cycle to make the creation of a new usage point life cycle easier and quicker by basing it on the configuration of an existing one.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Select the usage point life cycle that has the states and transitions you want to start from to create your new usage point life cycle.

3.   Click the action icon  at the end of the row or the actions button in the details pane and select Clone. As all other actions on usage point cycles, Clone is also available from the actions button on the usage point life cycle’s landing page.

4.   Click Clone.

The new usage point life cycle is now listed in the overview table.

Edit a usage point life cycle

You can edit an existing usage point life cycle to correct eventual mistakes or to make it match your current needs. You will only be able to edit the name of the usage point life cycle. If you also want to edit the associated states and transitions, then you must open the life cycle’s landing page. See “Edit a state on a usage point life cycle” and “Edit a transition on a usage point life cycle”.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

1.   Select the usage point life cycle that you want to edit.

2.   Edit the field Name.

3.   Click Save.

Remove a usage point life cycle

You can remove a usage point life cycle that is no longer relevant. A usage point life cycle can only be removed if it is no longer associated with any usage point.

5

Steps

1.   Go to Administration > Usage point lifec ycle management > Usage point life cycles.

2.   Select the usage point life cycle that you want to remove.

3.   Click the action icon  at the end of the row or the actions button in the details pane and select Remove. As all other actions on usage point cycles, Remove is also available from the actions button on the usage point life cycle’s landing page.

4.   Click Remove to confirm removal of the usage point life cycle.

The removed usage point life cycle is no longer listed in the overview table.

Mark a usage point life cycle as default

You can mark a usage point life cycle as the default one. A new usage point that is added to Connexo Insight will automatically use the default usage point life cycle.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Select the usage point life cycle that you want to mark as the default one.

3.   Click the action icon  at the end of the row or the actions button in the details pane and select Set as default. As all other actions on usage point cycles, Set as default is also available from the actions button on the usage point life cycle’s landing page.

The value in the field Default is changed to Yes and an icon  is added in the column Default of the defaulted usage point life cycle.
