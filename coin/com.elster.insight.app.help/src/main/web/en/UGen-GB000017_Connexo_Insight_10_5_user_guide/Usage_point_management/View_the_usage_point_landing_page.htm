View the usage point landing page

See “Landing page with essential information for an object” for general information on the layout of an object landing page.

When opening a usage point you see the usage point overview page. Here, you can see the usage point’s main attributes and metrology configuration settings as well as an overview of what is going on with the usage point. You get a quick status summary of the selected usage point all on one page.

The icons next to the usage point’s name in the vertical navigation bar to the left represent its service category and connection state. Hover over them to see a tooltip. The navigation bar contains links to more detailed information about the usage point.

Because a lot is happening all the time involving a usage point, you can refresh the information that is shown on the page at any time.

5

Steps

1.   Click the Refresh button in the upper right corner of the screen.

2.   The data on the page is updated and the Last refreshed timestamp is adjusted.

 

Click one of the links in the vertical navigation bar to see detailed information about the current usage point’s:

1.   Usage point attributes.

2.   History.

3.   Processes.

4.   Service calls.

5.   Metrology configuration.

6.   Calendars.

7.   Purposes.

Understand the What’s going on? Widget for a usage point

On the What’s going on? widget you can see the activities around the current usage point. These activities can be processes, service calls, issues, etc. See “The “What’s going on?” widget” for more information on the What’s going on? widget.

View the usage point summary

The Usage point summary widget shows the main usage point attributes including service category and connection state with their corresponding icons. Click the link More attributes to open the Usage point attributes page where you find all attributes. See “View usage point attributes”.

View basic metrology configuration information

The Metrology configuration widget summarises information about the metrology configuration that is linked to the current usage point. Under the name of the metrology configuration its activation date is shown.

If you have sufficient privileges you can click the metrology configuration’s name to open its landing page and see more details. You can open the same page by clicking Metrology configuration in the vertical navigation bar. See “View the metrology configuration details on a usage point”.

In case there is no metrology configuration linked with the usage point yet and if you have sufficient privileges, then you have a button to add one.

In Metrology configuration you can also see the active purposes of the usage point. Each purpose is accompanied by an icon indicating its validation status. Hover over it to see a tooltip.

Finally, you can see the devices that are activated on the usage point along with their activation date. You can click the device names to see more details. If there are more than two devices activated on the usage point, then only the number of linked devices is shown. In case there are no meters activated on the usage point but there is a metrology configuration linked to it and you have sufficient privileges, then you see a button to add a meter.

You can see the history of meters that have been installed and removed on the usage point.

5

Steps

1.   Click History in the vertical navigation bar on the left.

2.   Click the tab Meters.

The icon in the column Current indicates the meters that are currently linked to the selected usage point. The value in the column Ongoing processes indicates the number of processes that have been triggered on the usage point that is related at the moment when it was related to the selected meter. The actual processes are shown in the details pane.

3.   Click the link on the process ID in the details pane to open the Ongoing processes page on the usage point.

View data completion

The data completion widget shows validation status of each channel/ register per purpose and for a specific period. The period selector contains all relative periods of the category Usage point validation overview. Use the Purpose selector to quickly switch to another purpose.

If there are more than three channels on the widget, then use the navigation dot below to switch pages. The white dot represents the current page.

Each channel is represented by a pie chart and accompanying attributes explaining the data quality. Hovering over a part of the pie chart shows a tooltip with a section breakdown. Each pie chart is headed by its channel name. You can click it to open its output details. See “View output lists of a metrology configuration purpose”.

View usage point attributes

View all attributes of the current usage point in one place.

5

Steps

1.   Open a usage point using the search page.

2.   Click More attributes on the Usage point summary widget on the usage point overview page.

The Usage point attributes page is divided into two columns. The first column on the left always contains General information and Technical information.

In the second column on the right side you find the custom attribute sets - if there are any and if you have the correct privileges to view them. There is one widget for each custom attribute set and they are sorted alphabetically by name. If there are none, then the second column stays empty.

MRID is the unique identifier of the usage point. The value in Service category determines what technical attributes and custom attribute sets are shown. The field Type determines if there are meters located at the physical location of the usage point or not. Because of their impact on the entire configuration the values in these fields can’t be edited after creation of the usage point.

In case of ‘time-sliced’ attribute sets, the time from which the attribute set is valid is shown under its title. At the bottom of the widget you can click the Versions link to open the version management page in History. See “View and add custom attribute set versions (History)”.

You can check the configuration of the custom attribute sets in Administration > Service categories. See “Understand service categories”. Custom attribute sets have been prepared for you by your Connexo delivery team.

For more details on editing usage point attributes, see “Edit usage point attributes”.

View and add custom attribute set versions (History)

Manage the versions of the current usage point’s time-sliced custom attribute sets. You must have sufficient privileges to see the custom attribute set in order to view its detailed version information.

5

Steps

1.   Open a usage point using the search page.

2.   Click History in the vertical navigation bar on the left.

On the History page there is one tab for each time-sliced custom attribute set.

3.   Click the desired tab to see detailed version information.

The versions for the selected custom attribute set are listed in a paged table with the newest one on top. In the table you see the start date of each version.

4.   Select a version to see its details in the details pane under the table.

5.   If you have sufficient privileges you can edit the selected version by selecting Edit from the action icon  at the end of its row or from the action button in the details pane.

6.   If you have sufficient privileges you can add a version by clicking the button Add version just above the table.

7.   To add a new version that is based on an existing version you can clone it by selecting Clone from the action icon  at the end of its row or from the action button in the details pane.

View processes on a usage point

Understand what processes are currently running on the selected usage point and see what processes have run in the past.

5

Steps

1.   Open a usage point from the search page.

2.   Click Processes in the vertical navigation bar to the left.

By default, the tab Running processes is shown. This tab shows the processes that are currently running on the selected usage point. The processes are listed in a paged table.

3.   Click a process in the table to see its details in the details pane under the table.

4.   Click the tab Status overview in the details pane to see the steps of the selected process and eventual variables for each these steps. These steps can be technical system steps supporting the functional business process.

5.   Click the tab History to see what processes have completed or are simply inactive on the selected usage point.

The inactive processes are listed in a page table.

6.   Use the filter bar on top to narrow down the number of processes.

You can filter by process name, start date and time, status and by the user who started the process. Don’t forget to click Apply to execute the filter.

7.   Click Status overview to view what happens in the process and eventual variables for these tasks and steps.

View service calls on a usage point

View the service calls on the current usage point to track the status of actions on it triggered by different systems using web services, api, importers, messages and others. This overview allows you to determine if actions must be taken on the usage point.

5

Steps

1.   Open a usage point using the search page.

2.   Click Service calls in the vertical navigation bar to the left.

By default, the tab Running service calls is shown. This tab shows the service calls that are currently running on the selected usage point. The service calls are listed in a paged table. Each one has a link to the corresponding service call landing page.

3.   Click the tab History to see the service calls that have been completed on the current usage point.

4.   Use the filter bar on top to narrow down the number of processes.

You can filter by ID or reference, Type, Status and Received date. Don’t forget to click Apply to execute the filter.

Cancel running service calls on a usage point

You can cancel one or multiple service calls simultaneously to interrupt a task that is taking an abnormal long time.

5

Steps

1.   Open a usage point using the search page.

2.   Click Service calls in the vertical navigation bar to the left.

3.   In the tab Running service calls select the service call you need to cancel.

4.   Click the actions icon and select Cancel.

5.   Click Yes to confirm cancelling the service call. The selected service call will stop running.

You can cancel all running service calls on the usage point simultaneously to save time.

5

Steps

1.   Click the Actions button on top of the overview table showing all running service calls.

2.   Select Cancel all.

3.   Click Yes to confirm cancelling all running service calls on the current usage point.

 

The cancelled service calls will disappear from the tab Running service calls and show up in the tab History with status Cancelled.

View the metrology configuration details on a usage point

View all metrology configuration details on a usage point such as activated meters with their roles, the list of purposes and their status all in one place to troubleshoot and understand how the metrology configuration was applied to the usage point and what meters were activated per meter role.

5

Steps

1.   Open a usage point using the search page.

2.   Click Metrology configuration in the vertical navigation bar to the left or click More details in the widget Metrology configuration.

 

The Metrology configuration screen has three sections.

In the Status section there are two fields.

Field

Description

Status

Shows the metrology configuration’s status. It represents the status of the active purposes of the usage point. If at least one of the purposes has the incomplete status, then the status of the metrology configuration becomes Incomplete.

Name

Name of the metrology configuration. If you have sufficient privileges it is a link that brings you to the landing page of the metrology configuration where you can its full details. See “Metrology configurations”.

The section Meter roles lists the meter roles that are defined on the metrology configuration in a table. This section isn’t shown in case of unmeasured connections. Four columns are shown in the Meter roles table.

Field

Description

Meter role

Possible meter roles depend on the service category. They can be consumption, production, main, check, peak consumption, off peak consumption. The meter role specifies how many meters must be linked to the usage point and which reading types are provided by each of these meters. Each potential meter is represented by a meter role on a metrology configuration.

Required

Required if the meter role is part of a formula in one of the active purposes.

Meter

MRID of the meter. If you have sufficient privileges you can click it to navigate to the device’s landing page.

Activation date

The activation date is initially the same as the creation date of the usage point. Later, this date can be changed if the meter is replaced via the appropriate process.

The section Purposes lists the purposes that are defined on the metrology configuration. Both active and inactive purposes are shown. They are sorted alphabetically after name. Four columns are shown in the table with purposes.

Field

Description

Name

Name can be Billing, Information, Voltage monitoring. Others may be available depending on what your Connexo delivery team installed.

Required

The purpose can be marked as required on the metrology configuration. Being required or not has an impact in processes such as validation, estimation and export. Required purposes also assure that data will be available for the usage point and that devices must be linked to the usage point in order to support these purposes.

Active

The required purpose of a metrology configuration becomes active on a usage point after the metrology configuration is linked to it. The optional purposes become inactive. Afterwards it is possible to (de)activate the optional purposes. See “(De)activate optional purposes”.

Status

A purpose is completed in case all meters and attributes that are required in calculations of outputs of the purpose have been specified.

Details of the selected purpose are shown in the details pane under the table. It has two sections.

Field

Description

Meter roles

The list of meter roles that are required for the calculation of purpose outputs. The roles are sorted alphabetically. In case of an unmeasured usage point you won’t see this group because the metrology configuration won’t have meter roles. If you have sufficient privileges, you can click the link on each item to see its landing page.

Attributes

The list of attributes that are required for the calculation of the purpose outputs. The attributes are sorted alphabetically. For each attribute a tooltip shows what attribute set it is part of.

View active calendars on a usage point

You can view the calendars that are active on a usage point to verify if the correct calendar is linked with your usage point.

5

Steps

1.   Open a usage point landing page.

2.   Click Calendars in the vertical navigation bar on the left.

If no metrology configuration is linked with the usage point, then the link Calendars isn’t shown.

All currently active calendars on the usage point are shown per category in a table including their activation date.

In the details pane you can see the calendar’s details under Current.

In the Planned section you see the scheduled calendar(s) with their future activation dates. The calendar that is scheduled to become active first is on top of the list.

3

Note

You can see all calendars that are linked to the usage point, i.e. including those from the past and the scheduled ones by looking at the tab Calendars on the History page of the selected usage point. See “View processes on a usage point”.

View all versions of a calendar on a usage point (History)

Over time, different calendars can be linked to a usage point. This history is stored in versions. You can view the evolution of these versions as a timeline.

5

Steps

1.   Open a usage point landing page.

2.   Click Calendars in the vertical navigation bar on the left.

3.   Select a calendar and select View timeline from the action button.

The usage point’s History page is opened on the Calendars tab. All calendars on the usage point are shown. This includes past, current and scheduled ones. For each calendar the corresponding details and active period are shown.

Preview a calendar on a usage point

You can view the contents of a calendar to verify if it fits your needs.

5

Steps

1.   Open a usage point landing page.

2.   Click Calendars in the vertical navigation bar on the left.

3.   Select a calendar and select View preview from the action button.

4.   In the filter on top of the page, select the start day for the week you want to view. You will see six days of the calendar starting from the selected day. The first day of the current week is set by default.

5.   The time of use calendar is graphically represented with the week days on the horizontal axis and the hours of the day on the vertical axis. For each day, you can see the tariff(s) used during that day. Hovering over each day segment shows a tooltip with the name, the start and the end hour of the tariff.

6.   In the details pane you see the periods, day types and tariffs that are used in the selected week.

View output lists of a metrology configuration purpose

View a list of outputs for the selected active purpose of the metrology configuration that is associated with the current usage point to see the data of that purpose.

5

Steps

1.   Open a usage point using the search page.

2.   Click the name of the needed purpose in the group Purposes in the vertical navigation bar to the left.

Name of the purpose and its status information is shown on top of the page in the left column.

Field

Description

Status

Indicates if all available data is present. Can be either complete or incomplete.

Validation status

Validation can be either active or inactive for this purpose.

All data validated

All data is validated if validation rules for all outputs and all inputs are executed for all readings.

Last checked

The moment when the last interval was checked by the validation process.

In the second column of the page you see the Validation schedule, Estimation schedule and Export schedule sections.

3.   CIick View schedule under Validation schedule to see the list of validation tasks that are configured for this purpose.

Each item is a link to its configuration page.

If there are no tasks to be shown, then click the link Manage validation tasks to take appropriate action.

3

Note

Select Validate now from the actions button to run the validation process for the current purpose.

 

4.   Click View schedule under Estimation schedule to see the list of estimation tasks that are configured for this purpose.

Each item is a link to its configuration page.

If there are no tasks to be shown, then click the link Manage estimation tasks to take appropriate action.

5.   At the bottom of the page there is a graphical representation of the reading quality. Hover over the info icon or the pie chart to see a breakdown of the value. For instance, valid values consist of estimated and not-suspect values.

6.   Click View schedule under Export schedule to see the list of export tasks that are configured for this purpose.

Each item is a link to its configuration page.

If there are no tasks to be shown, then click the link Manage export tasks to take further action.

7.   Click the tab Interval data view to see interval data for the purpose.

8.   Click the tab Register data view to see register data for the purpose.

View the output details of a metrology configuration purpose

View the output data of a channel or register of the selected purpose of the metrology configuration that is associated with the current usage point to view the suspects and trends. You can compare data and see where actions are needed. Each output that has suspect values after validation is marked with the corresponding icon .

1

Important

The usage point output data contains only the data from the period when an active metrology configuration is linked with the usage point. Gaps between metrology configurations aren’t shown. On a register data page, missing data isn’t shown but it is possible to add register readings manually. So, a register output either shows aggregated readings from linked meters or manually added readings.

5

Steps

1.   Open a usage point using the search page.

2.   Click the needed purpose in the group Purposes in the vertical navigation bar to the left.

3.   If needed, filter the list of outputs using the relative periods filter. Don’t forget to click Apply to execute the filter.

4.   Click the needed item in the table listing the outputs.

The corresponding output details page is opened. By default, the Readings tab is shown.

5.   Use the filter bar to filter the output data.

You can filter on start date and time and on the duration for the readings that you want to consider. You can also choose to see only suspect values. Don’t forget to click Apply to execute the filter. The default filter is set to the current date and time and to a one year duration.

6.   Click the blue up and down arrows in the upper right corner to switch outputs.

If at least one device is linked to the usage point within the specified period and if that device has some data for this period, then all the corresponding data is shown on a graph. The interval (date and time) is on the X-axis and the units are on the Y-axis.

The bars in the graph have different colours indicating validation results.

Colour

Description

Blue

Not validated.

Green

Validated without suspects.

Dark green

Estimated.

Red

Validated with suspects.

Yellow

Validated with informative notes.

7.   Hover over any spot on the graph to show a tooltip containing the date, actual data with units of measure, interval boundaries and reading qualities. A column chart is used in case of volume units such as kWh or m³. A line chart is used in case of flow units such as kW or m³/h.

8.   Drag the handles on the navigator bar under the graph to narrow down or extend the period for the data that is shown.

3

Note

When reducing the period, take into account that the minimum number of shown intervals is set to seven.

 

9.   Use the zoom functionality on top of the graph (on the left) to select the appropriate zoom level. The available zoom levels depend on the selected interval.

 

Under the graph, the readings are listed in a table showing the end of the interval and its value. A flag indicates the validation result.

Flag

Description

White flag

Not validated.

Red flag

Suspect.

Green flag

Estimated

The details of the selected reading are displayed in a details pane under the table. By default the General tab is shown. This tab shows the interval and the reading time. It also indicates if the output has been validated by all applicable validation rules in Connexo Insight and Connexo MultiSense and finally, the validation result is given. It can be suspect, not suspect or not validated.

The Reading value tab shows the value followed by the validation flag and the formula that is used in data calculations. If there are suspects, then they are shown as reading qualities with a link to the corresponding validation rule and an indication of the application where the information originates from.

3

Note

When viewing the output details of a register the data is simply displayed in a paged table showing measurement time and the value.

The Reading quality tab shows detailed reading quality information organised per category. There are four reading quality categories:

  • Device quality (1).

  • MDC quality (2).

  • MDM quality (3).

  • Third party quality (4 & 5).

Each reading quality is represented by a name followed by its code between brackets. The code is a number with three sections, e.g. 3.10.1. The first number indicates the reading quality category. When hovering over the information icon next to the reading quality, additional information is shown for a better understanding of the reading quality:

  • System.

  • Category.

  • Index.

The index attribute for estimated and validated values contains the ID of the rule that created the reading quality.

The unique reading qualities of the underlying data for aggregated readings are also collected.

Reading qualities are available to validators. This means that you can configure validation rules that generate suspects based on reading qualities. You can also do this using underlying reading qualities for aggregated readings.

3

Note

In case a reading quality for external data exists, i.e. coming from outside Connexo Insight,  then an icon is shown in the column Measurement time (register) of End of interval (channel) next to the concerned value in the table with readings.

View history data for a value on an output (channel/register)

You can see the historical values of channel/register data on an output to see who changed what and when.

5

Steps

1.   Open a usage point using the search page.

2.   Click the needed purpose in the group Purposes in the vertical navigation bar to the left.

3.   If needed, filter the list of outputs using the relative periods filter. Don’t forget to click Apply to execute the filter.

4.   Click the needed item in the table listing the outputs to see its values.

5.   Select a value in the table and then View history from the action button at the bottom of the row.

On the History page the current version of the value is shown along with eventual previous versions. The most recent version is on top. For each version you can see who changed the value, when it was changed and the value that was entered. Next to each value is an icon that indicates its validation status when it was the active version.

Parameter

Description

 

Suspect

 

Edited

 

Estimated

 

Confirmed

 

Restored

The cell in the column Changed on is empty for the initial version of an interval value if it is aggregated. The initial value, suspects (not edited), removed suspects (in case of revalidation) and restored values contain the latest aggregated value.

If no changes were made to the selected value, then only the current value is shown on the history page.

6.   Under the table there is a details pane with three tabs where you can see the same information as on the screen where you are looking at the current values. See “View the output details of a metrology configuration purpose”.

View history data for multiple values on an output (channel/register)

5

5.   Click View history from the bulk action button on top of the table.

The measurement time filter and the relative period are set by default on the same values as the filter on the readings tab of the output details page. This is the page from where the history page was opened in step 5. 

You have the option to show also values that haven’t been changed by deselecting the Changed data checkbox. By default, it is selected.

7.   On the History page you can see the historical values for each output value that is within the specified range. The rows in the table are grouped by measurement time of the output value. In each block, the current version of the value is shown along with eventual previous versions. The most recent version is on top. For each version you can see who changed the value, when it was changed and the value that was entered. Next to each value is an icon that indicates its validation status when it was the active version.

5

5.   Click the tab Specifications.

The following general information is shown on a form.

Field

Description

Name

Name of the output.

Reading type

Reading type attached to this output. Click the information icon to see the reading type details.

Interval

The interval of the channel.

Formula

The formula that is configured for this output to calculate its data.

In the validation section you find a summary of the validation activities for this purpose output.

Field

Description
