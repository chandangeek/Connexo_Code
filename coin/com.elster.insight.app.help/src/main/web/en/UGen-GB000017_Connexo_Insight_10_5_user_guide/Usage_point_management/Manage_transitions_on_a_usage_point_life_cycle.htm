Manage transitions on a usage point life cycle

Transitions describe how usage points move from one state to another during their life cycle.

View transitions on a usage point life cycle

You can view the transitions that are available to move from one state to another during the life cycle of a usage point. Each transition is associated with pretransition checks that verify if all conditions to proceed to this transition are met. If the pretransition checks have a positive result, then the configured auto actions are executed.

The pretransition checks and auto actions are listed in the transition details. Hover the information icon next to each value for more detailed information about the pretransition check or auto action.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click Transitions in the left navigation bar.

The available transitions are listed in a paged table and are sorted by their name.

Details of the selected transition are shown in the details pane under the table.

Add a transition on a usage point life cycle (including pretransition checks and auto actions)

You can add and configure a new transition to a usage point life cycle to add extra possibilities to move from one state to another.

1

Important

You can’t add a transition with the same name and ‘From state’ as an already existing transition.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click Transitions in the left navigation bar.

4.   Click Add transition to open the Add transition page.

5.   Enter a name in the field Name and specify a starting state in From.

The combination Name and From must be unique in the current usage point life cycle. The name by itself doesn’t need to be unique in the usage point life cycle.

The state selected in From is no longer available in To.

6.   Select the destination state in To.

The pretransition checks and auto actions that are available for this transition are added to the screen.

7.   In Privileges, specify the privileges that are needed to trigger the transition.

Privileges are assigned to roles in Connexo Admin. Refer to the Connexo Admin user guide for more detailed information about this topic.

8.   In Pretransition checks, select the optional pretransition checks that must be done to verify if the conditions for the current transition are met.

For out-of-the-box pretransition checks only the optional items are shown by default. Click Show all to see all pretransition checks including the mandatory ones. The mandatory pretransition checks are selected and can’t be edited. In case of custom pretransition checks all available items are displayed and there are no Show all and Show optional only links.

9.   In Auto actions, select the optional auto actions that must be done when the transition takes place and when the pretransition checks were finished successfully.

For out-of-the-box auto actions only the optional items are shown by default. Click Show all to see all auto actions including the mandatory ones. The mandatory auto actions are selected and can’t be edited. In case of custom auto actions all available items are displayed and there are no Show all and Show optional only links.

10.   Click Add.

The new transition is now listed in the overview table.

1

Important

The auto action Reset validation results will shift the last validation date on all outputs of the usage point to the date and time when the transition took place. If the transition date/time is in the middle of an output interval, then the last validation date equals the end of the previous interval. If the last validation date/time is earlier than the transition date/time, then this auto action won’t do anything for this output.

This auto action is selected by default on all the transitions in the standard usage point life cycle.

Edit a transition on a usage point life cycle

You can edit existing transitions to change what is verified and what actions must be done when a transition occurs.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click Transitions in the left navigation bar.

4.   Select a transition in the overview table.

5.   Click the action icon  at the end of the row or the actions button in the details pane and select Edit.

6.   You can edit the name, from and to state, required privileges to trigger the transition on a usage point, the pretransition checks that verify the conditions for the transition and the auto actions that must be done during the transition.

Limitations:

  • The combination Name and From must be unique in the current usage point life cycle. The name by itself doesn’t need to be unique in the usage point life cycle.

  • A state that has already been selected in From is no longer available in To.

  • In Pretransition checks and Auto actions, by default, only the optional items are shown. Click Show all to see the all auto actions including the mandatory ones. The mandatory auto actions are selected and can’t be edited. In case of custom auto actions all available items are displayed and there are no Show all and Show optional only links.

3

Note

Privileges are assigned to roles in Connexo Admin. Refer to the Connexo Admin user guide for more detailed information about this topic.

7.   Click Save.

Remove a transition from a usage point life cycle

You can remove transitions from a usage point life cycle if it is no longer relevant.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click Transitions in the left navigation bar.

4.   Select a transition in the overview table.

5.   Click the action icon  at the end of the row or the actions button in the details pane and select Remove.

6.   Click Remove to confirm the removal.
