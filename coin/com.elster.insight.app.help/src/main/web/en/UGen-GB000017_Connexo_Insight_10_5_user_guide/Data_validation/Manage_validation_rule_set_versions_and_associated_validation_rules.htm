Manage validation rule set versions and associated validation rules

Validation rule set versions are used to manage validation over time. They define the validity period of validation rules and different versions can have different validation rules associated with them. This mechanism allows to switch between applicable rules in line with a validation calendar.

View validation rule set versions and their associated validation rules

Discover the different versions with their proper validity periods and associated validation rules.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the desired validation rule set in the first table listing the validation rule sets.

3.   In the second table, under the validation rule sets, the versions of the selected validation rule set are listed. The validity period is shown in the column Period. Additionally, the number of active and inactive rules in this version are indicated.

3

Note

Alternatively, you can click the desired validation rule set to open its detail page. Then click Versions on the left navigation menu. Use the breadcrumb to navigate back to the validation rule set overview.

4.   Select the desired version to see the associated validation rules in the third table under the versions. In this table, the status of each rule is shown.

5.   Select the desired validation rule to see its full details in the details pane under the validation rules table.

Add a version to a validation rule set

You can add multiple versions to a validation rule set with their own validity period and associated validation rule sets. Only one version per validation rule set can be active simultaneously, i.e. the start date must be after that of an already existing version. If there already is a version, then the start date of a new version can’t be None.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the desired validation rule set.

3.   If there is no version associated yet, then click Add validation rule set version under the table listing the validation rule sets.

If at least one version is already associated with the selected validation rule set, adding further versions is only possible by opening the details page of the validation rule set:

a. Click desired validation rule link in the overview table to open its details page.

b. Click Versions in the left navigation menu.

c. Click the button Add version above the version overview table.

4.   Complete the fields in Add validation rule set version.

Field

Description

Description

Type a short description explaining the specific use of this validation rule set version.

Start

Define when the new version becomes valid.

Select None if you want no restriction in validity. In the versions overview table the validity period will be indicated as Always in the column Period.

If you specify a start date, this date will be shown as From /date/ in the column Period.

If there is already at least one version associated with the current validation rule set, then the validity period in the column Period of the currently valid version will automatically be changed into Until /date/ when the new version becomes valid where /date/ is the start date of the next version. Versions of the same validation rule set can’t be valid simultaneously.

5.   Click Add.

6.   The version is now listed in the versions overview table of the current validation rule set.

7.   Next task is to add validation rules to the new version.

Clone a validation rule set version

To save time you can base a new validation rule set version on an existing one.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the desired validation rule set.

3.   Select the desired version.

4.   Click the actions icon  and select Clone.

5.   Enter Description and Start date. See “Add a version to a validation rule set”.

6.   The new version is now listed in the versions overview table. The associated validation rules can be adjusted as needed.

Add a validation rule to a validation rule set version and activate it

As a final step you need to add each needed validation rule to the correct version of the validation rule set. Don’t forget to activate the added validation rules. In case you don’t want to use certain validation rules anymore, you can deactivate them.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the needed validation rule set.

3.   Click the correct version to open its detail page.

4.   Click Validation rules in the left navigation menu.

5.   Click Add validation rule.

6.   Complete the fields in Add validation rule.

Parameter

Description

Validation rule

Type a unique name to identify the validation rule.

Validator

Select how the validation must work, e.g. Check missing values, Reading qualities, Register increase, etc. Depending on the selected validator, additional parameters may require completion. The complete list of validators is shown under this procedure.

Reading types

Determine the data that will be validated by selecting the corresponding reading types. See “Add reading types”.

Data quality level

With the data quality level you can configure the behaviour of the validation rule if validation fails on a particular value.

 

Suspect: The data is invalid and needs attention. The concerned value will be subjected to estimation and is indicated in red on the graphic data representation.

Informative: The data is valid but potentially wrong. The concerned value will not be estimated and is indicated in yellow on the graphic data representation.

7.   Click Add.

By default, validation rules are inactive.

8.   To activate a validation rule, in the validation rule overview table, click the actions icon  and select Activate.

Add reading types

At several occasions it is required to define the concerned data by selecting reading types, e.g. when adding a validation rule.

5

Steps

1.   Click the button Add reading types.

2.   Use the filters on top of the page Add reading types to narrow down the number of reading types to choose from. Don’t forget to click Apply to apply your filters.

3.   Select the needed reading types.

Click on the information icon next to each reading type to see its CIM code details.

4.   Click Add.

Edit or remove validation rule set versions and validation rules

It is possible to edit or remove a validation rule set version or a validation rule to correct mistakes.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the needed version or rule.

3.   Click the actions icon  and select Edit or Remove.

4.   Apply the desired changes and click Save or confirm the removal message.

 

1

Important

Changing the name of a validation rule can potentially remove validation attributes that were changed on the usage point output. See “View/Edit/Restore validation attributes on reading type level”.

Deactivate a validation rule

You can deactivate a validation rule if you don’t want it to be applied any longer.

5

Steps

1.   Go to Administration > Data validation > Validation rule sets.

2.   Select the needed version and then the validation rule.

3.   Click the actions icon  and select Deactivate.
