Out-of-the-box estimators

In Connexo Insight multiple estimators are available out-of-the-box. They cover all the basic estimation needs. When selecting an estimator on an estimation rule, additional parameters appear on screen and need to be completed to finish the configuration of the estimation rule.

3

Note

The standard out-of-the-box estimators are marked by STD between square brackets at the end of the estimator’s name. For instance Average with samples [STD]. Custom estimators are described in the specific project documentation.

Average with samples

When estimating using average with samples every suspect channel delta value is estimated by calculating the average of reading values from this channel or other reading types from the same time and day in the past.

For this rule you need to configure the following parameters

Parameter

Description

Max number of consecutive sus­pects

The maximum number of allowed consecutive suspects. If this amount is exceeded, data isn’t estimated using this rule. It can then be estimated using the next estimation rule in the estima­tion rule set or it can be manually edited or estimated.

Allow negative values

Select if negative values can be used to estimate data.

Use advance readings

Use advance readings if you want to scale the estimated values using the values that are adjacent to the estimation interval. you can choose to use the channel’s own values or use another read­ing type.

  • Bulk reading: use the channel’s bulk reading for estimation.

  • Reading type: Select another reading type for estimation. This reading type needs to have readings at exactly the same day and time as the suspect data and also the same type of data.

Minimum sam­ples

The minimum amount of samples that is needed for estimation.

Maximum sam­ples

The maximum amount of samples that is needed for estimation.

Relative period

The period in which Connexo will search for samples:

  • All: Search in all past readings.

  • Period: Select a relative period to define where Connexo has to search for samples. These relative periods are configured in Connexo Admin. Refer to the Connexo Admin user guide for more detailed information about relative periods.

Reading quality

Mark data as projected to add the 3.12.0 reading quality. This is a temporary estimation.

Reading types

The data that will be estimated. Works only with delta reading type.

Estimation com­ment

Select one of the configured comments to understand later on how the readings were modified. Estimation comments are stored within specific reading qualities.

Equal distribution

To estimate delta values using the equal distribution rule, Connexo Insight  calculates the difference between the bulk or advance reading after the estimation interval and the bulk or advance reading before the estimation interval. It will then divide this difference by the amount of samples in the interval plus one. Connexo Insight  will then use this value as delta value for every suspect value in the interval.

Linear interpolation

With this rule Connexo Insight estimates the suspect bulk readings. The delta values are calculated based on these bulk readings, i.e. the difference between two consecutive bulk readings. To estimate the bulk readings, Connexo Insight  calculates the difference between the bulk reading after the estimation interval and the bulk reading before the interval and divides this difference by the amount of samples in the interval plus one.

In a second step Connexo Insight  adds this difference to the bulk reading before the estimation interval to calculate the first suspect reading. The second estimated suspect reading will be the previous estimated reading plus the calculated difference from the first step. Connexo Insight repeats these steps until all suspect readings are estimated.

Main/Check substitution

This rule substitutes suspect data in the main channel of the usage point with data from a reference channel on the same usage point but on another purpose at the exact same interval. If an interval record in the reference channel isn’t available or marked as suspect, then it won’t be taken into account. If the estimation block with this interval contains other suspects, they will also be skipped.

This rule only works for delta values on channels.

Parameter

Description

Check purpose

List of all purposes in Connexo. Data from an output with match­ing reading type on the selected purpose will be used to replace the estimated values.

Nearest average value (Day)

This rule replaces suspect interval records with an average value that is calculated over x sample interval records taken from the same reading type at the same time but from a different day. If the day of the suspect value is a bank holiday, then the rule won’t be applied. if the day used for the replacement value is a bank holiday, then the next day will be used. If no valid data for one or more of the interval records was found on a replacement day, then this day will be skipped entirely and the next replacement day will be used. If there aren’t sufficient valid samples available, then the rule won’t be applied for the whole period.

This rule only works for delta reading types with an interval type that is less than or equal to one day.

Number of sam­ples

Maximum threshold for the number of weeks where the samples can be searched. The maximum number of weeks must be higher than the needed number of samples.

Estimated and sample days can be discarded if they contain a chosen event code.

Enabled if the user decides to discard samples based on the event codes in a calendar. Contains all time of use calendars that are available in Connexo.

Event code

Enabled when a calendar is selected. Only one event code can be selected. The estimated days with the selected event code won’t be estimated and the days with samples containing the selected code won’t be used in calculations.

Power gap fill

Using the power gap fill rule Connexo Insight  replaces every suspect reading in the estimation interval except the last one by zero. The last reading equals the difference between the bulk reading after the interval and the bulk reading preceding the interval.

Reference substitution

This rule is identical to the “Main/Check substitution” but instead of using the reference output on the same usage point it uses data from another usage point with some flexibility in the output matching. Both reference and main output should be the same. A difference in the multiplier is allowed and values are compared with the multiplier taken into account.

The estimation rule will replace the suspect interval record with an interval record from the reference reading type (channel), taken at the exact same time, and mark it as estimated.

For this rule you need to complete the following parameters:

The reference usage point, purpose and reading type are specified on the output of the usage point where the estimation is run.

Steps

1.   Open the needed usage point landing page.

2.   Under Data on the vertical navigation bar, click thee needed output.

3.   Open the tab Estimation configuration.

4.   Click the action button and select Edit ‘Reference substitution’ to specify its editable attributes.

5.   The following attributes can be specified:

Parameter

Description

Check usage point

Contains all usage points in Connexo.

Check purpose

Contains all purposes in Connexo.

Check reading type

Type ahead finding all channels in Connexo.

3

Note

These attributes are required and mutually independent.

If the selected usage point doesn’t have the selected purpose, then the attributes won’t be saved. If the selected reading type isn’t available on the selected purpose of the usage point, then the attributes aren’t saved either.

Note also that the attributes are indicated by a black pencil once they have received a value.
