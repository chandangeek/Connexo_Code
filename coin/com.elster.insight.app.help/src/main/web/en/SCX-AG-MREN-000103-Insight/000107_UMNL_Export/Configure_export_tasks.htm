Configure export tasks

Understand export configuration

Add an export task

Add and configure an export task to run it on request or using a schedule.

1

Important

Before starting, you need to make sure that the needed usage point group is available. You will have to select one when selecting data for export.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Click the button Add export task.

The Add export task page is divided in four sections:

  • Name.

  • Schedule.

  • Data selection.

  • Destinations.

The sections are described under this procedure.

3.   Complete the required fields.

4.   Click Add.

The new export task is shown in the export task overview table.

The four sections of an export task configuration:

1.   Name is a required field and its value must be unique.

2.   In Log level, specify the needed log level for eventual troubleshooting.

Log level

Description

Error

An exception that stops the export task. The message will contain an error code and an error message.

Warning

This is the default value. An exception that could cause a problem in the future. Warnings aren’t blocking the export task. The message will contain a warning code and message.

Information

Informative message indicating that a step in the process has been executed as expected.

Debug

Detailed logging of all the events that take place during the process enabling you to isolate issues. This extensive contextual information is helpful during testing, debugging and development. It is indispensable for troubleshooting.

Trace

Most detailed log level. It is mainly used during development.

3.   Under Schedule there are two fields that define when the export task is running.

Field

Description

Recurrence

The frequency with which the export task is running. Select None if you want to run the task ad hoc.

Start on

Date and time when the export task must start. By default, this field’s value is the current date and time.

3

Note

By default, Recurrence is None and the Start on date is the current date and time.

If you select 1 month(s) in Every and a specific date and time in Start on, then the export task will run every month on the specified day and time.

4.   Under Data selection you select the data for export by selecting an item in the Data selector drop-down. After selecting the needed data selector the corresponding data selection properties are shown in Data selection and the Output format section is added.

5.   Under Output format, the available output formats in Data formatter and the corresponding output format properties are determined by the selected data selector.

6.   Finally, under Destinations, you specify where and how the export file(s) must be delivered. An export task must have at least one destination. It is possible to send an export file to multiple destinations at once. So, you don’t need to configure a new export task for each delivery address. Refer to “Add a new destination” for more information about configuring destinations.

7.   Click Add.

The new export task is added to the overview table on the Data exports page. The configured export will be executed on the specified time(s).

Add a new destination

To make an export task work, it must have at least one destination for the exported data.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Click the button Add export task.

Alternatively, select an export task and select Edit from the action icon  at the bottom of the selected row.

3.   In the section Destinations click Add destination.

4.   In Method, select the needed destination.

After specifying the method, a number of extra fields appear to configure how the selected method must work. The following methods are available:

  • Save file: Simply save the export file in a folder that is accessible from the application server that is running the export task. After selection of this method, the following parameters appear on screen:

    Parameter

    Description

    File name

    Example: DataExporter2. You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

    File extension

    Example: csv.

    File location

    Type only the relative path where the export file needs to be dropped. The root folder containing all export folders and files is defined on the application server that is handling the exports. See the Connexo Admin user guide for more detailed information about configuring an application server. Contact your Connexo administrator if you need assistance in this matter.

     

    Example: DataExporter2\MyFolder\MySubFolder

    If the folder doesn’t exist yet, then it will be created.

    You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

  • Mail: Send the export file by email. The mail server that is sending the emails must be configured in the config.properties file after installation of Connexo. The following properties are concerned:

  • mail.smtp.host

  • mail.smtp.port

  • mail.user

  • mail.password

  • mail.from: This is the e-mail address that will be used as the sender of the e-mails. Typically, it is an address in the style of donotreply@domain.com.

After selection of the e-mail method, the following parameters appear on screen:

Parameter

Description

To

Addressee email addresses. The addresses need to be separated by a semicolon.

Subject

Subject of the email message to which the export file is attached.

Attachment name

You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

Attachment extension

Example: csv.

  • FTP: Send the export file via a file transfer protocol. After selection of this method, the following parameters appear on screen:

    Parameter

    Description

    Hostname

    Type the correct host name.

    Port

    21 is default.

    User

    User to log on to the ftp server.

    Password

    Password for the ftp connection.

    File name

    You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

    File extension

    Example: csv.

    File location

    You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

  • FTPS: Send the export file via file transfer protocol over SSL (Secure Sockets Layer).

  • After selection of this method, the following parameters appear on screen:

    Parameter

    Description

    Hostname

    Type the correct host name.

    Port

    21 is default.

    User

    User to log on to the ftp server.

    Password

    Password for the ftp connection.

    File name

    You can add multiple tags. A full list of the available tags can be obtained by clicking the info icon next to the text box.

    File extension

    Example: csv.

    File location

5.   Click Add.

The new destination is shown in a table on the Add export task window.

View destinations

You can verify if destinations have been configured for an export task, how many of them and with what configuration details.

1.   Go to Administration > Data exchange > Export tasks.

2.   Select the needed export task in the overview table.

3.   On the details pane of the selected export task on the Export tasks page or on its detail page you can see only the number of destinations that are configured for this export task.

4.   Hover over the info icon next to the indicated number of destinations to see a summary of the configured data.

5.   To see the destinations in a table, you need to edit the export task.

Edit destinations

You can edit existing destinations of an export task to see the available destinations in a clear overview table and to make changes to their configuration. To edit a destination, you must first edit the export task it is associated with. See “Edit export tasks”.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Select the needed export task in the overview table.

3.   Select Edit from the actions icon at the bottom of the selected row.

Alternatively, select Edit from the Actions button on the details pane or detail page.

4.   On the Edit page, find the needed destination in the table under Destinations.

5.   Click Edit from the action icon at the bottom of the selected row.

6.   Make the needed changes on the Edit destination page.

7.   Click Save to keep your changes.

8.   Click again Save on the Edit export task page to store the export task with the new destination information.

Remove destinations

You can remove existing destinations of an export task if you no longer want the related export file to be delivered to a particular destination.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Select the needed export task in the overview table.

3.   Select Remove from the actions icon at the bottom of the selected row.

Alternatively, select Remove from the Actions button on the details pane or detail page.

The selected destination is no longer shown in the destinations table.

4.   Click Save to store the export task with the new data export information.

The export file that is related to the concerned export task will no longer be delivered to the removed destination.
