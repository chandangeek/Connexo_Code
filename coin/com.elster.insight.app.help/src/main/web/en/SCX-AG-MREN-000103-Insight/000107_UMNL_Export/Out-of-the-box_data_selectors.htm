Out-of-the-box data selectors

The standard out-of-the box data selectors are marked by STD between square brackets at the end of the data selector’s name. For instance Usage point readings data selector [STD]. Custom data selectors are described in the specific project documentation.

Add export task with the data selector Usage point readings

The Usage point readings exporter is used to export aggregated usage point readings for billing or other purposes.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Click the button Add export task.

3.   Type a unique name in Name.

4.   Select a log level.

5.   Enter the needed recurrence and start date and time.

6.   In Data selector, select Usage point readings.

The following properties are added to the page:

Part  of  

Field

Description

Usage point group

Select the usage point group to export data from.

Purpose

Select a purpose for the export.

Reading types

Click the button Add reading types to select the needed reading types. See “Add reading types”. At this point, you are indicating what data will be exported. See “View the metrology configuration landing page” to read more about how the outputs of a usage point are configured.

Export window

This is the period for which to export the selected data. These periods are based on the available relative periods. If you have sufficient privileges, you can add additional relative periods in Connexo Admin. See the Connexo Admin user guide for more information. The following export windows are available out-of-the-box:

  • Last 7 days.

  • Previous month.

  • Previous week.

  • This month.

  • This week.

  • Today.

  • Yesterday.

 

There are two options to use the export window. The export window can start from:

  • start of export window

Default value.

  • last exported data (continuous data)

This option overrules the start of the export window by the last export date. The exported file will contain data starting where the previous export ended. With this option, there won’t be overlapping data in the export, i.e. the data will be continuous. For instance, if the export window is Previous month and the export task was run manually two weeks ago, then the export will only contain the data of the last two weeks. This continuity can only be interrupted by choices in the export strategy regarding updated, missing and validated data.

In the preview of the five next periods (see bottom of this step) the start of the export period can’t be shown in an exact manner. This is because the last export date may vary between usage points from the selected usage point group. In this case, the start of the export window is shown as the defined start of the export window. So, the preview will look as if you had selected start of export window.

When the continuous blocks or data and the updated data strategies are combined, the start of the update window won’t shift if the last export date is before or after the export period.

 

If strategies are edited, the last exported date won’t be reset. The edit doesn’t work retro-active. This means that when the last exported date was already set and the export task is edited, then the historical data that was skipped won’t be exported.

Updated data

Export updated data for delivery to an external billing system.

 

There are two options available:

  • Don’t export.

  • Export within the update window.

Select the update window from the dropdown. In this dropdown you can select only the relative periods in the category Update window. You can add a new period by clicking the Add update window link in Connexo Admin. You will need corresponding privileges.

After selection of the update period, the start and end of the update window are added to the preview section.

Following, you can determine what needs to be exported from this period:

  • Updated values only.

  • Updated values and adjacent data within timeframe.

When selected, the update timeframe must be selected in the dropdown. In this dropdown you can select only the relative periods in the category Update timeframe. You can add a new period by clicking the Add update timeframe link in Connexo Admin. You will need corresponding privileges.

Missing data

With Missing data you determine your strategy to deliver complete data without gaps in the export file. There are two options:

  • Skip intervals with missing data (data with gaps).

Example: If there isn’t any data from 10:00 until 11:00, then the interval 10:00-11:00 won’t be exported.

  • Skip export window for reading types with missing data (complete data).

Example: If there isn’t any data from 10:00 until 11:00, then the entire export window for this reading type won’t be exported. The export complete data strategy is mostly used in combination with the Export continuous blocks of data strategy.

  • Skip export window for usage points with missing data (complete data).

If selected, then data for a usage point with missing data on the required reading types on the chosen purpose won’t be exported. If not selected, then no data from any purpose will be exported for a usage point with missing data on the required reading types.

If an interval or an export window isn’t exported, then this is logged with log level Info.

Validated data

With Validated data you determine your strategy to deliver validated data to market parties. There are three options:

  • Export all data (including suspect/not validated data)

All intervals or reading types are exported regardless of the validation result.

  • Skip intervals with suspect/not validated data

An interval which contains suspect data or not validated data won’t be exported.

  • Skip export window for reading types with suspect/not validated data

The entire export window for a reading type which contains suspect data or not validated data won’t be exported.

  • Skip export window for usage points with suspect/not validated data

If selected, then data for a usage point with suspect/not validated data on the required reading types on the chosen purpose won’t be exported. If not selected, then no data from any purpose will be exported for a usage point with suspect/not validated data on the required reading types.

If an interval or an export window isn’t exported, then this is logged with log level Info.

After the Schedule, Export window and - if needed - the Updated data fields have been completed, a preview with the next five default periods is shown.

7.   In the field Data formatter, CSV formatter is selected.

Additional formatters may be available.

The tooltip on info icon next to this field shows the format of the export file.

After selecting CSV formatter, the following properties are shown:

Parameter

Description

Identifier

Custom tag to add to the file name of all exported files related to the selected export task. The following symbols aren’t allowed: ": \ / * ? " < > and |

Update identifier

Custom tag to distinguish updated data from regular data in the export file. The following symbols aren’t allowed: ": \ / * ? " < > and |

Separator

Specify the symbol that separates values in the exported file. Can be a comma (default) or a semicolon.

3

Note

  • If no specific purpose is specified on the export task, then data is exported for all the purposes of the usage points.

  • Specific purpose data will only be exported if a purpose is specified on the export task.

  • New data is marked with the tag that is defined in the Identifier field from the Output formatter.

  • Updated data is marked with the Update identifier tag from the Output formatter.

 

The exported file has the following format:

Column

Type

Comment

Interval timestamp

String

Same for interval and non-interval reading types, i.e. channels and registers. The interval timestamp is in the ISO format. It is the complete date plus hours, minutes, seconds and a decimal fraction of a second (if different from 0). The format is YYYY-MM-DDThh:mm:ss.sTZD.

Example: 1997-07-16T19:20:30.45+01:00

MRID

String

 

Name

String, maximum 80 characters

The name can’t have the defined separator symbol, i.e. a comma or a semicolon, in its value.

Purpose

String, maximum 80 characters

 

Reading type

String, 18 digits that are separated by dots

The reading types are grouped per device. Within a reading type, the values are ordered chronologically on the interval timestamp. The oldest interval timestamp comes first.

Aggregated value

Big decimal

 

Validation result

String

If the data wasn’t validated, the value is an empty string.

 

If the data was validated and deemed valid, the value is Valid.

 

If the data is missing, the value is Missing.

 

If the data is present and suspect, the value is Suspect.
