Manage usage point life cycle on a usage point

You can link a particular life cycle to a usage point and move it through its life cycle by changing its state and starting a transition.

Set life cycle transition after usage point creation

When adding a new usage point you can specify what life cycle transition must be triggered after the creation of the usage point.

5

Steps

1.   Click Usage points in the quick access bar on the left side on any page.

2.   In Usage point life cycle management, click Add usage point.

3.   Complete the needed information for the new usage point.

4.   In the step Life cycle transition select the needed transition.

Change the state of a usage point

A transition to another state can be manually triggered via the actions button on a usage point landing page when the usage point’s context requires it.

5

Steps

1.   Search for a usage point that needs to move to another state in its life cycle.

You may use State from the default criteria to search for usage points that are currently in a particular state.

2.   Click the actions button to see the available transitions.

The available transitions depend on:

  • The current state of the selected usage point.

  • The privileges of the logged on user.

3.   Select the needed transition to open the wizard that will guide you through the necessary steps.

4.   In step one, Set properties, specify the transition date.

If Now or a date in the past is selected, then the transition starts without any delay and the usage point’s state will be changed as soon as all configured pretransition checks and auto actions are done.

If a date in the future is entered, then the transition is planned and listed in History (See “View planned transitions”). All configured pretransition checks and auto actions that are associated with this transition will be done at the set date and time. Only one transition at a time can be planned on a usage point.

In case at least one of the pretransition checks or auto actions fails, then the usage point’s state remains unchanged and its status can be tracked in History. See “View planned transitions”. Here, you will see what pretransition checks and auto actions have failed.

5.   Click Next to proceed to step two.

A progress bar is shown indicating that pretransition checks and auto actions are being executed or that the transition is being scheduled.

6.   Step two shows you the status of the transition, i.e. its result.

7.   Click Finish to close the wizard and return to usage point’s landing page.

If the transition was successful and if the transition date was set to Now or a date in the past, then you can see the new state in Usage point summary on the usage point’s landing page. In case of future transition date, wait until the scheduled date to check the usage point’s state.

8.   Click View history next to the state’s name to see a track of changed states for the current usage point’s life cycle.

1

Important

The transition won’t be done precisely on the specified date and time. Because the transition is done by a task in Connexo that is picked up with a certain frequency, i.e. five minutes, it is possible that the transition will start a little later than it was scheduled.

View planned transitions

You can see a chronological overview of the states in the current usage point’s life cycle to help you solving issues with transitions in the usage point life cycle.

5

Steps

1.   Search for a usage point that needs to move to another state in its life cycle.

You may use State from the default criteria to search for usage points that are currently in a particular state.

2.   Open a usage point.

3.   In Usage point summary, find the current usage point’s state and click View history to open the usage point’s history page.

Alternatively, click History in the vertical navigation bar on the left.

On the History page, make sure that you are looking at the tab Usage point life cycle.

The items in the table are shown in a table that is sorted by date with the most recent items on top.

Details of the selected history item are shown in the details pane under the table. Here, you will see eventual failed pretransitition checks and auto actions. Use the tooltips next to each item to see a more detailed description.

Abort a planned transition

You can cancel a transition on a usage point that is scheduled on a date in the future if it is no longer relevant.

5

Steps

1.   Search for the needed usage point on which a transition to another state in its life cycle is planned.

You may use State from the default criteria to search for usage points that are currently in a particular state.

2.   Open the needed usage point.

3.   In Usage point summary, find the current usage point’s state and click View history to open the usage point’s history page.

Alternatively, click History in the vertical navigation bar on the left.

On the History page, make sure that you are looking at the tab Usage point life cycle.

4.   In the overview table, find the planned transition and click the abort icon  in the Actions column.

5.   Click Abort to confirm removal.

The transition receives the status Aborted in the overview table.
