Understand the object landing page

When opening an object - for instance from the search results - in most cases you get an object landing page which is a quick overview of the object’s essential information. Widgets are used to create a dashboard-like page that shows you the object’s attributes, status, performance and other information that is relevant for the selected object. Click one of the links in the left object navigation bar or in the widgets to access more detailed information on a particular subject related to the selected object.

An object landing page always has a number of default widgets. Any additional widgets depend on the type of object that is shown. For example, if a usage point has no purpose with a validation configuration, then there will be no data completion widget.

A. Vertical navigation bar

From here you can navigate to detailed information about the current object. The links that are shown depend on the functions and the characteristics of the object. Click a link on the navigation menu to navigate away from the object landing page. If you want to keep it open, then open the links in a new window or tab.

B. What’s going on? widget

Quick overview of alarms, issues, processes and service calls that are open for the object. Click an item in the widget to see its landing page. This widget gives an idea of the object’s performance. See “The “What’s going on?” widget”

C. Metrology configuration

Quick overview of the metrology configuration of the selected usage point showing its purpose, the name of the active configuration and a list of the linked meters. Click the metrology configuration name to view more details on its landing page.

D. Usage point summary

This widget shows a summary of the object’s attributes. It gives you a clear view on what the object can do and where it fits in the structure. Click Manage usage point attributes to view all usage point attributes and edit them.

The “What’s going on?” widget

The What’s going on? widget tells you more about the object’s performance by listing issues, processes, alarms and service calls that are currently open for the object.

Filter the list using the drop-down filter in the upper right corner of the widget.

There are ten items per page that are arranged in two columns. Click the dots under the list to navigate to another page. The current page is marked by a white dot with black border.

Each item in the widget can be clicked to navigate to its landing page.

Each item is preceded by a circular icon which has a letter and a colour code.

The letter indicates the type of information in an item:

Letter

Hovering over the icon shows a tooltip with more detailed information like type, status, assignee and due date.

The colour of the tooltip title and the icon border match. This colour code indicates the severity (high, warning, none) and the assignment (current user, other user, unassigned).
