View service calls

View all service calls

On this page you see an overview of all service calls in Connexo. You can quickly find the service call you are looking for and view its specifications. From this page, you can click through to its full details or - if applicable - to an overview of its child service calls. This allows you to monitor and follow up the service calls in Connexo.

3

Note

In the list of service calls, the top level service calls (without parent) come first and are sorted after their modification date with the most recent ones first. Then, the child service calls come with the most recent ones first. Their level in the hierarchy is not taken into account.

5

Steps

1.   Go to Workspace > Service calls.

2.   Use the filter bar to narrow down the list of service calls. Can filter on the following fields:

a. ID.

Both service call ID and external reference ID, i.e. ID from the third party system.

b. Type.

c. Status.

d. Received date.

e. Modification date.

3.   The following columns are available in the service call overview table.

Parameter

Description

Service call

Unique ID provided by Connexo to each service call.

External reference

ID given by the requester attached to the service call, i.e. the third party system.

Type

The service call type as defined by your service & delivery team. See “Understand service call types”.

Status

The state of the top level service call. See “Understand service call type life cycles”. Note that the state depends on the activity of the child service calls. If one of them is in an ongoing state, this is reflected in the parent service call.

Received date

Date when the service call was received by Connexo.

Modification date

Date when an action has last been retried or a state has been changed.

4.   The following fields are available in the details pane.

Parameter

Description

Top level service call

ID of the first service call in the structure of nested service calls that the selected service call belongs to. This service call has no parent service call. If there is only one level above the selected service call, i.e. parent and top level service call are identical, then Top level service call is not shown. If there is not top level service call, then this field is not shown either.

Parent service call

ID of the service call that is the parent of the selected service call. If there is no parent service call, then this field is not shown.

Origin

Indication of the third party system from where the service call originates.

Children

Number of children of the selected service call, i.e. the service calls that are below the selected service call in the hierarchy of nested service calls. Clicking this value opens the parent service call’s landing page listing all of its child service calls.

States

Percentage of child service calls for each state.

View a service call landing page with children

On the service call landing page you view the service call’s details. If the selected service call has child service calls, the screen has two tabs. By default, the overview of the service call’s children is shown. On the other tab, you can view the parent service call’s details and logging information. This screen provides a handy summary of a parent service call.

View service calls on an object

By viewing the state of a running service call on an object such as a device or usage point, you can determine if further actions can be done on that object. It is also interesting to consult the history of service calls on an object to monitor what has happened in the past.

5

Steps

1.   Open an object using Search.

2.   On the object landing page, click Service calls in the left vertical menu.

3.   By default, the Running service calls are displayed.

The following fields are available in a paged table:

Parameter

Description

Service call ID

ID of the selected service call.

External reference

ID given by the requester attached to the service call, i.e. the third party system.

Type

The service call type as defined during setup. See “Understand service call types”.

Status

The state of the top level service call. See “Understand service call type life cycles”. Note that the state depends on the activity of the child service calls. If one of them is in an ongoing state, this is reflected in the parent service call.

Received date

Date when the service call was received by Connexo.

Modification date

Date when an action has last been retried or a state has been changed.

4.   Click the tab History to display the service calls that are no longer running on the selected object. The following fields are available in a paged table:

Parameter

Description

Service call

System ID of the service call with a link to the corresponding service call landing page.

External reference

ID given by the requester attached to the service call, i.e. the third party system.

Type

The service call type as defined during setup. See “Understand service call types”.

Status

The state of the top level service call. See “Understand service call type life cycles”. Note that the state depends on the activity of the child service calls. If one of them is in an ongoing state, this is reflected in the parent service call.

Received date

Date when the service call was received by Connexo.

Modification date

Date when an action has last been retried or a state has been changed.

The table can be filtered on:

a. ID.

Both service call ID and external reference ID, i.e. ID from the third party system.

b. Status.

c. Received date.

By default, the service calls of the past 60 days are displayed.

View commands on a device that are tracked by a service call

Refer to “Monitor commands on a device” for more information on commands on a device.

5

Steps

1.   Open a device using Search.

2.   On the left vertical menu bar, select Commands and select the desired command.

3.   In the details pane, locate the field Service call. This field’s value is the corresponding service call ID. Click it to open its landing page.
