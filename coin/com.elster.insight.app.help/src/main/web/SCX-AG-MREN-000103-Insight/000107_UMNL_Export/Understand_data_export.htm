Understand data export

Data export in Connexo

In Connexo Insight you can export aggregated readings for further analysis or for use in third-party systems such as billing systems. Exports can be done only once or on a regular basis by configuring a schedule. They can be sent to multiple destinations in a variety of output formats depending on the kind of data that you select for export. A number of exporters are available out-of-the-box. If needed, custom exporters can be considered, which allow you to export any data you need. Your Connexo service and delivery team can assist you with this.

Manage export configuration options

There are five important features in the configuration of export tasks:

1.   Schedule.

2.   Data selection.

3.   Output format.

4.   Destinations.

5.   Monitoring.

 

With the schedule you can configure the export task to run manually on a given point in time or you can specify a frequency to let it run on a regular basis at specific moments in time.

In data selection you define exactly what data is exported by selecting the needed data selector. Depending on this choice, additional options become available to fine-tune the data you need in your export. For example:

  • Select the usage point group containing the usage points for which you need to export data.

  • Select reading types to specify the type of data that will be included for each or the selected usage points.

  • Select the period for which to export the data and whether or not to include updated data, intervals with missing data, intervals with suspect or not validated data (See “Add a validation rule to a validation rule set version and activate it”). These export windows are based on the default relative periods. If you have sufficient privileges, you can create your own relative periods in Connexo Admin. See the Connexo Admin user guide for more information.

The above mentioned parameters don’t list all the possible data selection options. The exact options are explained for each exporter separately. See “Configure export tasks”.

Note

The standard out-of-the-box data selectors are marked by STD between square brackets at the end of the data selector’s name. For instance Usage point readings data selector [STD]. Custom data selectors are described in the specific project documentation.

In Output format you specify how you want the export file to be delivered. Some formatters are provided out-of-the-box in every Connexo installation. They are marked with STD between brackets at the end of the formatter’s name. For instance, CSV formatter [STD]. Eventual custom formatters are described in the customer project documentation.

In Destinations you determine where to deliver the export files. One file can be exported multiple times to different destinations or to just one. Export files can be delivered by email, to a file server, ftp server, etc.

Once configured and launched, all export tasks are monitored and their history and logging is tracked. This is an important feature allowing you to troubleshoot failing export tasks.
