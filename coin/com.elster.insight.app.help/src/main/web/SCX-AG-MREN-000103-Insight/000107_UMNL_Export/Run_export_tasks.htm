Run export tasks automatically using a schedule

Let export tasks run automatically at the configured intervals to have regular exports of the configured data.

To make automatic runs possible, the export task must be configured with a schedule. See “Add an export task”. After saving the export task, Connexo calculates the next execution time based on the configured recurrence and start date and time.

Example: On 5 May, configure an export task to export data of the previous month and start on 1 February. Result: Data of January is exported.

At the time of the next execution, Connexo informs the application server that a new run of the export task is due.

The application server picks up the export task and executes it.

After execution of the export task, the next calculation date and time is calculated. The calculation of the next scheduled occurrence is based on the current time and not on the time when the previous occurrence was scheduled.

Example: On 5 May, configure an export task to export data of the previous month and start on 1 February. Result: Data of January is exported. The export task is rescheduled to run again on 1 June to export the data of May. Result: The data for February, March and April won’t be exported. Suggestion: Create suitable relative periods to export the skipped months also.

Run an export task manually

You can run an export task manually if you need to execute it now or at another specific moment in time.

5

Steps

1.   Go to Administration > Data exchange > Export tasks.

2.   Select the needed export task.

3.   Select Run from the actions icon at the bottom of the selected row.

Alternatively, select Run from the Actions button on the details pane or detail page.

4.   Click Run to confirm that the export task will be run at the earliest possible time.

The value in the column Status changes to Ongoing since /date when run was initiated/.

You can monitor progress of the triggered export task in the overview table on the Export tasks page.
