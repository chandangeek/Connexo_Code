Understand data estimation configuration

The configuration of data estimation is similar to that of data validation. Estimation rule sets group estimation rules that are associated with a metrology configuration purpose that - in its turn - is associated with one or more usage points.

Within the rule set you define the order of the rules to determine what estimation rule will be applied first. Typically, the estimation rules that yield the most realistic values are on top in the estimation rule set. Once a value has been estimated using one of the estimation rules, this value is no longer available for estimation by the other estimation rules in the estimation rule set.

In Connexo Insight multiple estimators are available out-of-the-box. They cover all the basic estimation needs. These estimators still need configuration in the estimation rule. You can read more about the default estimators in the topic “Add an estimation rule to an estimation rule set and activate it”. Your Connexo delivery team can help you with custom estimators.

After creating the estimation rule sets, estimation tasks are associated with a usage point group to enable scheduled estimation on regular intervals. Data estimation can also be executed on request.
