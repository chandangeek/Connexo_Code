Execute estimation

Estimate data on schedule with estimation tasks

You can configure estimation tasks to run data estimation automatically using schedules.

View estimation tasks

Get a complete overview of the available estimation tasks to see if your estimation needs are covered.

5

Steps

1.   Go to Administration > Data estimation > Estimation tasks.

2.   The available estimation tasks are shown in a table with the status of their last execution and the timing of their next run.

3.   Further details are shown in the details pane under the table.

4.   Click the linked estimation task name in the column Name to show the estimation task’s full details on a separate page.

View history of an estimation task

See when a validation task has run and with what result.

5

Steps

1.   Go to Administration > Data estimation > Estimation tasks.

2.   Click the link on the estimation task name to open its detail page.

3.   Click History in the left navigation menu.

4.   On the History page, define a filter to narrow down the range of historical estimation task runs that you want to see.

5.   The corresponding estimation task runs are shown in a table with details for each line in a details pane below. For each run, duration and status are shown.

6.   Click the actions icon  and select View log to see the corresponding detailed log file for the selected validation task run.

Add an estimation task

Add an estimation task to make it available when adding estimation tasks to a metrology configuration purpose.

5

Steps

1.   Go to Administration > Data estimation > Estimation tasks.

2.   Click Add estimation task.

3.   In the field Name, type a name for the new estimation task.

4.   In Log level, define how much detail you want to log what is happening during an estimation task.

Log level

Description

Error

Exception that stops the estimation task. Contains an error code.

Warning

Default value. Exception that indicates a potential issue. Isn’t blocking. contains a warning code.

Information

Including informative messages indicating that a step in the process has been executed as expected.

Debug

Detailed logging of all the events that take place during the process enabling you to isolate issues. Indispensable for efficient troubleshooting.

Trace

Most detailed log level. It is mainly used during development.

5.   Select Re-validate estimated data if you want Connexo to automatically revalidate the estimated data.

6.   In Usage point group, select the usage point group with which the estimation task must be associated.

7.   In Purpose, you can specify the configuration contract for this estimation task.

If specified, estimation will only occur on data in the concerned purpose. If no selection is made, then estimation will be run on all data for the usage points in the usage point group independent of the purpose.

8.   In the section Schedule, define the schedule for this estimation task. You need to specify a recurrence and a start date.

9.   In the section Data options, you can select a relative period to narrow down the scope of the estimation task.

Only the relative periods with the category Data estimation task are available.

10.   Click Add.

The new estimation task is now visible in the table with estimation tasks. It will also be shown on the associated metrology configuration. See “View and manage the estimation rule sets on a metrology configuration”.

Edit or remove an estimation task

You can edit or remove estimation tasks to correct mistakes.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   Click the actions icon of the needed validation task.

3.   Select Edit or Remove.

4.   Make the necessary edits and click Save or confirm the removal.
