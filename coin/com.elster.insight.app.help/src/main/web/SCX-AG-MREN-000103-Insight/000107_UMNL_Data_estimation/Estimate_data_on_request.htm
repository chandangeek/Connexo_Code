Estimate data on request

You can estimate usage point data manually if there is an immediate need for making the data more reliable.

Run estimation on data of a particular usage point

You can run data estimation on one particular usage point. This will trigger the estimation rule set that is associated with the metrology configuration purpose that is linked with the concerned usage point.

5

Steps

1.   Open the needed usage point using Search.

2.   Click the needed purpose in the in the left navigation menu under Data.

3.   Select Estimate now from the actions button.

4.   Choose if you want Connexo to automatically revalidate the estimated data.

5.   Click Estimate.

You are able to monitor the estimation’s progress and when finished a message is shown.

1

Important

Estimation is done immediately. If the estimation process takes more than 60 seconds, then estimation will continue in the background and you can proceed with other tasks. Note that no logging is created. So, if estimation fails, it won’t be possible to check why it failed. If you run estimation from the estimation task, then logging is created. See “Run estimation on data for all usage points associated with a particular metrology configuration and purpose”.

Run estimation on data for all usage points associated with a particular metrology configuration and purpose

You can run an estimation task manually to perform data estimation for all of the usage points linked with the metrology configuration purpose with which the concerned estimation task is associated.

5

Steps

1.   Go to Administration > Data estimation > Estimation tasks.

2.   Select the needed estimation task.

3.   Click the actions icon  at the end of its line and select Run.

4.   In the confirmation popup, click Run.

The estimation task will be queued to run at the earliest possible time.

1

Important

Estimation is queued to be executed at the earliest possible time. In this case, logging will be created. So, if estimation fails, it will be possible to check why it failed. If you run estimation from a usage point, then no logging is created. See “Run estimation on data of a particular usage point”.
