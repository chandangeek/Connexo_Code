Validate data on request

You can validate usage point data manually if there is suspicion that it may contain errors or if the data wasn’t validated automatically by the validation tasks.

Run validation on data of a particular usage point

You can run data validation on one particular usage point. This will trigger the validation rule set that is associated with the metrology configuration purpose that is linked with the concerned usage point. It is even possible to revalidate data that has been validated before.

5

Steps

1.   Open the needed usage point using Search.

2.   Click the needed purpose in the left navigation menu under Data.

3.   Select Validate now from the actions button.

4.   In the popup choose the date from which the data validation should be started.

By default, validation will start from the last validation date. The last validation date is added between brackets.

5.   Click Validate.

Before the actual validation starts, it is first verified if there are any validation rules configured for the selected purpose of the usage point starting from the specified date. This check takes all outputs from the same purpose on all metrology configurations starting from the specified date into consideration. Also the usage point states that were defined on the metrology configuration purpose of the associated validation rule set are considered. If states were specified, then the interval data from a period when the usage point was in a particular state must be validated by the specified corresponding validation rules. If no states were specified on the metrology configuration purpose of the associated validation rule set, then all the validation rules must be applied to all usage point data no what state the usage point is in at that moment.

 If validation isn’t configured on the selected purpose of the concerned usage point from the specified date, then the user is notified.

You are able to monitor the validation’s progress and when finished a message is shown.

1

Important

Validation is done immediately. If the validation process takes more than 60 seconds, then validation will continue in the background and you can proceed with other tasks. Note that no logging is created. So, if validation fails, it won’t be possible to check why it failed. If you run validation from the validation task, then logging is created. See “Run validation on data for all usage points associated with a particular metrology configuration and purpose”.

Run validation on data for all usage points associated with a particular metrology configuration and purpose

You can run a validation task manually to perform data validation for all of the usage points linked with the metrology configuration purpose with which the concerned validation task is associated.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   Select the needed validation task.

3.   Click the actions icon  at the end of its line and select Run.

4.   In the confirmation popup, click Run.

The validation task will be queued to run at the earliest possible time.

1

Important

Validation is queued to be executed at the earliest possible time. In this case, logging will be created. So, if validation fails, it will be possible to check why it failed. If you run validation from a usage point, then no logging is created. See “Run validation on data of a particular usage point”.
