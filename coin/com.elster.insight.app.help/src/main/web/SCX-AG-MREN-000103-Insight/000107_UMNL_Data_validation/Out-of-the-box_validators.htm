Out-of-the-box validators

The standard out-of-the-box validators are marked by STD between square brackets at the end of the validator’s name. For instance Meter advance [STD]. Custom validators are described in the specific project documentation.

Check missing values

Check for missing data. These gaps may be caused by failed communication or a device problem preventing the device from registering or sending its data. This data can be provided at a later stage using estimation, editing or a later data collection.

As opposed to the Consecutive zeros validator, this validator checks for instances where no data is registered at all.

This validator only works for interval data (load profiles). It hasn’t got any additional parameters.

Parameter

Description

Reading types

Data that will be validated.

Data quality level

The flag that is used in case validation fails.

Consecutive zeros

Create suspect and informative flags when consecutive zero values are found within a configured period.

Because certain loads - such as solar panels - are never exactly zero a threshold can be configured to count values that aren’t zero but that are less than the threshold parameter as zero.

This validator works for delta reading types on channels.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Minimum period

Two hours is the default value. The minimum period can be overwritten by the reading type.

Maximum period

One day is the default value. the maximum period can be overwritten by the reading type.

Minimum threshold

Certain loads are never exactly zero, e.g. solar panels. In these cases, a threshold is needed below which all values are considered to be zero.

Zero is the default value. It can be overwritten by the reading type.

Check retroactively

If selected, the validator will start looking for a consecutive period of zeros starting from the moment when the last check occurred minus the configured maximum period. So, the block of consecutive zeros at the end of the previously validated period can be added to the consecutive zeros at the beginning of the newly validated data. Consequently, the validation status of the data before the last checked moment can change from valid into suspect. On the other hand, suspect data can’t change into valid data.

3

Note

Detecting large periods of consecutive zeros can also be done using the average consumption validation check.

Main/check comparison

Compare delta values on the validated output channel on a usage point to those on another output channel with the same reading type but on a different purpose. If compared values with an identical time and date aren’t equal or if the difference exceeds the configured parameter, then a suspect is created.

This rule compares the values of a MAIN meter with a CHECK meter. The absolute difference is compared with a limit that can be a percentage or an absolute value. The values are flagged as suspect if they are out of tolerance or if the value of the check meter is also flagged.

If the absolute difference is below a defined minimum threshold, then the comparison won’t be done to avoid imprecisions which are typically registered when dealing with very low loads.

If there is no reference data available in the CHECK meter, then the rule can be configured to pass.

The rule can be configured to use only validated data..

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Check purpose

List of available purposes in Connexo. Data from an output with a matching reading type on the selected purpose is used to compare it to the validated values.

Maximum absolute difference

If the difference between the two compared values is bigger than the configured maximum absolute difference, then the value will be marked as suspect. Can be overwritten by the reading type.

Pass if no reference data

If no data is available on the reference channel and this checkbox is selected, then no further checks are done. The interval is marked as valid and the rule moves to the next interval. If this checkbox isn’t selected, then the validation is stopped where the last reference data was found for the channel.

Use validated data

When selected, only validated data is used.

If data is available on the reference channel but it isn’t validated and this checkbox is selected, then the validation is stopped where the last validated reference data was found for the channel. If this checkobx isn’t selected, then the validation continues.

Minimum threshold

Configure a minimum threshold to make the validator move to the next interval if the difference is minimal. Can be overwritten by the reading type.

Meter advance

Check the sum of the interval data between two cumulative register readings (discreet values that match the interval data) and compare it to the register advance, i.e. the difference between two register readings while applying a threshold range. This threshold can be a percentage or an absolute value. If the absolute difference is outside the threshold range, then the whole interval data period is flagged as suspect.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Reference reading type

Select the register to which the channel will be compared.

Maximum absolute difference

If the absolute difference between registers delta and sum of delta interval value is larger than the configured maximum absolute difference, then the period between the two register readings is marked as suspect. Can be overwritten by the reading type.

Reference period

If a reference period is defined, the validator can skip the data from the reference period if no cumulative register readings have been found for a period that is longer than the reference period. For instance, if no readings were found for more than one week, the rule can skip one week of data. When a new reading becomes available, the rule will only flag as suspect the part of the data that hadn’t been validated yet.

Minimum threshold

If the difference between the two registers is less than the minimum threshold, then the data for the validated period is marked as valid. This parameter is meant to avoid suspect flags when the difference is minimal.

3

Note

If a new meter (meter activation) is made, then the data from before the first available register reading will be skipped since the register reading from the previous meter can’t be used.

Reading qualities

Add suspect and informative flags on data based on reading qualities. This rule detects reading qualities at the interval level and matches them with a list of predefined reading qualities that should be flagged.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Reading qualities

When interval data with corresponding reading qualities is received, this data will be marked as suspect or informative.

3

Note

This rule won’t create reading quality flags. The reading quality flags on interval or register records need to be imported from a data source or set by the protocol driver that is used to collect the data from the device. Refer to the protocol driver or interface specification to understand which reading qualities are supported.

Reference comparison

This rule is identical to “Main/check comparison” but it compares a reading type with another reference channel. The validator compares delta values on the validated output channel on a usage point to another output channel from a different usage point with a similar reading type - allowed flexibility in the multiplier. If the compared values with identical time and date aren’t equal or if the difference exceeds the configured parameter, then a suspect is created. If the minimum threshold is configured, then the check for the interval is skipped and the validation moves to the next interval.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Maximum absolute difference

The maximum absolute value can be defined as a natural number or as a percentage. The percentage value can’t be bigger than 100.

Pass if no reference data

If no data is available on the reference channel and this checkbox is selected, then no further checks are done. The interval is marked as valid and the rule moves to the next interval. If this checkbox isn’t selected, then the validation is stopped where the last reference data was found for the channel.

Use validated data

When selected, only validated data is used.

If data is available on the reference channel but it isn’t validated and this checkbox is selected, then the validation is stopped where the last validated reference data was found for the channel. If this checkobx isn’t selected, then the validation continues.

Minimum threshold

Configure a minimum threshold to make the validator move to the next interval if the difference is minimal. Can be overwritten by the reading type.

3

Note

This rule should allow a certain flexibility between the source and the reference because the reading types may not always perfectly match.

Register increase

Check consumption over time. If the value of a register is lower or equal than the previous register value, then this value is marked as suspicious.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Fail equal data

If selected validation fails if the consumption is equal to the previous consumption. If it isn’t selected, then the value is valid.

Threshold violation

Check each interval value against a minimum and a maximum value. Values outside these boundaries are marked as suspicious.

Parameter

Description

Reading types

The data that will be validated.

Data quality level

The flag that is used in case validation fails.

Minimum

Any value below this threshold is marked as suspect or information. Can be overwritten by the reading type.

Maximum

Any value above this threshold is marked as suspect or information. Can be overwritten by the reading type.
