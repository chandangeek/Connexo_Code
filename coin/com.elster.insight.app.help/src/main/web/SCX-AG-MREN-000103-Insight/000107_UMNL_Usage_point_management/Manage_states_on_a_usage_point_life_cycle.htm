Manage states on a usage point life cycle

States are the different milestones in the life of a usage point. They are important for understanding the status and capabilities of a usage point.

View states on a usage point life cycle

You can view the states that are available in a usage point life cycle to understand the evolution of the usage point.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click States in the navigation bar on the left to open the page States.

The states are listed in a paged table and are sorted by their name. The initial state is marked by and icon  in the column Initial.

Details of the selected state are shown in the details pane under the table.

Set a state as the initial state on a usage point life cycle

You can set one of the states of a usage point life cycle as the initial state to indicate where the usage point life cycle starts.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle.

3.   Click States in the left navigation bar.

4.   Select the state that you want to mark as the initial one.

5.   Click the action icon  at the end of the row or the actions button in the details pane and select Set as initial state.

The value in the field Initial is changed to Yes and an icon  is added in the column Initial of the initial state.

3

Note

There can be only one initial state for each usage point life cycle. When changing the initial state, the previous one is replaced by the current one.

Add a state on a usage point life cycle

You can add states to a usage point cycle to make the usage point life cycle better match your needs.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click States in the left navigation bar.

4.   Click Add state.

5.   Enter a unique name and specify an appropriate stage.

6.   In On entry, click Add processes to open the Add processes page and select the processes that must run when entering this state.

7.   In On exit, click Add processes to open the Add processes page and select the processes that must run when leaving this state.

Added processes can be removed afterwards by deselecting the corresponding checkbox.

8.   Click Add.

The new state is now listed in the overview table and available in the current usage point life cycle.

3

Note

Only processes of the type TransitionBusinessProcess can be added to a state. Other processes may have been defined in Connexo Flow that can’t be used on a transition.

Edit a state on a usage point life cycle

You can edit the state of a usage point life cycle to make it match your needs.

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click States in the left navigation bar.

4.   Select the needed state.

5.   Click the action icon  at the end of the row or the actions button in the details pane and select Edit.

6.   Click Save.

3

Note

When adding a process to a state that is currently associated with one or more usage points, then this process will only be triggered on the usage point if it enters/exits the mentioned usage point.

When removing a process from a state that is currently associated with one or more usage points and where the existing process is currently running, the existing process will keep running and the new one will be applicable upon the next entry/exit for this state.

Remove a state from a usage point life cycle

You can remove a state from a usage point life cycle to correct eventual mistakes or if a state is just no longer relevant in the current state.

Restrictions:

  • You can’t remove the initial state.

  • Each usage point life cycle must have at least one state.

  • A state that is currently in transition can’t be removed.

  • A state that is currently used by a usage point can’t be removed.

 

5

Steps

1.   Go to Administration > Usage point life cycle management > Usage point life cycles.

2.   Click the name of the needed usage point life cycle to open its landing page.

3.   Click States in the left navigation bar.

4.   Select the needed state.

5.   Click the action icon  at the end of the row or the actions button in the details pane and select Remove.

6.   Click Remove to confirm the removal.

The state is no longer shown in the overview table.
