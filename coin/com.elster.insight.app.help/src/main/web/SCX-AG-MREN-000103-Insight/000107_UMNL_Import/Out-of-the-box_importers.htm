Out-of-the-box importers

The standard out-of-the box importers are marked by STD between square brackets at the end of the importer’s name. For instance Correction factors importer [STD]. Custom importers are described in the specific project documentation.

Correction factors importer

This importer is used to import correction factor values for a specific period of time to enable the calculation of data on usage points. The import file can contain data for multiple correction factor specifications with the same Interval and Duration.

This importer’s file format is as following:

Header title

Values type

Comment

Timestamp

Start date and time of an interval. Can be configured on the importer.

This must always be the first column. The lines in the file must be ordered by this value with the oldest one on top. Example: '02.03.2017 0:00’

<Correction factor specification 1 name>

Floating point values parsed as big decimal.

This must always be the second column in the file. Example: '0.86002504'

<Correction factor specification N name>

Floating point values parsed as big decimal.

A second correction factor and all other specifications are optional in the import file. Example: '0.86002504'

3

Note

Floating point values are used to represent numbers of different orders of magnitude with a fixed number of digits. They are a formulaic representation of real numbers as an approximation to support a trade-off between range and precision. A number is represented by a fixed number, i.e. the significand and scaled using an exponent in a fixed base. A number that can be represented exactly is formed with this formula:

 

significand x baseexponent

 

The decimal point can be placed anywhere relative to the significant digits of the number. In fact, the floating point representation can be seen as a scientific notation.

 

Example:

7.6543 = 76543 x 10-4
