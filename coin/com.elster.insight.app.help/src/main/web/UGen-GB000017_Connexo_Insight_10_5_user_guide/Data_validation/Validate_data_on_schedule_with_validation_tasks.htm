Validate data on schedule with validation tasks

You can configure validation tasks to run data validation automatically using schedules.

View validation tasks

Get a complete overview of the available validation tasks to see if your validation needs are covered.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   The available validation tasks are shown in a table with the status of their last execution and the timing of their next run.

3.   Further details are shown in the details pane under the table.

4.   Click the linked validation task name in the column Name to show the validation task’s full details on a separate page.

View history of a validation task

See when a validation task has run and with what result.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   Click the link on the validation task name to open its detail page.

3.   Click History in the left navigation menu.

4.   On the History page, define a filter to narrow down the range of historical validation task runs that you want to see.

5.   The corresponding validation task runs are shown in a table with details for each line in a details pane below. For each run, duration and status are shown.

6.   Click the actions icon  and select View log to see the corresponding detailed log file for the selected validation task run.

Add a validation task

Add a validation task to make it available when adding validation tasks to a metrology configuration purpose.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   Click Add validation task.

3.   In the field Name, type a name for the new validation task.

4.   In Log level, define how much detail you want to log what is happening during an estimation task.

Log level

Description

Error

Exception that stops the estimation task. Contains an error code.

Warning

Default value. Exception that indicates a potential issue. Isn’t blocking. Contains a warning code.

Information

Including informative messages indicating that a step in the process has been executed as expected.

Debug

Detailed logging of all the events that take place during the process enabling you to isolate issues. Indispensable for efficient troubleshooting.

Trace

Most detailed log level. It is mainly used during development.

5.   In Usage point group, select the usage point group with which the validation task must be associated.

6.   In Purpose, you can specify the configuration contract for this validation task.

If specified, validation will only occur on data in the concerned purpose. If no selection is made, then validation will be run on all data for the usage points in the usage point group independent of the purpose.

7.   In the section Schedule, define the schedule for this validation task. You need to specify a recurrence and a start date.

8.   Click Add.

The new validation task is now visible in the table with validation tasks. It will also be shown on the associated metrology configuration. See “View and manage the validation rule sets on a metrology configuration”.

Edit or remove a validation task

You can edit or remove validation tasks to correct mistakes.

5

Steps

1.   Go to Administration > Data validation > Validation tasks.

2.   Click the actions icon of the needed validation task.

3.   Select Edit or Remove.

4.   Make the necessary edits and click Save or confirm the removal.
