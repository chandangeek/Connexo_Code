Edit data on the usage point

Add a register reading

In case the output is a register you have the possibility to manually add a reading.

5

Steps

1.   On the output details page of a register on the purpose of a usage point, click Add reading.

2.   Specify the measurements time and the value of the reading.

3.   Click Add.

The value is now shown in the table with register readings.

1

Important

It isn’t possible to add a register reading if the metrology configuration wasn’t yet linked to the usage point at the measurement time.

Copy from reference channel and paste into one or more intervals of the main channel

Copy values from another reading type and paste them in your current output. You can paste values into one interval of the main channel or into multiple intervals simultaneously.

5

Steps

1.   On the output details page of a channel on the purpose of a usage point, select a row and select Copy from reference from the actions button at the end of the row.

2.   Complete the attributes in the dialogue Copy from reference to specify what data must be copied.

Parameter

Description

Check usage point

Select the usage point where the reference value must be pasted.

Check purpose

Select the purpose on the previously selected usage point where the reference value must be pasted.

Check reading type

Select the reading type of the previously selected purpose and usage point where the reference value must be pasted.

Start date/time

Set by default to the oldest interval (in case multiple intervals have been selected to copy values to). Setting a different date and time will cause an offset when copying values. For instance, if the original date is 29 March and the time is 14:00 and following the start date and time is changed to 27 March at 13:00, then the values on the main channel will be replaced with check values with a timestamp equal to the original date and time minus 2 days and one hour.

Allow suspect data

Select to also copy suspect data.

Complete period

Select to avoid copying the values for the selected intervals if at least one value for replacement on the check channel is missing or suspect - if Allow suspect data is selected.

Estimation comment

Select one of the configured comments to understand later on how the readings were modified. Estimation comments are stored within specific reading qualities. See “Understand the field Estimation comment”.

Projected value

Mark data as projected to add the 3.12.0 reading quality. This is a temporary estimation. See “Understand projected readings”.

3.   Click Copy.

4.   On the output details page, click Save changes.

1

Important

In order to go faster you can copy reference channel values into multiple intervals of the channel simultaneously. Just select one or more rows while holding the CTRL-key. Then, select Copy from reference from the bulk action button and specify all the needed attributes to determine exactly what data should be copied. To finish, click Copy and then Save changes.

Correct one or more values

You can correct values by scaling them in line with certain rules. You can use a factor to multiply, add or subtract an amount to the selected value. You can choose to correct only suspect and estimated values by checking Limit values and you can mark the value as projected. See “Understand projected readings”.

5

Steps

1.   On the output details page on the purpose of a usage point, select a row and select Correct from the actions button at the end of the row.

2.   Complete the attributes in the dialogue Correct to specify how the value must be corrected.

Parameter

Description

Type

The kind of calculation that is needed to correct the selected value. It can be multiplied with added to the correction amount or it can subtract the correction amount.

Correction amount

The factor that is used to make the corrective calculations.

Limit values

Select to correct only suspect and estimated values.

Projected value

Mark data as projected to add the 3.12.0 reading quality. This is a temporary estimation. See “Understand projected readings”.

Estimation comment

Select one of the configured comments to understand later on how the readings were modified. Estimation comments are stored within specific reading qualities. See “Understand the field Estimation comment”.

3.   Click Correct.

4.   On the output details page, click Save changes.

 

1

Important

Then, select Correct from the bulk action button and specify all the needed attributes to determine exactly how the values must be corrected. To finish, click Copy and then Save changes.

Edit the estimation comment

After you have edited a reading inline, you can select a pre-defined estimation comment for a better understanding later on about how the readings were modified.

5

Steps

1.   On the output details page on the purpose of a usage point, select a channel reading.

2.    Select Edit estimation comment from the actions button at the end of the row.

3.   In Estimation comment, select the needed predefined estimation comment.

4.   Click Edit.

An additional attribute Estimation comment is added to the tab Reading values and the estimation comment also appears in the tooltip on the edit icon in the table.

5.   Click Save changes.

3

Note

  • In case there are multiple reading types (e.g. Collected, Calculated), then this estimation comment can be shown for both calculated and collected value.

  • Estimation comments are only available on edited and estimated readings.

  • The estimation comment can be cleared by editing the estimation comment and not specifying any comment.

Pre-validate manually updated channel data

You can pre-validate updated data before saving to make sure that no suspects will appear on the modified interval of data later on and avoid the need to revalidate data manually after editing.

5

Steps

1.   On the output details page on the purpose of a usage point, click Pre-validate.

This button is above the table. It is enabled only if at least one reading has been updated. All readings starting from the first modified reading until the latest visible reading are pre-validated. Reading qualities and estimation comments that aren’t yet saved must be taken into account during pre-validation.

2.   Save changes or close and review the potential suspect values.

When the pre-validation is finished, a popup is shown. If no potential suspects were found, then you can save the applied changes by clicking Save changes. In the other case, you can click Close to have a look at the potential suspects that have been identified.

Potential suspect values that have been identified by pre-validation are marked in the table with an empty red flag. The tooltip on this flag shows the applicable validation rule names. The potentially suspect readings are indicated on the graph with a light red colour.

Already existing suspect values which have been identified as suspect during pre-validation have both flags, i.e. full red and empty red.

When saving the applied changes, the last validation date isn’t reset and the identified potential suspects aren’t converted into real suspects values.

View/Edit/Restore validation attributes on reading type level

You can see what validation rules are used for a certain reading type in the output of a usage point and override certain parameters that will be valid only on the current usage point. The changed parameters can be restored back to their original values afterwards.

This is applicable for channels and registers.

5

Steps

1.   Open a usage point using the search page.

2.   Click the needed purpose in the group Purposes on the vertical navigation bar to the left.

3.   If needed, filter the outputs using the relative period filter. Don’t forget to click Apply to execute the filter.

4.   Click the needed output in the outputs table.

5.   Click the tab Validation configuration.

All active and inactive, current, past and future validation rules that match the reading type of the output are shown and marked with the following icons:

Parameter

Description

 

Active validation rule.

 

Inactive validation rule.

 

Future or past version of the validation rule.

The attributes of the validation rule are shown under its title. Editable values are marked with the following icons.

Parameter

Description

 

The value has been edited and no longer corresponds to the origi­nal value in the validation rule.

 

The value can be edited but is currently still identical to original value in the validation rule.

3

Note

If attributes of a validation rule haven’t been edited on reading type level and only future and past versions of the rule exist, then the values of the future version are shown.

If validation rules with identical names have been configured for both the suspects and informatives, then the action items in the action button have (suspect) or (informative) added to it at the end.

6.   Click the actions button to edit any of the listed validation rules.

The edit window shows only the attributes that can be edited. The validation rule name, validator and reading type are shown for information.

7.   Click Save to save the edited attributes.

The changed attributes are now marked with a white pencil and a restore functionality is now available from the actions button. You can click this button if you want to remove the edited values on the current level and revert the attributes back to those of the actual validation rule.
