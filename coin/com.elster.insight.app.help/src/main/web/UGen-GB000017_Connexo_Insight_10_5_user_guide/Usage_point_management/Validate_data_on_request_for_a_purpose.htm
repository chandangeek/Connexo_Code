Validate data on request for a purpose

Run the validation process manually to re-validate data for a purpose on the metrology configuration that is associated with the current usage point if you suspect that validated data may contain mistakes or if the validation schedules failed and data was not validated automatically. It is even possible to revalidate data that has been validated before.

5

Steps

1.   Open a usage point with configured validation rules using the search page.

2.   Click the desired purpose in the group Purposes in the vertical navigation bar to the left.

3.   Select Validate now from the action button.

4.   In the popup, choose if you want to validate data from the last validation date or from any other date.

The last validation date is added between brackets.

5.   Click Validate.

Before the actual validation starts, it is first verified if there are any validation rules configured for the selected purpose of the usage point starting from the specified date. This check takes all outputs from the same purpose on all metrology configurations starting from the specified date into consideration. Also the usage point states that were defined on the metrology configuration purpose of the associated validation rule set are considered. If states were specified, then the interval data from a period when the usage point was in a particular state must be validated by the specified corresponding validation rules. If no states were specified on the metrology configuration purpose of the associated validation rule set, then all the validation rules must be applied to all usage point data no matter what state the usage point is in at that moment.

 If validation isn’t configured on the selected purpose of the concerned usage point from the specified date, then the user is notified.

A progress bar appears while the data is validated. After successful validation a message is shown. If validation takes longer than 60 seconds, then the user is notified and the validation continues in the background.

 

1

Important

Validation is done immediately. If the validation process takes more than 60 seconds, then validation will continue in the background and you can proceed with other tasks. Note that no logging is created. So, if validation fails, it won’t be possible to check why it failed. If you run validation from the validation task, then logging is created. See “Run validation on data for all usage points associated with a particular metrology configuration and purpose”.
