About Connexo Insight

What is Connexo Insight?

Connexo Insight is a web based application with a complex back end providing long-term data storage and management of the data delivered by data collection applications such as Connexo MultiSense.

It imports, validates, cleans and processes these data in line with customer specified rules to make them available for integrated analytics and exchange with many different utility systems such as billing applications, workforce management systems, prepayment systems, asset management systems and many others. All meter data management features are used and configured in Connexo Insight.

Connexo Insight uses international industry standards to guarantee interoperability and a fast time-to-market.

Aggregation

Data exchange

Easily leverage the data in Connexo Insight in other applications such as demand response, utility analytics, prepayment and many others.

Performance

Read, store, process and exchange all data in four to six hour windows.

Data life-cycle management

Store your data with rigorous versioning and maintain data integrity.

Who is using Connexo Insight?

Connexo Insight is developed with the data operator, data expert and system admin in mind.

The data operator works within the boundaries set by the data expert. He is responsible for the daily operations, the treatment and transformation of data enabling utilities to provide extra services such as billing, data aggregations, data transformations, data export, etc. The data operator needs to have good knowledge of the existing contracts and the needed configurations. He monitors the consumption of a building or groups of buildings (site),  checks validation status of the data and resolves eventual issues.

The data expert’s responsibility is to have valid data ready for other business processes such as billing, forecasting, etc. He will optimise the data quality process to make data quality meet the existing KPI      s. He will identify, analyse and resolve bottlenecks and eventual non-compliance. An important part of this configuration are the validation and estimation rules. The data expert is also responsible for compliance with the utility’s data retention rules.

The system administrator manages the whole system and its IT architecture. He is responsible for the integration of the application in the existing IT infrastructure. His objectives are to achieve a stable and secure system decreasing downtime and increasing performance.
