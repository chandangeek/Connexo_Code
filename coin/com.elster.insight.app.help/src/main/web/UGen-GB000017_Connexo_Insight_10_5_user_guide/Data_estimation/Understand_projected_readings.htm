Understand projected readings

When data couldn’t be read from the meter, a substitution is done and this data is flagged as non-permanent. It is a temporary estimation because it is possible that later on new data will be entered - either automatically or manually - overwriting the estimated values. When the data is re-read, imported or edited, this flag will be removed unless the operator decides that the new data must still be considered as temporary. The flag can also be cleared manually. When the flag has been removed the new data will be considered as valid, i.e. not suspect, in Connexo and will be generally available for data export.

1

Important

Only edited and estimated data can be marked as projected. When flagged the value receives the 3.12.0 reading quality and the value is marked with the corresponding icon.

 

Example: Between 0:00 and 04:00 in the morning a communication task starts which asks meters for data. Once received, the data is validated, the suspect data is estimated and the results are exported. If a meter can’t be read or if there are just a few hours for which we can’t read data a ‘Forward estimation task’ is started. This task estimates the missing data and flags it as projected. This flag indicates to the billing system that it isn’t actual data but that it was estimated by Connexo. By estimating this data, we are able to export a full day’s data to the billing system. On the next day, the actual data is read and received. It will replace the projected data with the actual data and all reading qualities are removed.

To flag a value as projected:

5

Steps

1.   Open a usage point.

2.   Open a purpose under Data in the vertical navigation bar on the left.

3.   Open one of the outputs.

4.   Click the action button of an edited or estimated value and select Mark as projected.

5.   Click Save changes.

The value is marked with the projected icon and has the 3.12.0 reading quality.

3

Note

Mark as projected is also available as a bulk action to mark multiple values simultaneously.

Most estimators have a parameter allowing to flag values as projected automatically when they are estimated. See “Out-of-the-box estimators”. The parameter becomes available when editing a channel reading as described in “Estimate channel readings”.
