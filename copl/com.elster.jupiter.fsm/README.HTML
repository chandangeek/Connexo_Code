com.elster.jupiter.fsm

This bundle provides support for finite state machines. The main interface is FiniteStateMachine, a named wrapper for a set of States and StateTransitions. Every State can be configured to trigger one or more external processes on entry and on exit. As events are produced, the finite state machine will respond to the event and change the state accordingly. When state actually changes, a new event is produced that can be picked up by other components that could e.g. change the actual persisted state of a device. Events are globally defined and can therefore be reused across different state machines. The triggering of external processes that are defined in jBPM is done by publishing an event.
