Device

A device represents a physical element in the grid, this can be e.g. a meter, concentrator, gateway, in home display, etc.

Device action

A device action is part of a communication task and needs to be executed on a device when communicating with this device during a single connection.

Device command

Is used for infrequent adjustments of device parameters such as resetting a device. Device commands are only available when they are specified in a device protocol and this protocol is applied to the device type.

Device configuration

Device configurations are used to customise or limit some of the functionality of the device type, but also configure additional parameters to interact with the physical device. The device configuration will determine the actual register specifications together with the number of channels. It also contains information on security levels, connection methods, etc.

Device type

A device type defines the capabilities of a physical device. Every device has exactly one device type that defines which data (such as registers, load profiles and logbooks) can be read from it and what communication protocol is used to read it. A device type doesn't necessarily map to a single physical device model. A device type may represent several models from various manufacturers as long as they have similar behaviour and characteristics.

Dynamic device group

is formed based on different search criteria. Single devices cannot be selected, all devices of the search result are part of the group. This group is dynamic because each time a new device that complies to the search criteria is added to the system, it will also be added to the device group.
