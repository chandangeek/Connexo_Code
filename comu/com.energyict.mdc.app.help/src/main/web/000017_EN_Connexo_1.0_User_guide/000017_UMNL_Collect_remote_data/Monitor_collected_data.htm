Understand reading quality

Reading quality indicates that there is something remarkable with the value. This could for example be that a power outage occurred or data was edited or tampered with. These reading qualities are based on CIM standards and consists out of three different fields.

  • The first field is the system identifier and indicates which system declared the issue, the following options are available:

    Code

    Description

    Name in MultiSense

    0

    Not applicable

    Not available

    1

    End device

    Device quality

    2

    Metering system (data collection)

    MDC quality

    3

    Data management system

    MDM quality

    4

    Other system (not listed)

    Third party quality

    4

    Externally specified (see accompany data)

    Third party quality

  • The second field is the categorisation and indicates the category of the issue.

  • The third field is the index and each category provides the context with which to understand a particular index.

You can see the description of the reading quality code when you hover over the information icon .

For example a suspect value can have the reading quality 2.5.258:

  • 2: it concerns an end device.

  • 5: failed reasonability testing, data at first sight appears to be invalid.

  • 258: data should be considered suspect.
