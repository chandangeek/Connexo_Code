How connections and communication work in Connexo

How connections are triggered and communication is established, is mainly determined by the connection strategy of the connection method, the connection schedule and the connection window of the device.

The connection strategy defines the time when a connection can be triggered. When this is minimize connections the connection is only triggered when the connection is scheduled. When the strategy is as soon as possible, a connection is triggered whenever a connection is needed. Connections are only possible when the device is active, the connection window defines this period. You can trigger connections on request.

It is also possible to run communication tasks on request. There are two options: Run and Run now. Run now is only possible when the connection strategy is as soon as possible. When you select to run a communication now, this task will be put at the end of the queue of all communication tasks scheduled on this connection. This connection is triggered right away if the device is active.

When you select Run you wish to execute the task before it is scheduled. The communication task is then put at the end of the queue containing all communication tasks scheduled on this connection. The connection is only triggered when it is scheduled, performing the tasks in the order they were added to the queue.
