Understand device commands

Device commands are used for infrequent adjustments of device parameters such as resetting a device. These device commands are only available when they are specified in a device protocol and this protocol is applied to the device type.

Device commands have a release date when they become available and can be sent to the device. When they are released a command receives the label “pending”. Only users with the correct privileges are be able to send device commands. These privileges are specified in the Admin application when defining users and roles.

Commands are split into categories. For each category and for each individual command the availability and execution level can be configured. Device command categories are specified in communications tasks and further configured on the device configuration. The command is interpreted by the device when it is communicated via the communication task, leading to the adjustment of the parameters. If a command is not picked up by a communication task, it will simply remain in the system without impact.
