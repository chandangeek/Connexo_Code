Configure communication tasks

Add/edit communication tasks

1

Important

To be able to add communications task with actions related to registers, load profiles or logbooks, data sources first needs to be configured. This can be done via “Configure register types”, “Configure load profile types” and “Configure logbook types”.

5

Steps

1.   Go to Administration > Device communication > Communication tasks.

2.   Click Add communication task. Different categories of actions can be added to the communication task.

3.   Click Add action.

Parameter

Description

Register

See “Add a register action to a communication task”

See “Add a load profile action to a communication task”

Logbook

See “Add a logbook action to a communication task”

Clock

See “Add a clock action to a communication task”

See “Add a topology action to a communication task”

Status information

See “Add a status information action to a communication task”

4.   Click Add another to add an additional action.

5.   Click Add to save the communication task

3

Note

Device commands are used for infrequent adjustments of device parameters and these can be added to a communication task to be executed via the connection of the communication task. See “Add device command categories to a communication task” for more information.

Edit communication tasks

5

Steps

1.   Go to Administration > Device communication > Communication tasks.

2.   Click the actions icon  and select Edit.

3.   Change the communication task parameters.

4.   Click Save.

5

Steps

1.   Go to Administration > Device communication > Communication tasks.

2.   Click the actions icon  and select Remove.

3.   Click Remove.

Add a register action to a communication task

Adding this action to a communication task will allow you to read out a selection of registers on your devices.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add action.

2.   Select Registers in the field Category.

3.   Select the Action Read.

4.   Select one or more Register group(s).

5.   Click Add.

3

Note

If a register was not added to a device configuration or device type, then the communication server will not attempt to read this register on the device. This means that you do not need to create different communication tasks for different device types.

Add a load profile action to a communication task

Adding this action to a communication task will allow you to read out a selection of load profiles and channels on your devices.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add Action.

2.   Select Registers in the field Category.

3.   Select the Action Read.

4.   Select one or more Load profile type(s).

5.   Select one of the options for the parameters Mark interval as bad time, Meter events from status flag and Fail if profile configuration doesn’t match.

For more information on these parameters hover over the information icon .

6.   Click Add.

Add a logbook action to a communication task

Adding this action to a communication task will allow you to read out a selection of logbooks on your devices.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add Action.

2.   Select Logbooks in the field Category.

3.   Select the Action Read.

4.   Select one or more Logbook type(s).

5.   Click Add.

Add a clock action to a communication task

Adding this action to a communication task will allow you to set, force or synchronise the clock of your devices.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add Action.

2.   Select Clock in the field Category.

3.   Select an Action.

When you select the action Set, define the minimum and maximum clock difference.

When you select the action Synchronise, define the minimum and maximum clock difference as well as the maximum clock shift.

4.   Click Add.

Add a topology action to a communication task

Adding this action to a communication task will allow to update or verify the topology of your devices.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add Action.

2.   Select Topology in the field Category.

3.   Select the Action Update or Verify.

The Update action will collect the topology information of the device and update when necessary. When an unknown device is detected this will result in an Unknown device event and a configuration warning.

The Verify action will collect the topology information of the device and log it if information is still up to date or not. When this is not up to date this will result in a configuration warning

4.   Click Add.

Add a status information action to a communication task

A status information task will retrieve firmware information from the device.

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task > Add Action.

2.   Select Status information in the field Category.

3.   Select the Action Read.

4.   Click Add.
