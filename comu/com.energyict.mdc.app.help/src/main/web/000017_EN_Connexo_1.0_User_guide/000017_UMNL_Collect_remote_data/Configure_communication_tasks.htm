Configure communication tasks

Add/edit communication tasks

1

Important

To be able to add communications task with actions related to registers, load profiles or logbooks, master data first needs to be configured. This can be done via “Configure register types”, “Configure load profile types” and “Configure logbook types”.

5

Steps

1.   Go to Administration > Device communication.

2.   Click Communication tasks.

3.   Click Add communication task. Different categories of actions can be added to the communication task.

4.   Click Add action.

Parameter

Description

Register

See “Add a register command to a communication task”

Load profile

See “Add a load profile command to a communication task”

Logbook

See “Add a logbook command to a communication task”

Clock

See “Add a clock command to a communication task”

Topology

See “Add a topology command to a communication task”

5.   Click Add another to add an additional action.

6.   Click Add to save the communication task

3

Note

Device commands can be added to a communication task to be executed via the connection of the communication task. See “Add device command categories to a communication task” for more information.

Add a register command to a communication task

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task.

2.   Select Registers in the field Category.

3.   Select the Action Read.

4.   Select one or more Register group(s).

5.   Click Add command.

3

Note

If a register was not added to a device configuration or device type, then the communication server will not attempt to read this register on the device. This means that you do not need to create different communication tasks for different device types.

Add a load profile command to a communication task

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task.

2.   Select Registers in the field Category.

3.   Select the Action Read.

4.   Select one or more Load profile type(s).

5.   Select one of the options for the parameters Mark interval as bad time, Meter events from status flag and Fail if profile configuration doesn’t match.

For more information on these parameters hover over the information icon .

Add a logbook command to a communication task

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task.

2.   Select Logbooks in the field Category.

3.   Select the Action Read.

4.   Select one or more Logbook type(s).

5.   Click Add command.

Add a clock command to a communication task

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task.

2.   Select Clock in the field Category.

3.   Select an Action.

When you select the action Set, define the minimum and maximum clock difference.

When you select the action Synchronise, define the minimum and maximum clock difference as well as the maximum clock shift.

4.   Click Add command.

Add a topology command to a communication task

5

Steps

1.   Go to Administration > Device communication > Communication tasks > Add Communication task.

2.   Select Topology in the field Category.

3.   Select the Action Update or Verify.

4.   Click Add command.
