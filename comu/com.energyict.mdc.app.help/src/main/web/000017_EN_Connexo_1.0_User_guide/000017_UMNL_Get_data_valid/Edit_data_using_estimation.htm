Edit data using estimation

Manually trigger estimation of one or more values

In some cases data cannot be estimated using the estimation rules defined in the estimation rule set. Using issue creation rules MultiSense can create an issue to indicate which values were not estimated. See “Configure issue creation rules” for more information. One option to makes this values no longer suspect is to manually estimate them.

5

Steps

1.   Select a device using Search, see “Advanced search” and go to Validation results or navigate directly to data that could not be estimated via Workspace > Data validation > Issues.

2.   Click the actions icon  of one suspect value and select Estimate value or select multiple values using the CTRL or SHIFT key and click Bulk action and select Estimate value.

3.   Select an Estimator.

4.   Define the estimation rule parameters.

5.   Click Estimate.

6.   If the estimated values are correct, click Save changes.

The estimated values will have the edit icon and will no longer be suspect.
