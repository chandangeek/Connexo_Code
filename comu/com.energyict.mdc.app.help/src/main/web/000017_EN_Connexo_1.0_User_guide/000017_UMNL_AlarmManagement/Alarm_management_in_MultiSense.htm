Alarm management in MultiSense

Explore the alarm page

Go to Workspace > Alarms > Alarms.

On top of the alarms page all available filters are shown and can be configured here. You can for example filter all alarms raised by specific reason or assigned to a specific user or workgroup. Which filters are applied, is shown in the menu filter menu itself.

Below the filter pane there is a possibility to sort alarms based on the parameters Priority and Due date. If sorting is active this is shown by a button below the filter menu. The arrow next to the sorting parameter shows the order of the sorting. Click the button to change the order. Remove the sorting parameter by clicking the remove icon . By default alarms are sorted by decreasing priority.

In the middle of the page the overview table is displayed. This table shows all alarms corresponding to the selected filters. In this table there is an option to handle alarms by using the actions icon next to each alarm.

Below the overview table you can find the details pane. Here you can find some more detailed information of the alarm selected in the overview table. Links are provided to devices, usage point and logbooks linked to the alarm. Next to some parameters you can find a filter icon . If you click this icon all alarms will be filtered showing only those containing this parameter.

View alarms

5

Steps

1.   Go to Workspace > Alarms > Alarms > click an alarm or click an alarm in the What’s going on widget on the device landing page or in the My worklist widget on the dashboard.

2.   The alarm details page is divided into three parts: Details, Related events and Contextual information. The details part shows the following parameters:

Parameter

Description

ID

Alarm specific ID. All alarm IDs start with ALM.

Reason

The reason for creating the alarm as defined by the alarm creation rule.

Usage point

Usage point linked to this alarm.

Device

Device on which this alarm was raised.

Location

Full location details of the device.

Logbook

Name of the logbook that contains the event that originally triggered the current alarm.

Status

The current status of the alarm.

Due date

Date when the alarm should be handled.

Priority

Shows the priority of the alarm:

o1 - 20: very low

o21 - 40: low

o41 - 60: medium

o61 - 80: high

o81 - 100: very high

Workgroup

Name of the workgroup assigned to the alarm.

User

Name of the user assigned to the alarm.

Creation date

This date is typically identical to the event occurrence date that originally triggered the alarm.

The Related events part contains events that are registered in the device’s logbook and are related to the current alarm or that are added to the alarm in line with specifications defined in the alarm creation rule.

The Contextual information parts shows extra information linked to the alarm: it contains comments as well as processes linked to this alarm. All changes made on this alarm are shown in the Time line.

View alarms overview

5

Steps

1.   Go to Workspace > Alarms > Alarms overview.

This page shows an overall overview of all alarms in MultiSense . Alarms are grouped per status, workgroup, user and reason.

2.   Click one of the links to see all alarms in that group.

3.   Use the filters on top of the page to see get an overview of all alarms corresponding to the selected filters.

Figure: Alarms overview

 

View alarm history

The alarm history provides a quick overview of all alarms to show you how many alarms still have to be handled.

5

Steps

1.   Go to Workspace > Alarms > Alarms overview > select the History tab.

The history page shows two different graphs:

  • Alarms per reason: shows the amount of alarms raised per day and divided into different colour categories based on the alarm reason.

  • Alarms open vs closed: shows the amount of alarms raised per day and how many alarms are closed or still open.

2.   Use the filters on top of the page to only see specific alarms.
