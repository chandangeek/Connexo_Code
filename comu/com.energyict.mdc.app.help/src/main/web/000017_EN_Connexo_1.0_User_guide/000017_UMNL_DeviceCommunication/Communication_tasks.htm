Understand communication tasks

Communications tasks are a group of actions the user defines and communicates with the device to make the device carry out a specific action, e.g. resetting the clock or reading data. These tasks are communicated to the device using a connection. You can schedule communication tasks using shared communication schedules.

Figure: Device communication model

 

Communication tasks in Connexo

In MultiSense  you first need to configure communication tasks configurations (Administration > Device communication > Communication tasks), you can then reuse these communication tasks for different device configurations. In the first step you will add the actions to the communication task, These actions are grouped in different categories, two examples are load profiles and clocks.

You can also add command categories, these will then be executed together with the communication task. See “Device commands” for more information.

Once you created these communication tasks, you will need to configure them further on the device configuration. This will allow you to specify the parameters that are specific for that device configuration. Here you can which connection method to use and link the task to a security set to define who can view or execute the communication task.

You can schedule communication using shared communication schedules, see “Schedule communication” for more information.
