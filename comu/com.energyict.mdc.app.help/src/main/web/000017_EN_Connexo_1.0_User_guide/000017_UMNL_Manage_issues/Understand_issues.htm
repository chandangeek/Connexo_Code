Understand issues

What are issues

It is often important that users are notified when something goes wrong during data collection and validation. Instead of having to manually verify all events that originate from different tasks, Connexo MultiSense allows you to configure rules to create issues for specific events. For more information see “Configure issue creation rules”.

Once issues have been created, they need to be handled. The way to solve an issue depends on the type of task from which it has been generated. The violation of a data validation rule requires a different investigation and solution than a failed communication task. Issues can be assigned to a particular user or user group to solve this issue.

Explore the issues page

Go to Workspace > Data collection > Issues.

On the left side of the Issues page all available filters are shown and can be configured here. You can for example filter all issues raised by a failed connection and assigned to you. The applied filters are shown in this menu.

On the top of the page there is a possibility to group issues based on the parameter Reason. If grouping is applied then the group will be shown on top of the page above the overview table. In this table it is indicated how many issues this group contains.

In the middle of the page the overview table is displayed. This table shows all issues corresponding to the selected filters. The due date, status and assignee of all issues are shown. In this table there is an option to handle issues by using the actions icon next to each issue.

On the bottom of the page a details pane is shown that contains all information of the issue that is selected in the overview table. By clicking the name of an issue, the corresponding issue detail page will open. On this page issue specific details can be found together with comments and a link to the device that raised the event.
