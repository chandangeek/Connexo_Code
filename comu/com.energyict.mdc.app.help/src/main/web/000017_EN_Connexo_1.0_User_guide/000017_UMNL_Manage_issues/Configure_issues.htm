Understand issue creation rules

Connexo MultiSense offers the possibility the configure issue creation rules. This option limits the time you spend managing issues because you create rules to define which events will automatically trigger an issue. You can also define when the issue needs to be solved as well as what needs to be done with the issue such as assigning it to a specific user.

Different parameters are required to complete the configuration of a creation rule. First you will have to indicate the type of issue, this can be for example a data collection issue. Next you need to select a rule template to define the parameters of the rule.

A rule template is a collection of parameters that are specific for a certain rule. The exact values of the parameters are defined in the rule itself. An examples of such a template is the template to create an issue when a slope is detected. All parameters will be shown, but then you will have to complete the reading type, the trend period and the threshold.

Configure issue creation rules

5

Steps

1.   Go to Administration > Issue management.

2.   Click Issue creation rules.

3.   Click Add rule. Complete the following fields:

Parameter

Description

Name

A unique name

Issue type

Data collection

Rule template

Select the template containing all parameters to configure the rule

Auto resolution

Close issues automatically when the connection or communication problem is solved.

Note that this option is only available for specific rule templates.

Issue reason

What is the cause of the issue

Due date

When does the issue need to be solved.

Comment

Add additional information to the issue.

Actions

Define what needs to be done when the issue is created. See “Add actions to issue creation rules” how to configure these actions.

3

Note

Depending on the selected rule template, additional fields will have to be completed such as thresholds and reading types.

Add actions to issue creation rules

5

Steps

1.   Go to Administration > Issue management > Issue creation rules.

2.   To add actions to a new creation rule: click Create rule.

To add actions to an existing creation rule: click the actions icon  > Edit.

3.   Click Add action.

4.   Define when to perform the action: when the issue was created or when the issue has passed its due date.

5.   Select action: send a notification or send someone to inspect the issue.

6.   Click Add.
