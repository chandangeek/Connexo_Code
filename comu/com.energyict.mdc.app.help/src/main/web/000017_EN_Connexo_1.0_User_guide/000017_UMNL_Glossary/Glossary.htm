Glossary

A

Authentication

Ensures that both ends of the connection are secure.

C

Channel

A channel contains the energy consumption data collected by a device.

Communication port

A communication port is a physical modem (GSM, PSTN, etc.) or a logical connection (TCP, UDP, LAN, etc.). A communication port sets up a connection between the communication server and the device.

Communication port pool

A communications port pool is a set of communication ports. The communication port pools can be grouped in ‘Inbound port pools’ and ‘Outbound port pools’. Typically communication port pools are spread over different communication servers.

Communication protocol

A communication protocol defines the syntax of communication between different systems (e.g. between a communication server and a device) It consists of a system of message formats and rules for exchanging those messages. A single protocol can   be used for multiple physical device models from various manufacturers.

Communication schedule

A communication schedule defines the time and recurrence for reading a particular set of meter data from a device or a group of devices.

Communication server

Communication servers are the part of the system responsible for communicating with various devices in the field. A communication server is capable of communicating with several devices simultaneously using ports and pools of ports. It reads data from, writes data to or performs actions on these devices.

Communication task

Is a group of commands the user defines and communicates with the device to make the device carry out a specific action, e.g. read registers, reset the clock, update firmware, etc.

Connection method

Allows you to define how to communicate with the device. Some devices can support multiple types of connection methods (GRPS, optical, SMS, etc.).

D

Device

A device represents a physical element in the grid, this can be e.g. a meter, concentrator, gateway, in home display, etc.

Device action

A device action is part of a communication task and needs to be executed on a device when communicating with this device during a single connection.

Device command

Is used for infrequent adjustments of device parameters such as resetting a device. Device commands are only available when they are specified in a device protocol and this protocol is applied to the device type.

Device configuration

Device configurations are used to customise or limit some of the functionality of the device type, but also configure additional parameters to interact with the physical device. The device configuration will determine the actual register specifications together with the number of channels. It also contains information on security levels, connection methods, etc.

Device command

Is used for infrequent adjustments of device parameters such as resetting a device. Device commands are only available when they are specified in a device protocol and this protocol is applied to the device type.

Device type

A device type defines the capabilities of a physical device. Every device has exactly one device type that defines which data (such as registers, load profiles and logbooks) can be read from it and what communication protocol is used to read it. A device type doesn't necessarily map to a single physical device model. A device type may represent several models from various manufacturers as long as they have similar behaviour and characteristics.

Dynamic device group

is formed based on different search criteria. Single devices cannot be selected, all devices of the search result are part of the group. This group is dynamic because each time a new device that complies to the search criteria is added to the system, it will also be added to the device group.

E

Encryption

Ensures that the information within a session is not compromised.

G

Gateway

Device enabling the communication with (sub)meters by routing/translating the communication. A gateway will not store any data and works solely as a pas-through. This has the advantage that the (sub)meter needs a protocol to read the data.

I

Inbound

Data is pushed from the device to for example a communication server.

Issue

An issue represents a problem that occurred during the execution of a manual or automated task. It usually requires human intervention to analyse and fix the problem, although some issues can be fixed simply by automatically retrying them. The details of an issue and how to solve them depends on the type of task from which it originates. Issues can be assigned to a particular user or user group to solve it.

Issue creation rules

Rules to define which events automatically trigger an issue and which actions should be taken.

L

Load profile

A load profile is an object containing one or more channels with the same reading interval.

Load profile configuration

A load profile configuration contains further specification of different load profile parameters that are specific for a certain device configuration.

Load profile type

A load profile type contains all essential load profile parameters so that this load profile type can be reused for different device types and device configurations, but it doesn’t contain values.

Logbook

A logbook categorizes events that have occurred on a device. A single logbook can contain different types of events. A device can have more than one logbook. Each logbook has its own unique identifier, which is needed to retrieve its contents from the device. The content of the logbook is recorded during periodic communication to a device. Typical events that are recorded are power outages, connection status, firmware logging, etc.

Logbook configuration

A logbook configuration contains further specification of different logbook parameters that are specific for a certain device configuration.

Logbook type

A logbook type contains all essential logbook parameters so that this logbook type can be reused for different device types and device configurations, but it doesn’t contain values.

M

Master data

Registers, load profiles and logbooks

O

OBIS code

OBIS stands for Object Identification System. OBIS provides standard identifiers for all data within the metering equipment, both measurement values and abstract values. OBIS uses six value groups in a hierarchical structure.

Outbound

Data from, for example a communication server, is pushed to a device.

P

Privileges

Designate what rights a user has to perform certain actions on a specific resource. This can be for example the viewing, adding, editing or removing of a communication server. A resource is a component of an application, e.g. role is a resource as well as issue rules, licenses and communication servers.

Protocol dialect

Some tasks require specific properties such as the ability to download a large amount of data as it is often the case for firmware upgrades. Such specific properties can be acquired by configuring protocol dialects.

R

Reading type

A reading type provides a detailed description of a reading value. It is described using 18 key attributes separated by a dot.

Register

A register is a placeholder for data about a device or data measured by the device. It contains snapshot values that are captured at specific moments in time.

Register configuration

A register configuration contains further specification of different register parameters that are specific for a certain device configuration.

Register group

Register groups are used to collect different register readings that belong together. A group will contain all necessary information needed for a certain process.

Register type

A register type contains all essential register parameters so that this register type can be reused for different device types and device configurations, but it doesn’t contain values.

Release date

Is the date when a command comes available to be executed when there is a connection.

Rule template (issue)

A rule template is a collection of parameters that are specific for a certain rule. The exact values of the parameters are defined in the rule itself. An examples of such a template is the template to create an issue when a slope is detected. All parameters will be shown, but the user will have to compete the reading type, the trend period and the threshold.

S

Static device group

is a group of devices that was selected out of a search result based on different search criteria. The composition of this group does not change over time, to add new devices to the group, the search has to be repeated and devices manually added.

Shared communication schedule

Is a system wide communication schedule that can be used for multiple devices.

V

Validation rules

Rules to check whether collected data is valid by verifying if the data is in accordance with specific market rules.

Validation rule set

Is a collection of validation rules that should be applied together.

Validator

The validator is a template that sets the rules to check if the collected data is correct.
