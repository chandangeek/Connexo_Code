Configure time of use on devices

Send time of use calendar to a device

A time of use calendar must be specifically assigned to a device. Only those time of use calendars that have been assigned t o the device’s device type will be available. Depending on the settings configured on the device type and the communication protocol, the options for assigning a time of use calendar may vary.

5

Steps

1.   Open a device using Search.

2.   Click Time of use in the menu on the left.

3.   On the Time of use page, click the Actions button in the upper right corner and select Send calendar.

4.   The following fields are shown on the Send ToU calendar page.

Parameter

Description

Time of use calendar

Select one of the time of use calendars that are available on the device type.

Release date (command)

Defines the date when the time of use calendar can be sent, i.e. when it will be ready to be picked up by the recurring communication task that is running for this purpose. The option Send without release date will make the time of use calendar immediately ready for pick up by the communication task.

Activate calendar

When to activate the calendar on the device. You can choose to send it without an activation date, activate it immediately or give a specific date.

Update

Change the complete calendar or only update special days for example holidays.

1

Important

Depending on the selected time of use options additional attributes will become available for completion, e.g. Type, Contract. See the Admin user guide on time of use calendars and the protocol documentation for more details.

5.   Click Commands in the menu on the left to follow up the command that will send the calendar.

6.   The sent time of use calendar will appear in the Planned section of the time of use page.

7.   When viewing the device’s time of use calendar you will see the newly sent time of use calendar in the section Planned at the bottom of the page. See “View time of use calendar on a device”. Connexo will regularly update its status. If the time of use calendar has actually changed on the device, the planned section will disappear and the time of use calendar information will be changed.

Check time of use calendar

you can manually verify the active (and passive) time of use calendar.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Configuration > Time of use in the menu on the left

3.   Click the action button and select Check time of use calendars.

This will trigger the communication task that reads the status information see “Add a status information action to a communication task” for more information.

If the device and protocol support passive calendars, you can see the passive calendar in the time of use details.

Manage passive time of use calendars

A device can have active and passive calendars, this concept is the same as for firmware version (“How is firmware managed in MultiSense”). When you upload a time of use calendar and do not activate it immediately this calendar will be considered as passive. When the activation date is reached this calendar will become active.

The action described here is only possible if a passive calendar is present on the device and if the protocol support passive calendars.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Configuration > Time of use in the menu on the left

3.   Click the action button and select Activate passive calendar. The calendar will become active before its planned activation date.

4.   Click the action button and select Clear passive calendar to remove the passive calendar on this device.
