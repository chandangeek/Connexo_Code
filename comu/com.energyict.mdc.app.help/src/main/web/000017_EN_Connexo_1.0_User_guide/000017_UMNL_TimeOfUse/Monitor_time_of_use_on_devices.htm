Monitor time of use on devices

View time of use calendar on a device

On a device you can see the time of use calendars that are already on the device or that will be sent to the device. For each calendar you can view its contents to verify that the device is using the right time of use calendar.

5

Steps

1.   Open a device using Search.

2.   Click Time of use in the vertical navigation bar.

3.   The following information is available.

Parameter

Description

Time of use calendar

Name of the used time of use calendar. If the calendar isn’t available in the system, the value of this field is Ghost. No further data will be shown for the time of use calendar if it isn’t in Connexo.

Last verified

Date, time when the time of use calendars on the device, both active and passive, were last checked.

Start of calculations

Start year of the used time of use calendar.

Periods

Periods used by the time of use calendar to cover the year. Refer to the Connexo MultiSense user guide under Import time of use calendar for a more detailed description of the time of use calendar attributes.

Day types

Day types used by the time of use calendar to cover days in the week template. Refer to the Connexo MultiSense user guide under Import time of use calendar for a more detailed description of the time of use calendar attributes.

Tariffs

Tariffs applied by the time of use calendar. Refer to the Connexo MultiSense user guide under Import time of use calendar for a more detailed description of the time of use calendar attributes.

Passive calendar(s)

Passive calendars are in a waiting queue to become active. This field is not shown if the device protocol doesn’t support passive calendars. Ghost is mentioned in case the time of use calendar doesn’t exist in Connexo.

Planned

This section is only present if a new time of use calendar has been sent. It disappears when the new time of use calendar has become active. It contains the following fields.

  • Time of use calendar: Name of the new calendar.

  • Release date (command): Date when the time of use calendar can be sent.

  • Activation date: Date when the new calendar can be activated on the device.

  • Status: Status of the command that has been sent for the time of use calendar.
