Default estimation rules

Estimate using average with samples

1

Important

You need delta values for this estimation rule.

With this rule suspect every channel delta value is estimated by calculating the average of reading values from this channel or other reading types at the same time and day in the past. For this rule you need to configure the following parameters:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be manually edited or estimated.

Allow negative values

Indicates that negative values can be used to estimate data.

Use advance readings

Select this when you want to use other data than the channel’s own delta values to estimate suspect data.

o   Bulk reading: use the channel’s bulk reading for estimation.

  • Reading type: select another reading type for estimation. This reading type needs to have readings at exactly the same time points as the suspect data.

Minimum samples

The minimum amount of sample needed for estimation.

Maximum samples

The maximum amount of samples needed for estimation.

Relative period

Select the period in which Connexo will search for sample.

o   All: search in all past readings.

  • Period: select a relative period, as configured in Admin. This period will define how many samples you will find and be able to use for estimation.

Estimation with equal distribution

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule.

To estimate delta values using the equal distribution rule MultiSense will calculate the difference between the bulk or advance reading after the estimation interval and the bulk or advance reading before the estimation and divide it by the amount of samples in the interval + 1. MultiSense will then use this value as delta value for every suspect value in the interval. For this rule you need to configure the following parameters:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be manually edited or estimated.

Use advance readings

Select which you want to use to estimate the suspect data.

o   Bulk reading: use the channel’s bulk reading for estimation.

o   Reading type: select another reading type for estimation. This reading type needs to have readings at exactly the same time points as the suspect data.

Estimate with linear interpolation

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule. This rule only works with bulk readings.

With this rule MultiSense will estimate the suspect bulk readings. The delta values are then calculated based on these bulk readings. To estimate the bulk readings MultiSensecalculates the difference between the bulk reading after the estimation interval and the bulk reading before the interval and divide it by the amount of samples in the interval plus one.

In the second step MultiSense adds this difference to the bulk reading before the estimation interval to calculate the first suspect reading. The second estimated suspect reading will be the previous estimated reading plus the calculated difference from the first step. MultiSense repeats these steps until all suspect readings are estimated.

You need to configure the following parameters for this rule:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be manually edited or estimated.

Estimate with power gap fill

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule.

Using the power gap fill rule MultiSense changes every suspect reading in estimation interval except the last reading by zero. The last equals the difference between the bulk reading after the interval and the bulk reading preceding the interval. The following parameters need to be configured:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be manually edited or estimated.

Estimate with value fill

The value fill rule is the most simple rule and estimates suspect readings by replacing all the readings by a user defined value. For this rule you need to complete the following parameters:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be manually edited or estimated.

Fill value

The value MultiSense will replace the suspect reading by.
