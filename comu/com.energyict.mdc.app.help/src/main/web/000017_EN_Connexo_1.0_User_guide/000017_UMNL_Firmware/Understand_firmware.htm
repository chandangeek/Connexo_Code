Understand firmware

What is firmware

Firmware is the software on the device. Different types of firmware exist. Two examples are the meter firmware and the communication firmware. The meter firmware contains all metrics. The communication firmware contains all features needed to allow communication between the device and the meter data collection system.

Firmware needs to be upgraded when new versions are available with new features or bug fixes. Sometimes firmware can also be downgraded when the current version contains errors.

How is firmware managed in MultiSense

In MultiSense firmware management consists out of different features configured on different levels. The first feature that helps determine the firmware management options is the communication protocol used by the device type. The communication protocol defines how communication server and device communicate and therefore if it is possible to upload firmware and which options are available.

The communication protocol is then linked to a device type. A device type determines functionalities that are the same for all devices using this device type. On the device type you can configure firmware management options. First you select whether it is possible to upload firmware on a device and secondly you select which options are available to activate the firmware version. These options are: activate later, activate immediately or use an activation date. For more information see “Understand firmware management options”.

In the next step you need to configure the firmware specific communication tasks. The firmware communication task configuration is available by default and contains the firmware upload action. This communication task still needs to be configured further on the device configuration. It is important to configure the connection as well as the protocol dialect and security settings because firmware tasks have a large file size and need to be very secure.

Besides the communication task to perform the firmware upload, there is also an action available to check the version of the firmware. This “Status information” action can be added to a communication task and needs to be configured further on the device configuration.

On the device itself you can activate the command to upload the firmware. The options that are available are determined by the communication protocol and the firmware management options on the device type as described above.

The firmware engine handles all these actions and sends the commands to the device.
