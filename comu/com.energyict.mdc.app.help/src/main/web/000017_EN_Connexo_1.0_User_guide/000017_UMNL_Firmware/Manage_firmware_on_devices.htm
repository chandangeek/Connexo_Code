How firmware is managed on devices

Firmware versions are added to MultiSense on the device type where also the upload options are defined, but the actual upload of the firmware to the physical device is done via the Connexo device. The firmware is uploaded and activated using a firmware specific communication task, see “How is firmware managed in MultiSense” for more information.

When you want to upload firmware on a device you have multiple options. Which options are available are defined by the communication protocol used and the firmware management options on the device type. If you choose to activate the firmware immediately the firmware will be sent to the device as soon as a connection is triggered according to the connection strategy. When the device receives the firmware, it will be installed immediately. To upload and activate firmware with a date the same methodology is used but the upload and activation will be executed on a specific date. If you choose to activate the firmware later, you will need two separate communication tasks. One to upload the firmware and one to activate and install the firmware.

On the device you can check which firmware version is installed. To be able to perform this action a communication tasks needs to be configured that contains the “Status information” action. To retrieve this information the communication task depends on the connection strategy of its connection method.

In MultiSense you can upload firmware on multiple devices using a single action. These are called firmware campaigns. These firmware campaigns are created for a specific device type and device group. Only the devices of the device group that have the specified device type will get a new firmware version.
