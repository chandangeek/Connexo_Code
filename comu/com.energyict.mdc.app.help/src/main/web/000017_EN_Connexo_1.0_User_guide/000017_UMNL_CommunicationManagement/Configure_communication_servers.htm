Configure communication servers

Add an online communication server

5

Steps

1.   Go to Administration > Device communication > Communication servers.

2.   Click Add online communication server. Complete the following fields:

Parameter

Description

Name

Unique name for the server

Server log level

General log level for the events of the communication server that will be stored in the database (error, warning, information, debug or trace). It is recommended to use the log level “warning”.

Communication log level

Level of logging for the communication sessions (error, warning, information, debug or trace). It is recommended to use the log level “warning”

Changes inter poll delay

Refresh frequency of the communication server properties for example when additional ports are added. The modification will be picked up after the time specified by this parameter.

Scheduling inter poll delay

Refresh pending tasks and pick up new tasks based on the defined interval.

Store task queue size

Number of tasks that can be handled by the communication server, before the data is sent to the database.

Store task thread count

The number of workers that get data from the queue to store in the database.

Store task queue priority

Java VM priority against other java applications running on the OS. The higher the value, the higher the priority.

The following fields are required for using the communication server monitor. See “Monitor a communication server”. They contain the information needed to build the URL to go the communication server monitor which consists of server name, port and path to the application. Example: http://server015:8080/apps/comservermonitor/index.html 

Server name

Name of the machine on which the communication server is running.

Event registration port

Port where the communication server monitor needs to listen to the published events to render the continuous logging on the Logging tab in the communication server monitor.

3.   Click Add.

4.   Click the actions icon  and select Activate.

3

Note

The higher the selected log level e.g. debug, the more storage that will be occupied in the database by this data. Be aware of this effect when you raise the log level of the communication server!

Edit communication servers

5

Steps

1.   Go to Administration > Device communication > Communication servers.

2.   Click the actions icon  and select Edit.

3.   Change the communication server parameters.

4.   Click Save.

Remove communication servers

5

Steps

1.   Go to Administration > Device communication > Communication servers.

2.   Click the actions icon  and select Remove.

3.   Click Remove.
