Monitor communication servers

Monitor a communication server

You can monitor status, performance and multiple loggings for each communication server using the communication server monitor. The displayed information is real-time and automatically refreshed. It is used to detect and investigate problems and to find a corresponding solution. The communication server cannot be used to change communication  or communication server settings in Connexo.

Examples:

  • At installation or migration, use the communication server monitor to check general polling and running information. Check if all needed ports are able to communicate.

  • For performance tuning, check the load, threads in communication ports and communication port pools.

  • To troubleshoot a device that has problems communicating, use the trace log level to check the real time of the device after scheduling an ad hoc communication task on that device.

  • For testing protocols you can put the log level on Trace to see the hexadecimal and binary notation.

 

1

Important

To use the communication server monitor on a device different from that on which the communication server is running, you will need to configure this. By default you are always able to monitor the communication server on the device it is running. See “Add an online communication server” for more information.

5

Steps

1.   Go to Administration > Device communication > Communication servers.

2.   Select the desired communication server.

3.   Click the actions icon  and select Monitor.

The communication server monitor opens in a new tab and shows status, performance and logging information for the current communication server.

4.   The Status tab is open by default.

a. Monitor polling information of the communication server. It shows you when to expect the communication server to do its task or apply the requested changes. The following information is shown.

Parameter

Description

Changes inter poll delay

Frequency to apply changes made in the communication server configuration, e.g. name change, changing number of threads, etc. The date and time between brackets indicate the next run of the changes inter poll delay, i.e. when the changes will be applied.

Scheduling inter poll delay

Frequency to look for new communication tasks that must be executed.

  

3

Note

When the values of Changes inter poll delay and/or Scheduling inter poll delay of the communication server are edited and saved in Connexo, they are updated in the communication server monitor after the Changes inter poll delay has expired.

b. Monitor running information of the communication server. It shows if the communication server is still working efficiently. The following information is shown.

Parameter

Description

Total amount of events

All published events since the communication server has been started.

Memory usage

Shows memory in use compared to the maximum heap space.

c. Monitor communication ports on a communication server to see what communication ports don’t communicate.    

  • Two lists of communication ports are shown: Ports in use and Inactive ports.

  • An icon in front of each port indicates whether it concerns an inbound or an outbound communication port.

  • You can click a communication port to see its a logging information in a new tab.

3

Note

A communication port is inactive when there is no communication on that communication port. It is possible that a communication port that is marked active in Connexo appears in the inactive list in the communication server monitor. Ports that are marked as deactivated in Connexo always appear in the inactive list.

d. Monitor communication port pools on a communication server to see which port pools are communicating and which not.

  • Two lists of communication port pools are shown: Pools in use and Pools not in use.

  • An icon in front of each port indicates whether it concerns an inbound or an outbound communication port.

3

Note

A communication port pool is inactive if all communication ports in that pool are inactive. A communication port is inactive when there is no communication on that communication port. It is possible that a communication port that is marked active in Connexo appears in the inactive list in the communication server monitor. Ports that are marked as deactivated in Connexo always appear in the inactive list.

e. Refresh communication server status information to follow up on changes over time and intervene if necessary.

  • Check the Auto refresh every checkbox in the upper right corner to refresh the status information automatically. If not checked, you have to refresh the page manually.

  • Select the refresh frequency. It can be every 1, 2, 5 or 10 minutes.

5.   Click the tab Performance to monitor performance of the communication server.

a. Data storage: See when the load becomes too high. Data storage performance is shown by three time series on a line chart:

  • The capacity of the collected data processor.

  • The amount of data storage threads.

  • The load in percentage.

The graph is updated every minute. After each update a new dot appears on the graph. You can also refresh manually by clicking Refresh.

When first opening the performance tab, the graph is empty and starts at the 5 minute rounded current time. Then, the one minute refresh interval starts.

After an hour the graph shows the latest hour.

When you go to another tab the graph will keep updating.

When the communication server monitor is closed, the graph is reset.

b. View performance of outbound communication ports. You may need to intervene when too many threads are in use on a communication port because tasks will not be finished efficiently. The information is presented in a pie chart with:

  • The threads that are in use.

  • The threads that are not in use.

c. View performance of communication port pools. You may need to intervene when there are too many threads in a communication port pool with only a few communication ports. The information is shown in two bar charts showing:

  • Number of threads in the communication port pools.

  • Number of communication ports in the communication port pools.

6.   Click the tab Logging to consult live logging information for that communication server.

  • The logging is shown in a new tab.

  • You can select a log level: Trace, Debug, Info, Warning, Error.

  • New log lines appear under the previous one.

  • You can stop the logging if you found an interesting fragment. Note that the logging will be completely halted. Clicking Start logging will start logging again from the moment of clicking. So, there will be no logging information for the time during these two clicks.

  • Logging is not kept in memory after stopping the logging.

  • It is possible to save the logging information into a text file by clicking Save logging.

a. Communication server logging - General: Detect and solve problems with updates on the communication server and database or network connection errors.

b. Communication server logging - Data storage. This information comes from the data storage threads in Connexo MultiSense.

c. Communication logging - Monitor the inbound and outbound communication of a specific device or port.

  • Enter the one or more device MRID’s or communication port names. Separate multiple values in the same field by a comma.

  • If log level is Trace, you see three log fields, i.e. one for the protocol logging and two for the communication tracing. Other log levels show only the protocol logging.

  • You can hide the communication tracing by clicking the down arrow in the blue bar. Th communication tracing shows binary and hexadecimal information.

  • In case of a concentrator, you will see the logging of that concentrator and of its slaves.

In case of a slave device, you will only see the logging of that slave device and not the logging of its concentrator.
