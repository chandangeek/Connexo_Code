Understand communication topology

What is communication topology

When your network consists out of a large amount of devices, cost can run high if you would connect to every single device. To reduce these connection costs you can reduce the amount of connections. This is achieved by connecting different devices with each other and sending the information from multiple devices to one single device which in turn will connect with the data collection system. For example a gas meter and electricity meter are not connected with the internet, but with a gateway. This gateway will then transfer all metering information to the data collection system.

Communication topology in MultiSense

Defining the communication topology in MultiSense starts when you configure a device configuration. At this point you need to indicate whether the device is directly addressable and therefore can connect directly to MultiSense using a connection method. Secondly choose whether the device is a gateway or not and which type. If it is defined in the communication protocol that the device cannot be a gateway then this option is disabled on the device configuration.

On the landing page of the meter or gateway you can then select the device to connect and send data to, the master device. On this landing page you can also view which devices forward their data to this device and therefore use it as a master device. The slave devices are shown in a table or list.
