Introduction to MultiSense

In Connexo MultiSense different data collection features are spread over different pages. A small introduction to the different pages available in MultiSense is given here.

Dashboard

Figure: Dashboard page icon

 

On the dashboard the meter operator can find the latest status of the items that he is responsible for. The overview shows the issues assigned to the operator together with the server status, active connections and communications and connection and communication KPIs.

Workspace

Figure: Workspace page icon

 

The workspace is the central location where you can find all device and asset related items that need to be monitored or require specific actions. For example you can see the status of data collection and validation as well as monitoring and handling different tasks, issues and service calls. Click the links to navigate to the different items.

The workspace is also the location to go to when you need to configure, execute and monitor firmware campaigns.

Note

Devices

Figure: Devices page icon

 

The Devices page is the place in the application where you can manage all devices. On this page devices can be added. Links are provided to different device specific reports created using Facts.

The device page is also the location where to create and manage device groups.

Usage point

Figure: Usage point page icon

 

On the Usage point page you can find all usage point information in MultiSense. This page is only available in MultiSense when you do not have a Connexo Insight license. Otherwise this information is available in Insight.

Administration

Figure: Administration page icon

 

Via the Administration page all device features can be configured. This consists of the complete communication infrastructure (communication servers, communication ports, communication tasks and communication schedules, etc.), the device types and configurations, as well as validation and estimation rules and rules to create issues. Configuration is simplified by the use of templates in the form of shared communication schedules and tasks, data sources and device types. These have been developed in such a way that they can be used for multiple devices.
