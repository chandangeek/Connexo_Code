Command limitation rules

Understand command limitation rules

Command limitation rules allow you to define the maximum allowed of instances of a command sent per day, week or month. This prevents one user from doing harm to the entire system.

For example sending a disconnect command from Connexo to a device will result in a breaker status switch. Consequently no energy can be consumed anymore. However this introduces a huge risk. If you can remotely disconnect one meter it is also possible to do a bulk operation and disconnect the whole meter landscape of a utility. This can result in a major power change on the net which can cause imbalance of the grid and results in consumers not being able to consume energy anymore.

View command limitation rules

5

Steps

1.   Go to Administration > Device communication > Command limitation rules.

2.   This page shows all configured command limitation rules together with their status. If the rule is awaiting approval then the information icon  is shown next to the status of the rule. Hovering over the icon shows additional information.

The details pane on the bottom of the page shows a detailed description of the command limitation rule selected in the table. Hover over the information next to the limits to see how many command instances have been sent.

3.   Click the command limitation rule name to see more detailed information.
