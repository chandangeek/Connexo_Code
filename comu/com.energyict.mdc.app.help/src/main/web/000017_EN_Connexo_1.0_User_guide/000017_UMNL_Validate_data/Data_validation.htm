Data validation

This chapter will handle all subjects that are related to validating your collected data. At the end of this chapter you will be able to automatically and manually check your data to identify suspect values that need your attention.

What would you like to do?

Validate residential data

To be able to use residential customer data for billing, all the data has to be valid to pre­vent incorrect billing. Residential customer data can be validated as follows:

1. Add validation rule sets by creating a new set for residential customers.

2. Add validation rules by adding three validation rules

    a. Threshold violation to monitor if values stay within a defined interval.

    b. Check missing values seeing that missing values are not allowed.

    c. Register increase because cumulative values should increase over time.

3. Add validation rule sets to a device configuration or Add device configurations to validation rule sets to ensure that the data from this device will be validated.

4. Manage validation status on a device by activating the validation rules thereby validating the data when it is stored.

5. After validation View suspect data and Data editing.
