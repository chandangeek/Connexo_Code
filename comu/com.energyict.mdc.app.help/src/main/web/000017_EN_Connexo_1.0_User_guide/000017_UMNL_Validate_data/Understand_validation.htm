Understand validation

Why validate

The data collected from the metering devices should be 100% reliable. If this is not the case, gaps or incorrect values could lead to the generation of erroneous reports and invoices to end-customers. To prevent this it is important to validate the generated data and check whether it is free of discrepancies and errors.

How to validate

In MultiSense data is validated by using validation rules. These validation rules consist out of a validator, validator specific parameters and reading types. The validator is a template that sets the rules to check if the collected data is correct. In some cases extra parameters need to be specified such as thresholds.

The reading type provides the link between the validation rule and the master data that needs to be validated. Consequently it also provides the link with different devices configurations.

Ultimately validation rules that are typically used together are grouped in a validation rule set.
