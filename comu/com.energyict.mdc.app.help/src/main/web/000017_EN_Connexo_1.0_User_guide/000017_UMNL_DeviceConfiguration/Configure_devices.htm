Understand devices

To be able to collect meter data, you need metering devices. These devices are located in the field. Different types of devices are available originating from different vendors. All of these have their own possibilities and characteristics. The ultimate goal is to collect the data coming from the meter and use this data for other purposes such as billing and energy management.

To be able to collect this data, you need a meter data collection system. Connexo MultiSense  connects and communicates with these devices. How this communication works is explained in “Understand communication”. Devices in Connexo MultiSense  are configured using device types and device configurations. Then, the devices are added (manually or via import) and the correct device type and configuration are assigned. Finally, only device specific parameters, such as passwords and connection details, need to be defined.

Device types and configurations in MultiSense

To collect data from a physical device, you have to create a device in Connexo MultiSense . A device uses one device type with a certain configuration. This device type defines the physical capabilities of the device, which data can be read and what communication protocol has to be used. When configuring a new device type you need to define whether it is a standard or a data logger slave device type. This selection will define the parameters to define in further steps.

A device type maps to a single physical device model, but devices of this model can have different capabilities. This is achieved through device configurations.

A device type can have multiple device configurations. Device configurations are used to customise and further configure the functionality of the device type. The device configuration will determine the actual register specifications together with the number of active channels. It also defines which security settings (authentication and encryption) are used and how a connection is set up (GPRS, GSM, etc.).

By using device configurations a more accurate representation of the physical device and its enabled functionality can be created. Configurations can be reused and facilitate the process of creating a large amount of similar devices.

Understand device multipliers

The need for a device multiplier relates to the usage of CT/VT (current transformer, voltage transformer) or pulse counting.

The first is used in heavy duty installations where the current or voltage of the system under measurement is too high for a measuring device to be connected directly. This is why a secondary circuit is built with reasonable current/voltage in which the measuring device is integrated. Anything that is measured by the device is then multiplied by the same factor that reduces the current/voltage from the main circuit to the secondary circuit. In some cases the value of the device multiplier can change.

The second situation is used for devices that do not count actual consumption units but for example count pulses instead and each pulse then needs to be converted to the desired consumption unit. For example the typical household meter with the rotating wheel that produces a pulse each time the wheel completes a full circle. The speed of the wheel is determined by the amount of energy that is consumed, meaning that each pulse relates to the same amount of energy and can therefore be converted to kWh. The speed of the wheel is determined by the design of the meter and will remain the same for the lifetime of this device.

The CIM model (an international standard developed and used in the utility industry) also talks about device multipliers and distinguishes the actual measurement from the multiplied measurement by using a different reading type in case of measuring electricity. The commodity field of a reading type (digit 6) is responsible for this. In CIM language, the actual measurement is called the secondary measured value (because it was measured on the secondary circuit) and the real consumption, i.e. the result of applying the device multiplier is called the primary measured value.

The use of a multiplier is optional and can be used on a device. However not all the data of the device requires a multiplier, for example measuring volt, text registers, status registers, etc. This is why it is possible to indicate if a multiplier will be used or not on a configuration level. The value of the multiplier it set on the device level as typically this is different for each device.
