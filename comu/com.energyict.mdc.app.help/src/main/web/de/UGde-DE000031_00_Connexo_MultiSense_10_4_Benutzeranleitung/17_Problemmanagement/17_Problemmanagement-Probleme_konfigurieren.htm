Problemerstellungsregeln verstehen

MultiSense bietet die Möglichkeit die Problemerstellungsregeln zu konfigurieren. Diese Option beschränkt die Zeit, die Sie für die Verwaltung von Problemen brauchen, da Sie Regeln schaffen, um zu definieren, welche Ereignisse automatisch ein Problem auslösen. Sie können auch definieren, wann das Problem gelöst werden muss, sowie was mit dem Problem passieren muss, wie es einem bestimmten Anwender zuzuweisen.

Verschiedene Parameter sind notwendig, um die Konfiguration einer Erstellungsregel zu vervollständigen. Zuerst müssen Sie den Problemtyp angeben, das kann beispielsweise ein Datenerfassungsproblem sein. Als nächstes müssen Sie eine Regelvorlage selektieren, um die Parameter der Regel zu definieren.

Eine Regelvorlage ist eine Sammlung von Parametern, die für eine bestimmte Regel typisch ist. Die genauen Werte der Parameter werden in der Regel selbst definiert. Ein Beispiel für eine solche Vorlage ist die Vorlage zur Schaffung eines Problems, wenn ein Rückgang festgestellt wird. Alle Parameter werden angezeigt, aber dann müssen Sie den Ablesetypen, den Trendzeitraum und den Grenzwert angeben.

Ein Problem repräsentiert einen Aspekt, der während der Ausführung einer manuellen oder automatisierten Aufgabe auftritt. Normalerweise ist menschliches Einschreiten notwendig, um das Problem zu analysieren und zu beseitigen, obwohl einige Probleme einfach beseitigt werden können, indem man sie automatisch erneut versucht.

Verfügbare Regelvorlagen

In MultiSense sind verschiedene Regelvorlagen standardmäßig vorhanden. Nachfolgend finden Sie eine Beschreibung dieser Regelvorlagen und wann sie zu nutzen sind:

Regelvorlage

Beschreibung

Datenerfassungsprobleme

 

Ereignisse von Messgeräten von Konzentratoren

Diese Regelvorlage wurde entwickelt um ein Problem zu erstellen, wenn mehrere zusammenhängende Ereignisse auftreten.

Problem erstellen, wenn ein spezifisches Ereignis auftritt

Wenn eine Regelvorlage entwickelt wurde, um Probleme für einfache Ereignisse wie eine verlorene Verbindung oder ein Kommunikationsfehler zu erstellen.

Datenvalidierungsprobleme

 

Probleme erstellen, wenn fehlerverdächtige Ereignisse nicht bewertet werden können

Ein Problem wird erstellt, wenn die Daten nicht mit den Bewertungsregeln bewertet werden können, die bei der konfigurierten Bewertungsaufgabe verwendet wurden.

Erstellungsregeln für ein Datenerfassungsproblem hinzufügen

5

Schritte

1.   Öffnen Sie Verwaltung > Problemmanagement > Problemerstellungsregeln.

2.   Klicken Sie auf Regel einfügen. Füllen Sie die folgenden Felder aus:

Parameter

Beschreibung

Name

Ein eindeutiger Name

Problemtyp

Selektieren Sie Datenerfassung

Regelvorlage

Selektieren Sie die Vorlage mit allen Parametern, um die Regel zu konfigurieren. Je nach selektierter Vorlage werden eventuell zusätzliche Parameter angezeigt.

1.   Ereignisse von Messgeräten von Konzentratoren: Damit wird ein Problem auf der Grundlage mehrerer zusammenhängender Ereignisse erstellt.

  • Grenzwert (%): Gibt an, wie oft das Ereignis auftreten muss, bevor ein Problem erstellt wird. Beispielsweise wird ein Problem erstellt, wenn 50% der Messgeräte des Konzentrators nicht kommunizieren können.

2.   Erstellung von Problemen, wenn ein bestimmtes Ereignis auftritt.

  • Ereignis: Ereignis, welches die Erstellung eines Problems auslöst.

3.   Probleme bei der Registrierung eines Messgeräts: Ein Problem erstellen, wenn ein Nebengerät einige Zeit nicht am Gateway registriert.

  • Verzögerung (in Stunden): Die Zeitdauer, die das betreffende Nebengerät nicht bei einem Gateway registriert ist, bevor ein Problem erstellt wird.

Auto-Auflösung

Schließt Probleme automatisch, wenn die Verbindung oder das Kommunikationsproblem gelöst ist.

Diese Option ist nur für bestimmte Regelvorlagen verfügbar.

Wiederkehrende Probleme

Selektieren Sie Dringlichkeit erhöhen (+1) um die Dringlichkeit automatisch um 1 zu erhöhen, wenn dasselbe Problem erneut auftritt.

Diese Option ist nur für bestimmte Regelvorlagen verfügbar.

Ursache für das Problem

Welche Ursache hat das Problem. Geben Sie einen benutzerdefinierten Grund ein oder selektieren Sie einen der vordefinierten Gründe.

Dieser Grund wird verwendet, um die Probleme auf der Problemübersichtsseite zu gruppieren “Problemübersicht anzeigen”.

Priorität

Die Priorität eines Problems ist die Summe der Dringlichkeit und der Einfluss des Problems. Prioritätsebenen sind wie folgt:

o 1 - 20: sehr gering

o 21 - 40: gering

o 41 - 60: mittel

o 61 - 80: hoch

o 81 - 100: sehr hoch

Dringlichkeit

Definiert die Dringlichkeit des Problems, maximal zulässiger Wert ist 50.

Auswirkungen

Definiert den Einfluss des Problems, maximal zulässiger Wert ist 50.

Fälligkeitsdatum

Wann muss das Problem beseitigt werden.

Kommentar

Zusätzliche Informationen zum Problem.

Aktionen

Definieren, was getan werden muss, wenn das Problem ausgelöst wird. Konfigurationen dieser Handlungen entnehmen Sie bitte “Aktionen zu Problemerstellungsregeln hinzufügen”.

3

Anmerkung

Abhängig von der selektierten Regelvorlage müssen zusätzliche Felder ausgefüllt werden, wie Grenzwert und Ablesetypen.

5

Schritte

3.   Klicken Sie auf Handlung einfügen.

5.   Selektieren Sie eine Handlung: bei Erstellung einem Anwender oder Arbeitsgruppe zuweisen oder das Problem zuweisen oder schließen, wenn es überfällig ist.

6.   Klicken Sie auf Einfügen.

Fügen Sie eine Webservice-Benachrichtigung ein, um das Problem in einer externen Anwendung zu bearbeiten

Durch Hinzufügen einer Webservicebenachrichtigung an einer Problemerstellungsregel können Sie Probleme zum weiteren Verfolgen und Bearbeiten an ein externes System weiterleiten. Ein Beispiel für ein solches externes System ist Jira (Atlassian). Das macht erforderlich, dass ein oder mehrere Webservice-Endpunkte in Connexo Admin erstellt wurden > Webservices > Webserviceendpunkte Wir beziehen uns auf die Gebrauchsanweisung von Connexo Admin für weitere detaillierte Informationen über Webservices.

1

Wichtig

Sie können nur eine Webservicebenachrichtigung pro Problemerstellungsregel hinzufügen.

5

Schritte

1.   Öffnen Sie Verwaltung > Problemmanagement > Problemerstellungsregeln.

2.   Um Aktionen zu einer neuen Erstellungsregel zu fügen, klicken Sie auf Regel hinzufügen.

Um Aktionen zu einer bestehenden Erstellungsregel zu fügen, klicken Sie auf das Aktionssymbol  und selektieren Sie Bearbeiten.

3.   Klicken Sie auf Aktion hinzufügen.

4.   Prüfen, dass Bei Erstellungszeit selektiert ist in Wann ausführen.

Webservicebenachrichtigungen können nur dann ausgelöst werden, wenn das Problem erstellt wird. Es ist nicht möglich Webservicebenachrichtigungen an bestehenden Problemen zu verwenden, die überfällig sind.

5.   Selektieren Sie in Aktion Webservicebenachrichtigung.

Das Feld Webservicebenachrichtigung wird eingefügt.

6.   Selektieren Sie in Webservicebenachrichtigung den benötigten Webserviceendpunkt.

Dieses Drop-Down zeigt alle Webserviceendpunkte, die in Connexo verfügbar sind. Das wurde in Connexo Admin erstellt > Webservices > Webserviceendpunkte. Wenden Sie sich an Ihren Connexo Administrator, wenn Sie zusätzliche Optionen in dieser Liste benötigen und entnehmen Sie weitere Informationen über Webservices der Gebrauchsanweisung von Connexo .

7.   Selektieren Sie das Kästchen Alarm schließen , wenn Connexo nach Weiterleitung an das externe System für weitere Behandlung geschlossen werden soll.

1

Wichtig

Wenn Problem schließen selektiert wurde, erhält das Problem den Status Weitergeleitet, nachdem er von der Webservicebenachrichtigung an den externen Service weitergeleitet wurde. Das Problem wird tatsächlich nicht in Connexo behoben. Bei Bedarf kann das externe System eine Anmerkung zu einem Problem veröffentlichen, sodass Sie seinen Status in diesem anderen System verfolgen können. Wenden Sie sich an Ihr Connexo Lieferteam für weitere Informationen zu den verfügbaren Optionen.

8.   Klicken Sie auf Hinzufügen.

Die Option Webservicebenachrichtigung ist im Feld Aktionnicht mehr verfügbar.

9.   Verifizieren Sie, dass alle anderen Parameter für Ihre Problemerstellungsregel korrekt sind und klicken Sie Hinzufügen (neue Regel) oder Speichern (bestehende Regel), wenn Sie Ihre Arbeit speichern möchten.
