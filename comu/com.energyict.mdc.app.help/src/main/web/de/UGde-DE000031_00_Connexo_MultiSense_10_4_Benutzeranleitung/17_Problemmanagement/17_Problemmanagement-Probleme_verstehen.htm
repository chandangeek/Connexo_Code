Probleme verstehen

Was sind Probleme

Es ist häufig wichtig, dass Anwender informiert werden, wenn während der Datenerfassung und –bewertung etwas schief geht. Anstatt manuell alle Ereignisse prüfen zu müssen, die von verschiedenen Aufgaben stammen, ermöglicht MultiSense Ihnen die Regeln zur Erstellung von Problemen für bestimmte Ereignisse zu konfigurieren. Für weitere Informationen, siehe “Problemerstellungsregeln konfigurieren”.

Wenn Probleme erstellt und nicht automatisch geschlossen wurden, müssen sie behandelt werden. Die Art der Problembeseitigung hängt von der Art der Aufgabe ab, aus der es erstellt wurde. Der Verstoß gegen eine Datenvalidierungsregel erfordert eine andere Untersuchung und Lösung als eine fehlerhafte Kommunikationsaufgabe. Probleme können einem bestimmten Anwender zugewiesen werden.

Probleme in MultiSense

Probleme werden in MultiSense automatisch erzeugt, auf der Grundlage der Regeln, die Sie konfigurieren. Wenn MultiSense ein Problem erstellt, erzeugt es auch eine ID für dieses Problem. Das ermöglicht Ihnen Probleminformationen mit anderen Personen zu identifizieren und zu kommunizieren.

Je nach Problemart wird ein anderer Code verwendet: DCI für Datenerfassungsprobleme und DVI für Datenvalidierungsprobleme . Sie können diese ID verwenden, um nach bestimmten Problemen zu suchen.

Die Problemseite entdecken

Öffnen Sie Arbeitsplatz > Datenerfassung> Problem.

Im oberen Teil der Problem-Seite werden alle verfügbaren Filter angezeigt und können dort konfiguriert werden. Sie können beispielsweise alle Probleme filtern, die durch eine fehlerhafte Verbindung entstehen und Ihnen zugewiesen sind. Die angewendeten Filter werden in diesem Menü angezeigt.

Unter dem Filterfenster gibt es eine Möglichkeit Probleme auf der Grundlage der Parameter  Grund zu gruppieren. Wenn Gruppieren angewendet wird, zeigt die Gruppe im oberen Teil der Seite die Übersichtstabelle. In dieser Tabelle wird angezeigt, wie viele Probleme diese Gruppe enthält.

In der Mitte der Seite wird die Übersichtstabelle angezeigt. Diese Tabelle zeigt alle Problemmeldungen entsprechend den selektierten Filtern. Das Fälligkeitsdatum, Status und Verwalter aller Probleme wird angezeigt. In dieser Tabelle gibt es eine Möglichkeit Probleme unter Verwendung des Aktionssymbols neben jedem Problem zu behandeln.

Im unteren Teil der Seite wird ein Detailfenster angezeigt, welches alle Informationen des Problems anzeigt, welches in der Übersichtstabelle selektiert wird. Nach Klicken auf den Namen eines Problems öffnet sich die entsprechende Problemdetailseite. Auf dieser Seite findet man problemspezifische Details gemeinsam mit Anmerkungen und einer Verbindung zum Gerät, bei dem das Ereignis entstanden ist.

Problemübersicht anzeigen

5

Schritte

1.   Öffnen Sie Arbeitsplatz > Probleme> Problemübersicht.

Diese Seite zeigt eine allgemeine Übersicht aller Probleme in MultiSense. Probleme werden pro Typ, Status, Bevollmächtigten und Grund gruppiert.

2.   Klicken Sie auf einen der Links, um alle Probleme in dieser Gruppe anzuzeigen.

3.   Verwenden Sie die Filter auf der Seite, um eine Übersicht aller Probleme zu erhalten, die dem selektierten Filter entsprechen.

Abbildung: Die Problemübersichtsseite

 

4.   Klicken Sie auf den Karteireiter Vergangenheit, um einen Überblick der Probleme im Laufe der Zeit zu sehen.

Jeder Balken zeigt die relevanten Probleme, die an einem bestimmten Tag entstanden sind, aber repräsentiert nicht die Gesamtmenge der Probleme, die den Kriterien an diesem Datum entsprechen.

5.   Wenn Sie mit der Maus über die Balken in den Diagrammen gehen, sehen Sie die Details der Daten.

Abbildung: Die Problemvergangenheits-Übersichtsseite.

 

Problemvergangenheit an einem Gerät anzeigen

An einem Gerät können Sie alle aktuellen und vergangenen Probleme anzeigen. Das ist praktisch, wenn Sie untersuchen, warum ein bestimmtes Problem immer wieder auftritt. Oder wenn Sie einfach sehen möchten, welches Problem das aktuelle Gerät hat.

5

Schritte

1.   Selektieren Sie ein Gerät mit Suche, siehe “Advanced search”.

2.   Klicken Sie auf Vergangenheit im linken Menü.

3.   Selektieren Sie den Karteireiter Probleme und Alarme.

Konfigurieren Sie die Filter, sodass Sie nur die benötigten Probleme sehen. Die folgenden Filterparameter sind verfügbar:

Parameter

Beschreibung

Typ

Definieren Sie den Problemtyp, den Sie anzeigen möchten. Normalerweise können Sie zwischen Datenerfassung und Datenvalidierungsprobleme n wählen.

Grund

Selektieren Sie einen oder mehrere der konfigurierten Problemgründe.

Zeitraum

Definieren Sie die Zeitspanne, in der Sie die entstandenen Probleme ansehen möchten.

Status

Verwenden Sie diesen Filter um Probleme mit einem speziellen Status anzuzeigen.

4.   Klicken Sie auf Anwenden.
