Hinzufügen eines Sicherheitszugriffsberechtigten für spätere Verwendung

Sie können einen neuen Sicherheitsschlüssel oder Zertifikattypen hinzufügen, um ihn in einer Sicherheitseinstellung, Verwendungsmethode oder als allgemeines Attribut zu nutzen.

5

Schritte

1.   Öffnen Sie Verwaltung > Sicherheit > Sicherheitszugriffsberechtigte.

2.   Klicken Sie die Schaltfläche Sicherheitszugriffsberechtigte hinzufügen.

3.   Klicken Sie auf die Seite Seite mit Sicherheitszugriffsberechtigten hinzufügenund füllen Sie die erforderlichen Felder aus:

Parameter

Beschreibung

Name

Geben Sie einen eindeutigen Namen für den Sicherheitszugriffsberechtigten ein.

Beschreibung

Geben Sie eine optionale Beschreibung ein, um Kontext über den Zugriffsberechtigten zur späteren Verwendung bereitzustellen.

Zugriffsberechtigter-Typ

Selektieren Sie den Typ des Zugriffsberechtigten. Das kann ein Schlüssel oder ein Zertifikat sein. Ihre Auswahl definiert die anderen Parameter, die angegeben werden müssen.

Wenn Sie den Zugriffsberechtigtenschlüssel selektieren, dann:

Schlüsseltyp

Selektieren Sie einen der Schlüsseltypen, die in Connexo verfügbar sind.

Speichermethode

Selektieren Sie eine der verfügbaren Speichermethoden, um die Schlüssel sicher zu speichern. Wenn die Speichermethode Worldline ESS ist, wird das Feld Label angezeigt.

 

Anmerkung

Worldline ESS ist ein optionales Sicherheitslösungsmodul, welches die Back-End-Server des Hardwaresicherheitsmodul (HSM), die mit dem Wordline ESS verbunden sind, verwaltet. Connexo verbindet mit diesem ESS unter Verwendung des Werts im Feld Label.

Label

Dieses obligatorische Textfeld speichert das HSM-Schlüssellabel, das mit dem aktuellen Sicherheitszugriffsberechtigten zusammenhängt. Dieses Label bezieht sich auf das Profil im ESS, das die Aktivitäten festlegt, die mit dem betreffenden Schlüssel erledigt werden können. Jeder Sicherheitszugriffsberechtigte hat ein Label, dass sich auf ein Profil bezieht. Mehrere Sicherheitszugriffsberechtigte können dasselbe Label verwenden. Sicherheitszugriffsberechtigte (und die entsprechenden Label) werden in Firmwareverifikationen, Protokollmanagement, CSR Importern, Versanddateien usw. verwendet. Diese Liste ist nicht beschränkend. Der Wert wird nicht in Richtung HSM validiert.

 

Anmerkung

Am Gerät können Sie das Label-Feld sehen und bei Bedarf übergehen. Siehe “Sicherheitszugriffsberechtigte am Gerät bearbeiten”.

Validitätsperiode

Definieren, wie lange der Schlüssel gültig ist.

Wenn Sie das Zugriffsberechtigtenzertifikat selektieren

Zertifikattyp

Selektieren Sie einen der Zertifikattypen, die in Connexo verfügbar sind.

Trust Store

Selektieren Sie einen der Trust Stores, die in Connexo Admin definiert wurden. Connexo verwendet diesen Trust Store, um die Trust Chain des Zertifikats zu validieren.

Speichermethode

Nur verfügbar, wenn der selektierte Zertifikattyp ein Kundenzertifikattyp ist. Selektieren Sie eine der verfügbaren Speichermethoden.

Zentral verwalten

Prüfen, um den Zugriffsberechtigten zentral zu verwalten und aktive und passive Zertifikate auf Verwaltungsebene anzugeben.

Aktives Zertifikat

Nur angezeigt, wenn Zentral verwalten angekreuzt ist. Selektieren Sie eines der verfügbaren Zertifikate. Die Liste hängt von dem selektierten Zertifikattyp, Trust Store und Speichermethode ab.

Passives Zertifikat

Nur angezeigt, wenn Zentral verwalten angekreuzt ist. Selektieren Sie eine der verfügbaren Zertifikate. Die Liste hängt von dem selektierten Zertifikattyp, Trust Store und Speichermethode ab. Dieses Feld ist optional

4.   Klicken Sie auf Hinzufügen.

3

Anmerkung

Wenn der Zugriffsberechtigten-Typ ein Schlüssel ist, erhält er standardmäßig Ansichts- und Bearbeitungsrechte der Ebene 1 und 2. Um dies zu ändern, siehe “Ändern der Rechte eines Sicherheitszugriffsberechtigten-Schlüssels” für weitere Informationen.
