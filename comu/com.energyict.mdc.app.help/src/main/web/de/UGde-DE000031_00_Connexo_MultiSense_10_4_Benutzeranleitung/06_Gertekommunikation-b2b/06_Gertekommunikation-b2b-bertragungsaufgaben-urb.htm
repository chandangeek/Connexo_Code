Kommunikationsaufgaben verstehen

Kommunikationsaufgaben sind eine Gruppe von Handlungen, die Anwender definieren und dem Gerät vermitteln, um das Gerät eine bestimmte Aktion auszuführen zu lassen, z.B. die Uhr zurückstellen oder Daten zu lesen. Diese Aufgaben werden dem Gerät über eine Verbindung mitgeteilt. Sie können Kommunikationsaufgaben unter Verwendung der gemeinsamen Kommunikationspläne planen.

Abbildung: Gerätekommunikationsmodell

 

Kommunikationsaufgaben in Connexo

In Connexo MultiSense müssen Sie zuerst Kommunikationsaufgabenkonfigurationen konfigurieren (Verwaltung > Gerätekommunikation > Kommunikationsaufgaben), dann können Sie diese Kommunikationsaufgaben für verschiedene Gerätekonfigurationen erneut verwenden. Im ersten Schritt fügen Sie die Handlungen zu den Kommunikationsaufgaben. Diese Handlungen werden in verschiedenen Kategorien gruppiert. Zwei Beispiele sind Belastungsprofile und Uhren.

Sie können auch Kommandokategorien hinzufügen. Diese werden dann gemeinsam mit der Kommunikationsaufgabe ausgeführt. Weitere Informationen finden Sie in “Device commands”.

Wenn Sie diese Kommunikationsaufgaben einmal erstellt haben, müssen Sie sie in der Gerätekonfiguration weiter konfigurieren. So können Sie die Parameter angeben, die für diese Gerätekonfiguration typisch sind. Hier können Sie angeben, welche Verbindungsmethode zu verwenden ist und die Aufgabe mit einer Sicherheitseinstellung verbinden, um zu definieren, wer die Kommunikationsaufgabe sehen oder ausführen kann.

Sie können Kommunikation unter Verwendung gemeinsamer Kommunikationspläne planen, siehe “Schedule communication” für weitere Informationen.
