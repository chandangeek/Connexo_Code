Kommunikationsserver konfigurieren

Einen Online-Kommunikationsserver hinzufügen 

1

Wichtig

Wenn Sie einen Kommunikationsserver erstellen, müssen Sie das Connexo-Gerät, das diesen Kommunikationsserver ausführt, ausschalten und neustarten.

5

Schritte  

1.   Öffnen Sie Verwaltung > Gerätekommunikation > Kommunikationsserver.

2.   Klicken Sie auf Online-Kommunikationsserver hinzufügen. Füllen Sie die folgenden Felder aus:

Parameter

Beschreibung

Name

Einmaliger Name für den Server

Protokollierungsebene des Servers

Allgemeine Protokollierungsebene für die Ereignisse des Kommunikationsservers, die in der Datenbank gespeichert werden (Fehler, Warnmeldung, Information, Debug oder Trace). Es wird empfohlen die Protokollierungsebene “Warnmeldung” zu verwenden.

Kommunikations-Protokollierebene

Protokollierebene für die Kommunikationssitzungen (Fehler, Warnmeldung, Information, Debug oder Trace). Es wird empfohlen die Protokollierungsebene “Warnmeldung” zu verwenden.

Ändert die Inter-Poll-Verzögerung

Erneuerungshäufigkeit für die Eigenschaften des Kommunikationsservers, wenn beispielsweise zusätzliche Ports hinzugefügt werden. Die Änderung wird nach der Frist, die von diesem Parameter angegeben ist, übernommen.

Inter-Poll-Verzögerung wird geplant

Erneuert ausstehende Aufgaben und übernimmt neue Aufgaben auf der Grundlage des definierten Intervalls.

Task-Queue-Größe speichern

Die Anzahl der Aufgaben, die vom Kommunikationsserver bearbeitet werden kann, bevor die Daten an die Datenbank geschickt werden.

Task-Thread-Zählung speichern

Die Anzahl der Mitarbeiter, die Daten von der Anfrage zum Speichern in der Datenbank erhalten.

Task-Warteschlange-Priorität speichern

Java VM Priorität gegenüber anderen Java-Anwendungen, die im OS laufen. Je höher der Wert, desto höher die Priorität.

 

Die folgenden Felder sind notwendig um den Monitor des Kommunikationsservers nutzen zu können.  Siehe “Einen Kommunikationsserver überwachen”.  Sie enthalten die Informationen, die für die Einrichtung der URL zum Kommunikationsserver-Monitor benötigt werden, und die aus Servername, Port und Pfad zur Anwendung bestehen. Beispiel: http://server015:8080/apps/comservermonitor/index.html 

Servername

Name des Geräts, auf dem der Kommunikationsserver läuft.

Ereignisregistrierungs-Port

Port, an dem der Kommunikationsserver-Monitor auf die veröffentlichten Ereignisse hören muss, um die ununterbrochene Protokollierung am Protokollier-Karteireiter im Kommunikationsserver-Monitor auszuführen.

3.   Klicken Sie auf Einfügen.

4.   Klicken Sie auf das Aktionssymbol  und selektieren Sie Aktivieren.

3

Anmerkung

Je höher die selektierte Protokollierebene z.B. Debug ist, desto mehr Speicherplatz nehmen diese Daten in der Datenbank ein.  Sie sollten sich dessen bewusst sein, wenn Sie die Protokollierebene des Kommunikationsservers erhöhen.

Kommunikationsserver bearbeiten

5

Schritte  

1.   Öffnen Sie Verwaltung > Gerätekommunikation > Kommunikationsserver.

2.   Klicken Sie auf das Aktionssymbol  und selektieren Sie Bearbeiten.

3.   Ändern Sie die Parameter des Kommunikationsservers.

4.   Klicken Sie auf Speichern.

Kommunikationsserver löschen

5

Schritte  

1.   Öffnen Sie Verwaltung > Gerätekommunikation > Kommunikationsserver.

2.   Klicken Sie auf das Aktionssymbol  und selektieren Sie Löschen.

3.   Klicken Sie auf Löschen.
