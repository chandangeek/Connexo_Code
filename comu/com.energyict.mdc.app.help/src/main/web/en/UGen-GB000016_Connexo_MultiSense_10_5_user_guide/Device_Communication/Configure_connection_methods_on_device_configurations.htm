Configure connection methods on device configurations

Understand connection methods

The connection method allows you to define how to communicate with the device. Some devices can support multiple types of connection methods (GRPS, optical, SMS, etc.). You can configure which is the preferred type. Using connection methods you can minimize your cost via the connection strategy. You can select to minimize the connections or connect as soon as possible.

If you choose to minimize the connections then Connexo MultiSense will only connect to a device when the connection method allows it. A communication task will then only be executed at the time the connection is scheduled. If you choose to connect a soon as possible, then a connection will be set up as soon as a communication task needs a connection. This is only possible for devices that use the master-slave setup.

Add outbound connection methods

1

Important

Before you add a connection method make sure that the device is Directly addressable. If not, then no connection methods can be added. This can be selected on the device configuration edit page. See “Configure device configurations”.

5

Steps

1.   Go to Administration > Device management > click Device type > select a device type > click Configurations > select a configuration > click Connection methods in the menu on the left.

2.   Click Add connection method > Add outbound connection method. Complete the following fields:

Parameter

Description

Name

A unique name.

Connection type

Select a connection types. The connection type how the device will connect with the system. Which type is selected determines which connection details need to be completed.

Protocol dialect

Select one of the protocol dialects available on the device configuration.

Communication port pool

Select a communication port pool

Connection strategy

As soon as possible: connect whenever a communication task needs a connection.

Minimize connections: only connect with the device at the scheduled time.

Connection schedule

Only available when “Minimize connections” is selected. The schedule when to set up a connection.

Connection window

Is the time window during which the device is active and it is possible to set up a connection and communication tasks can be executed. Is not used when the device is always accessible.

Retry delay

The period of delay before the connection will be retried.

Number of simultaneous connections

Allows multiple connections at the same time to make parallel scheduling of devices possible.

By default this is set to 1 and the maximum allowed value is 16.

Connection details

Specify the connection details such as port number and security keys and certificates. All devices that use this configuration will inherit these parameters. These can be overruled on the device itself.

These fields are not mandatory.

On the device itself you can set device specific parameters such as IP address.

3.   Click Add.

4.   Click the actions icon and select Set as default to make the connection method the default method.

Edit connection methods

5

Steps

1.   Go to Administration > Device management > click Device type > select a device type > click Configurations > select a configuration > click Connection methods in the menu on the left.

2.   Click the actions icon  and select Edit.

3.   Change the connection method parameters.

4.   Click Save.

Steps

2.   Click the actions icon  and select Remove.

3.   Click Remove.
