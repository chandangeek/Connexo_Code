Configure device commands on a device

Add commands to a device

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Commands in the menu on the left.

3.   Click Add command. Complete the following fields:

Parameter

Description

Command category

Select the category of commands.

Command

Select the command. If the command is not part of the device’s communication tasks, a warning message will be shown.

If the command is part of a communication task that is not scheduled on the device, then also here a warning message will be shown. However the commands can still be added to the device.

Release date

Select the data when the command will become available to be picked up when there is a connection.

Attributes

Some command require extra attributes to be specified such as the message that needs to be sent to the device.

4.   Click Add.

5.   After adding the command, Connexo MultiSense asks if you want to trigger the command now. If yes, click Trigger otherwise click Cancel.

Add a command to multiple devices in a device group

In order to avoid adding a command to each device separately, you can use a device group to add it to multiple devices at once. This saves you a lot of time.

5

Steps

1.   Go to Workspace > Data communication > Commands.

2.   Click the button Add command.

3.   In the first step of the Add command wizard, select the device group containing the devices to which the command needs to be sent.

4.   Click Next to go to the second step.

5.   In Command category, select a command category.

This determines the list of available commands in the field Command.

6.   In Command, select a command.

Only the commands that are enabled on ALL devices in the selected device group and for which you have sufficient privileges on the device configuration are available.

After selecting the command, specific attributes for the command may appear on screen. For instance, for the command Set time, you need to specify the new meter time.

7.   Verify the release date. By default, the current date and time are proposed.

8.   Complete the command attributes. The attributes are all mandatory.

9.   Click Next to go to the Confirmation step.

At this point Connexo verifies if any command limitation rule applies to this command. In case the limit is reached you won’t be able to add this command. If there isn’t any obstruction, then you will be able to confirm adding the command.

While confirming you have the opportunity to trigger the communication task immediately to execute the command. By default, this option is set to No.

10.   Click Confirm to add the command to all devices in the device group and to see the status in the final step.

11.   Click Finish to close the status step and to return to the commands overview.

Trigger communication for a command

If needed, you can trigger the command immediately without waiting for the next communication task to pick it up.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Commands in the menu on the left.

3.   Click the actions icon  next to the command you want to trigger and select Trigger now. Note that only pending and waiting commands can be triggered.

4.   Click Trigger.

or

5

Steps

1.   Go to Workspace > Data communication > Commands.

2.   In the table, select the command that you need to trigger.

3.   Click the actions icon and select Trigger now.

4.   Click Trigger.

Change the release date of a command

If needed, you can make the command available sooner than planned.

1

Important

Can only be done when the status of the communication task is “Pending”.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Commands in the menu on the left.

3.   Click the actions icon  next to the command you want to change the release date of and select Change release date.

4.   Change the release date en click Save.

or

5

Steps

1.   Go to Workspace > Data communication > Commands.

2.   In the table, select the command for which you need to change the release date.

3.   Click the actions icon and select Change release date.

4.   Enter the new release date and time.

5.   Click Save.

Revoke commands

If the command is no longer applicable, then you can revoke it. It will no longer be available.

1

Important

Can only be done when the status of the communication task is “Pending”.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Commands in the menu on the left.

3.   Click the actions icon  next to the command you want to revoke and select Revoke.

4.   Click Revoke.

Steps
