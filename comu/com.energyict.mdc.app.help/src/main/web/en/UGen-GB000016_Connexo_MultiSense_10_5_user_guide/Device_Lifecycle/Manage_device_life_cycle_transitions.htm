Manage device life cycle transitions

Understand transitions

A transition is the action where a device goes when one state to another. This transition is linked with pretransition checks and auto-actions. The pretransitions checks are different requirements that need to be met before the device can transition. The auto-actions are automated activities that ensure the device is ready for the next state.

When configuring a transition you will need to select the trigger that activates the transition. Besides the trigger event you have to select the state the transitions starts from and the state the transition ends in. The combination of these three parameters needs to be unique in the device life cycle. Another important aspect of the configuration of transitions is defining who is allowed to execute the transition.

During the configuration of the transition you can also add additional optional pretransition checks and auto-actions. By default every transition has different mandatory pretransition checks and auto-actions that are always executed together with specific states.

Add transition

5

Steps

1.   Go to Administration > Device life cycle management > Device life cycles > click a device life cycle name > click Transitions in the menu on the left.

2.   Click Add transition and complete the following fields:

Parameter

Description

Triggered by

Select the event that will trigger the transition.

Name

Automatically generated based on the trigger, but can be changed.

From

The start state.

To

The end state

Privileges

Which users can execute the transition. The different levels are configured in Connexo Admin.

Pretransition checks

Select which checks should be monitored before a device can transition from one state to another. More information is available when you click the info icon. By default only the pretransition checks that are optional are shown, but you can choose to also see those that are selected by default by clicking Show all. For more information on pretransition checks see “Understand transitions”.

Auto-actions

Select which auto-actions should be executed before a device can transition from one state to another. More information is available when you click the info icon. By default only the auto-actions that are optional are shown, but you can choose to also see those that are selected by default by clicking Show all. For more information on auto-actions see “Understand transitions”.

3.   Click Add.

3

Note

The combination of the trigger, from state and to state needs to be unique in the device life cycle.

Edit transition

5

Steps

1.   Go to Administration > Device life cycle management > Device life cycles > click a device life cycle name > click Transitions in the menu on the left.

2.   Click the actions icon  and select Edit.

You can edit every parameter however check if the combination trigger/from/to is unique. If not an error message will be shown and the change will not be saved.

3.   Click Save.

Remove transition

5

Steps

1.   Go to Administration > Device life cycle management > Device life cycles > click a device life cycle name > click Transitions in the menu on the left.

2.   Click the actions icon  and select Remove.

A transition can always be removed even when it is part of a life cycle that is in use.

3.   Click Remove.
