Add validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > click Validation rules in the menu on the left.

2.   Click Add validation rule. Fill in the following fields:

Parameter

Description

Name

A unique name

Validator

Select how to perform validation. Depending on the validator selected, extra parameters may need to be defined.

Reading type(s)

Select the data that will be validated. Multiple reading types can be added as described in “Add reading type”.

Data quality level

Suspect: the data is invalid and needs attention. This value will be subject to estimation. These values are indicated in red on the graphical representation of the data.

Informative: the data is valid, but could be an indication that something is going wrong. This value will not be estimated. These values are indicated in yellow on the graphical representation of the data.

3.   Click Add.

4.   Click the actions icon to activate or inactivate the rule.

3

Note

The following validators are available by default, they are indicated by the [STD] label. Validators without this label are custom validators. For more information on these validators please contact your system administrator.

Validator

Description

Check missing values[STD]

Check for missing data. This data can be provided in a later stage by estimation and editing.

Reading quality [STD]

Reading qualities are comments added to values to indicate that something occurred at the time of that reading, for example a power outage.

The reading quality validator checks interval data and register readings if they contain one or more of the specified reading qualities. Note that for the system and index code of a reading qualities you can use the wildcard “*”. In this case, a reading with a reading quality with the correct category will be marked as suspect, regardless of its system or index code.

Click Add reading quality to specify the reading qualities that will be checked with this validator.

Register increase [STD]

Check consumption over time. If the value of the register is lower or equal to the previous register value then this data is marked as suspicious.

Check Fail equal data if you want to mark equal values as suspicious.

Threshold violation [STD]

Checks each interval value against a minimum and maximum value. Values outside these boundaries are marked as suspicious.

You can edit the threshold parameters on the validation configuration page of the channel.

Consecutive zeros [STD]

This validator allows you to mark data that consists out of consecutive zeros in a predefined period. This rule only works for delta reading types.

Complete the following parameters to configure this estimator:

oMinimum period and Maximum period: defines the period to look for consecutive zeros. If the period is larger than the minimum period and smaller than the maximum period then data from this period will be marked as suspect. If the period is smaller than the minimum period or larger than the maximum period, this block will be marked as valid.

  • Minimum threshold: for loads that are never zero you can configure the minimum threshold. Values smaller or equal to this threshold will be counted as a zero.

  • Check retroactively: if you check this option, values before the lasted checked date will be taken into account if they reside in the defined time window.    Connexo MultiSense will check for consecutive zeros in the period calculated by the lasted checked date minus the maximum period. Values that were marked as valid by the previous validation run, can now become suspect.

Meter advance [STD]

Use this validator to check the sum of the interval values against two meter register values. If the difference is higher than the maximum defined, the rule will mark all values between the two register readings as suspect.

Complete the following parameters to configure this estimator:

oReference reading type: select the register to compare the channel readings to.

  • Maximum absolute difference: define the absolute or relative difference between the sum of interval channel data end the difference between the two register readings.

  • Reference period: skip readings if no cumulative register readings are found for a period larger than the reference period.

  • Minimum threshold: to avoid comparing small consumptions set a minimum threshold to skip the intervals for which the difference between the two cumulative register readings is too small.

Edit validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > Validation rules.

2.   Click the actions icon  and select Edit.

3.   Change the validation rule parameters.

4.   Click Save.

Edit threshold violation rule on channel validation configuration page

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Data sources in the menu on the left and click Channels. Select a channel and select the Validation configuration tab on top of the page.

3.   The editable parameters are marked by an empty pencil icon .

4.   Click the actions button and select Edit “validation rule”.

5.   Change the threshold values and click Save.

6.   Changed threshold values are marked by a filled pencil icon .

Remove validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > Validation rules.

2.   Click the actions icon  and select Edit.

3.   Click Remove.

Add reading type

5

Steps

1.   Click Add reading types.

2.   Complete the filter fields to find the correct reading types.

3.   Click Apply.

Deselect items in the drop down menu to remove single filters or click Clear all to remove all applied filters.

4.   Select one or multiple reading types from the table.

You can see the reading type details by clicking the information icon .

5.   Click Add.
