What are alarms

Alarms are triggered by logbook events. These events are detected by the sensors that are installed on the device and are written to the event logbook. Such an event can for example be a tamper event, power loss, pressure loss, low battery, low voltage, etc. Typically an alarm will require an action or input in the field.

Whether an event can trigger an alarm, is defined by the alarm creation rule (see below). Not all events will result in alarms because not all of these require attention. For example. communication failures or time sync errors. Other more critical events can be pushed by the meter. The priority of an alarm is determined by the sum of its impact and urgency. In addition, thresholds can be used to add an alarm. For example an alarm is only created if more than 100 power downs occur within one hour or when 5 failed login attempts are detected on a device within one day.

The creation of alarms depends on the state of the device. For example, when installing a meter, tamper alarms will be generated but these are not relevant. Therefore they are discarded. However when a tamper alarm occurs on an installed meter, this is valid and needs to be addressed.
