Understand privileges

In Connexo Admin the system administrator adds privileges to different user roles. These define what a user is allowed to do within the different parts of the applications. These privileges are further divided in levels. With these levels you can group user roles. For example, level one can be used for people working with electricity meters while level two is used for people working with gas meters.

In Connexo MultiSense you can link these privileges to the security settings on the device type. This will define what different users can do on a device. The following scenarios are possible:

  • No privileges:

  • A user that has no privileges will not see the different security attributes.

  • All action buttons are disabled.

  • Only view privileges:

  • The names of the different security attributes are shown. Values are encrypted and shown as ********, if the value is not empty.

  • All action buttons are disabled.

  • Only edit privileges:

  • The names of the different security attributes are shown. Values are encrypted and shown as ********, if the value is not empty.

  • All action buttons are enabled and it is possible to edit values.

  • View and edit privileges:

  • The names of the different security attributes are shown. Values are encrypted and shown as ********, if the value is not empty.

All action buttons are enabled and it is possible to edit values and show or hide the value.
