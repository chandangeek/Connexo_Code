Understand issues

What are issues

It is often important that users are notified when something goes wrong during data collection and validation. Instead of having to manually verify all events that originate from different tasks, MultiSense allows you to configure rules to create issues for specific events. For more information see “Configure issue creation rules”.

Once issues have been created and are not closed automatically, they need to be handled. The way to solve an issue depends on the type of task from which it has been generated. The violation of a data validation rule requires a different investigation and solution than a failed communication task. Issues can be assigned to a particular user.

Issues in MultiSense

Issues are created automatically in MultiSense based on the rules you configure. When MultiSense creates an issue, it also generates an ID for this issue. This allows you to identify and communicate issue information with other people.

Depending on the issue type a different code is used: DCI for data collection issues and DVI for data validation issues. You can use this ID to search for specific issues.

Explore the issues page

Go to Workspace > Data collection > Issues.

On top of the Issues page all available filters are shown and can be configured here. You can for example filter all issues raised by a failed connection and assigned to you. The applied filters are shown in this menu.

Below the filter pane there is a possibility to group issues based on the parameter Reason. If grouping is applied then the group will be shown on top of the page above the overview table. In this table it is indicated how many issues this group contains.

In the middle of the page the overview table is displayed. This table shows all issues corresponding to the selected filters. The due date, status, assignee and a reference to the concerned device of all issues are shown. In this table there is an option to handle issues by using the actions icon next to each issue.

On the bottom of the page a details pane is shown that contains all information of the issue that is selected in the overview table. By clicking the name of an issue, the corresponding issue detail page will open. On this page issue specific details can be found together with comments and a link to the device that raised the event.

View issues overview

5

Steps

1.   Go to Workspace > Issues > Issues overview.

This page shows an overall overview of all issues in MultiSense. Issues are grouped per type, status, assignee and reason.

2.   Click one of the links to see all issues in that group.

3.   Use the filters on top of the page to see get an overview of all issues corresponding to the selected filters.

Figure: The issues overview page

 

4.   Click the tab History to see an overview of the issues over time.

Each bar shows the relevant issues that were raised on a particular day and doesn’t represent the total number of issues that answer the criteria on that date.

5.   Hover over the bars in the charts to see the details of the data.

Figure: The issues history overview page.

 

View issue history on device

On a device you can see all its current and past issues. This comes in handy when you are investigating why a particular issue keeps reoccurring. or if you just want to see what issues the current device has been running into.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click History in the menu on the left.

3.   Select the Issues and Alarms tab.

Configure the filters to only see the issues you need. The following filter parameters are available:

Parameter

Description

Type

Define the type of issue you want to see. Typically you can choose between data collection and data validation issues.

Reason

Select on or more of the configured issue reasons.

Period

Define the time period in which you want to see the issues raised.

Status

Use this filter to only see issues with a specific status.

4.   Click Apply.
