Manage data collection issues

View data collection issues

5

Steps

1.   Go to Workspace > Data collection > Issues or click My open issues to see all open issues assigned to you.

2.   Click a data collection issue. This will take you to the issue’s overview page. This page shows all issue details and is divided into four parts.

Part A (as indicated in the figure) shows all general issue information such as the reason and type of the issue as well as the status and when the issue is due.

Part B shows the communication or connection details of this issue. Here you can see the device linked to this issue together with the device type and configuration and usage point. Here you can also see other details linked to communication and connections such as the communication task used together with the last used connection or the connection method and the latest status.

Part C shows the most recent communication logging, which will help you in solving this data collection issue.

Part D contains additional contextual information. This part contains three tabs: Timeline, Comments and Processes. Timeline combines all latest comments and processes. The Comments tab show all comments added to this issue and the Processes tab shows all processes linked to the selected issue.

Figure: Data collection issue overview

 

On this page you can also perform different actions such as adding a comment or handling the issue by clicking the actions button  and selecting an action. For more information see the topics described below.

3

Note

Alternatively, you can go to Workplace > Issues > Issues and filter on type Data collection.

Assign issues

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Filter issues as described in “Filter issues using the filter menu”.

3.   Click the actions icon . next to the issue, the actions button  in the details pane or click the issue name and click the actions button on the details page.

4.   Click Assign. Select the user to assign the issue to by completing these fields:

Parameter

Description

Workgroup

Assign to a group of users.

User

Assign to a single user.

Comment

Add a comment for the assigned user(s).

Alternatively, you can click Assign to me to assign the selected issue directly to yourself.

5.   Click Assign.

Note that you can also assign multiple issues simultaneously. See “Manage issues in bulk”.

3

Note

After assigning an issue the action Unassign becomes available for the selected issue. Click this option to remove the link between the issue and its assignee.

Add a comment to an issue

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue, the actions button  in the details pane.

3.   Click Add comment.

4.   Write the comment on the bottom of the page.

5.   Click Add.

or

1.   Go to Workspace > Data collection > Issues.

2.   Click an issue title.

3.   Click Add comment on the detail page of the selected issue.

4.   Click Add.

Edit or remove a comment on an issue

You can edit and remove the comments that you added yourself to the data collection issue. Comments added by other users can only be edited by those users.

5

Steps

1.   Open the issue for which you want to edit or remove the comment. See “View data collection issues”.

2.   Find the comment in the section Contextual information under the tab Comments at the bottom of the page.

3.   Click the pencil icon that you find at the end of the comment to edit the comment.

4.   Click Save to save your edits.

5.   Click the remove icon that you find at the end of the comment to remove the comment.

6.   Click Remove to confirm removal of the comment.

Close issue

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue, the actions button  in the details pane or click the issue name and click the actions button on the details page.

3.   Click Close. Complete the following fields:

Parameter

Description

Close status

Reason for closing the issue.

Comment

Possibility to add a comment.

4.   Click Close.

Note that you can also close multiple issues simultaneously. See “Manage issues in bulk”.

Snooze issue

You can snooze an issue to remove it from all operational screens for a specific amount of time. This is handy if you decide that the issue shouldn’t have your attention during that time. Note that you can also snooze multiple issues simultaneously. See “Manage issues in bulk”.

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue, the actions button  in the details pane or click the issue name and click the actions button on the details page.

3.   Click Snooze.

4.   Enter the date and time until when the selected item shouldn’t appear on the default overview table or on any operational screen. By default, the snoozing time is one day.

5.   Click Snooze.

The snoozed issue is no longer visible on the issue overview table with the default filter parameters. You can still retrieve the snoozed items by selecting Snoozed in the filter parameter Status.

You can change the snoozing parameters of a snoozed issue by repeating the above procedure.

When the configured snoozing end time is reached, the snoozed status is removed and the selected issues becomes visible again on all operational screens.

1

Important

To check snoozing end time of snoozed issues a recurring task is used. This task must be active in Connexo Admin > Application servers > Your application server > Message services. Contact your Connexo administrator for support or refer to the Connexo Admin user guide.

Retry connection/communication

1

Important

Only available for connection and communication issues.

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue, the actions button  in the details pane or click the issue name and click the actions button on the details page.

3.   Select Retry now. The connection/communication task will be re-triggered.

Note that you can also retry a connection/communication on multiple issues simultaneously. See “Manage issues in bulk”.

Set issue priority

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue or the actions button  and select Set priority.

3.   Change the urgency and/or the impact of the issue.

4.   Click Save.

Note that you can also set the priority of multiple issues simultaneously. See “Manage issues in bulk”.

Manage issues in bulk

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click Bulk action to handle multiple issues at the same time.

3.   In step 1 of the wizard select the issues. Select all open issues or make a custom selection.

4.   Click Next.

5.   In step 2 select the action to handle the issue.

6.   Click Next.

7.   In step 3 specify the details of the action.

8.   Click Next.

9.   In step 4 click Confirm to confirm the action.

10. In step 5 a message is displayed with the details of the action and the issues.

11. Click OK.

When the action was not successful a detailed error message is shown together with the possibility to retry or finish the action.

View connection/communication log

1

Important

Only available for connection and communication issues.

5

Steps

1.   Go to Workspace > Data collection > Issues.

2.   Click the actions icon  next to the issue, the actions button  in the details pane or click the issue name and click the actions button on the details page.

3.   Click View connection log or View communication log the connection or communication logging is shown as described in “View connection log” and “View communication log”.

This information will help you to identify a connection/communication problem and how it could be solved.
