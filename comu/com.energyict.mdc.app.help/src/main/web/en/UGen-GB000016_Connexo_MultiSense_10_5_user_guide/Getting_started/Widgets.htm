Figure: Common widgets on the object landing page.1. Vertical navigation bar.2. What’s going on? widget.3. Device summary.

Description

Vertical navigation bar

From here you can navigate to specific information about the current object. The links that are shown here depend on the functionality of the object. For example, in case of a gateway, the Logbooks, Register types and Load profiles links will be hidden from the navigation menu because they are not configured on the gateway. When clicking a link on the navigation menu you will navigate away from the object landing page. If you want to keep it open, then open the links in a new window or tab.

What’s going on widget

Quick overview of alarms, issues, processes and service calls that are open for the object. Each item can be clicked to navigate to its corresponding landing page. See “The “What’s going on” widget”. This widget gives an idea of the object’s performance.

Device summary

This widget shows a summary of the device attributes. It gives you a clear view on what the object can do and where it fits in the structure. The device attributes can be edited. See “Edit device attributes on a device”. In this section you can see an image of the device. This image is defined on the device type with which the device is related. See “Edit or remove a device icon on a device type”.

The “What’s going on” widget

The “What’s going on” widget informs you about the object’s performance by listing issues, processes, alarms and service calls that are currently open for the object.

You filter the list using the drop-down filter in the upper right corner of the widget.

There are ten items per page that are arranged in two columns. Use the dots under the list to navigate to another page. The current page is marked by a white dot with black border.

Each item can be clicked to navigate to its corresponding landing page. For processes the user will be directed to the table with ongoing processes for the current object.

Each item is preceded by a circular icon which has a letter and a colour code.

The letter indicates the type of information in an item:

Letter

Information type

A

Alarm

I

Issue

P

Process

S

Service call

Hovering over the icon shows more detailed information like type, status, assignee and due date. The colour of the tooltip border and title reflect the icon border colour.

The colour code indicates the severity (high, warning, none) and the assignment (current user, other user, unassigned)

Other widgets on the object landing page

Parameter

Description

Communication topology

Shows the most recently added slave devices. In case of a HAN gateway, all slave devices are shown. In case of a LAN gateway, only the most recently added slave devices are shown. The list includes links to the landing pages of the corresponding devices.

Data logger slaves/ Multi-element devices

If the device is a data logger or multi-element device, this widget will show the most recently added slave devices. The list includes links to the landing pages of the corresponding devices.

Data validation

Shows the status of the validation, together with the last validation date. Depending on the device status the device resides in, validation information may not be shown. If there are no registers, load profiles and channels configured, there will be no validation information either.

Connections

Shows all connection methods that are available on this device together with their status. Click the actions icon  to perform the following actions: Run now, Deactivate or Activate the connection method or View history.

Communication tasks

Shows all communication tasks available on this device. Click the actions icon  to perform the following actions: Run now: Deactivate or Activate the connection method or View history. Activate or deactivate all communication task at once by clicking Activate all or Deactivate all respectively.
