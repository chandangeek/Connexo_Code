Firmware campaigns

In MultiSense you can upload firmware on multiple devices using a single action. These are called firmware campaigns. These firmware campaigns are created for a specific device type and device group. Only the devices of the device group that have the specified device type will get a new firmware version.

View firmware campaigns

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

In the overview table you can see all configured firmware campaigns and their details. The table shows the name of the firmware campaign, the device type on which the firmware will be uploaded, the type of firmware, the firmware version and when the campaign was started.

In the table you can also see the status of the different devices in the firmware campaign. Here you see the amount of devices with a successful upload, the amount of failed uploads and the amount of uploads failed due to a configuration error. Next to these options you can also see the total amount of ongoing, pending and cancelled uploads.

The data is in this table can be exported to a CSV using the grid export button .

2.   Click the firmware campaign name to see the overview page of the firmware campaign. This page shows the same details as the table as well as the firmware management options defined by communication protocol and device type.

3.   Click Devices in the menu on the left to see which devices are included in the campaign and what their status is.

You can filter the devices in the firmware campaign on their status using the status filter on top of the page. This allows to quickly find the devices that failed to upgrade their firmware and take the appropriate action.

Add firmware campaign

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

2.   Click Add firmware campaign. Complete the following fields:

Parameter

Description

Name

Name of the firmware campaign. A default name is provided, but can be overwritten.

Device type

Select the device type to define which device will upload the firmware.

Device group

Select the device group that contains all devices that need the firmware upload. Only the devices with the device type defined in the device type parameter will be subject to the firmware campaign.

Time boundary start

Start time of the time boundary within which the firmware campaign can start new firmware uploads.

Time boundary end

End time of the time boundary within which the firmware campaign can start new firmware uploads.

Firmware management option

Select when to upload and activate the firmware on the device. See “Understand firmware management options” for more information.

Firmware type

Select the type of firmware to upload. When there is only one type of firmware available on this device type this will be selected by default.

Firmware version

Select the firmware version to upload on all devices.

Activation date

Define the activation date of the firmware version when you select to upload the firmware with activation date.

3.   Click Add.

3

Note

Equal Time boundary start and Time boundary end will pause the firmware campaign.

Cancel firmware campaign

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

2.   Select the desired firmware campaign.

3.   Click the actions icon  and select Cancel.

It is only possible to cancel firmware campaigns that have the status ongoing.

4.   Click Cancel campaign.

When you cancel the firmware campaign the upload to all devices that still needs to be executed will be cancelled, ongoing uploads will be finished and successful uploads will not be reversed.

Edit firmware campaign

Editing a firmware campaign is only possible if the user has sufficient privileges and for firmware campaigns that have the status Ongoing.

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

2.   Select the desired campaign.

3.   Click the actions icon  and select Edit.

4.   Make the needed adjustments.

Only Name and Time boundaries can be edited. The change on time boundaries only impacts devices that have the status Pending.

5.   Click Save.

Cancel a device in a firmware campaign

It is possible to skip one or more devices in a firmware campaign to prevent that the firmware version is uploaded to these devices. This action may be required when a device is ‘trapped’ in the Ongoing status - for example when the device is no longer available (installed) in the field. Skipping this particular device will allow the firmware campaign to be completed.

Cancelling a device in a firmware campaign is only possible if the user has sufficient privileges, for firmware campaigns that have the status Ongoing and only for devices that have the status Pending or Ongoing.

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

2.   Select the desired firmware campaign.

3.   Click the tab Devices and select the desired device.

4.   Click Devices and select the desired device.

5.   Click the actions icon  and select Cancel.

6.   Confirm the cancellation.

Retry a device in a firmware campaign

A device for which the firmware upgrade did not succeed during a firmware campaign can be retried without restarting the whole firmware campaign as long as the firmware campaign is still running.

Retrying a device in a firmware campaign is only possible if the user has sufficient privileges, for firmware campaigns that have the status Ongoing and only for devices that have the status Cancelled, Failed or Configuration error.

5

Steps

1.   Go to Workspace > Firmware management > Firmware campaigns.

2.   Select the desired firmware campaign.

3.   Click the tab Devices and select the desired device.

4.   Click the actions icon  and select Retry.

The status of the device becomes Pending. This means that it will be picked up by the firmware engine for retrying the upgrade. Keep an eye on the status of this device to follow up on its progress. See “View firmware campaigns”.

View firmware history on a device

A history of firmware changes is kept for each device. It allows you to see what firmware was active at a particular moment in time. Because firmware contains bug fixes and new features, firmware history on the device will help you understand from what moment you may expect benefit from those new features, e.g. extra meter events, register objects, fixed issues, etc.

5

Steps

1.   Open a device using search. See “Advanced search”.

2.   Click History.

3.   In the History window, click the tab Firmware.

All firmware versions that have been active including the firmware version that is currently active on the device are shown in a table with their activation date, firmware version and firmware type. The most recent firmware versions are ordered on top.
