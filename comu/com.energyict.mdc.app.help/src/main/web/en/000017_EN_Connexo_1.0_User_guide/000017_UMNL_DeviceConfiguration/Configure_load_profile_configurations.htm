Configure load profile configurations

Add load profile configurations

1

Important

Add load profile types to device types to be able to add and configure load profile configurations.

5

Steps

1.   Go to Administration > Device management > click Device type > select a device type > click Configurations > select a configuration.

2.   Click Load profile configurations in the menu on the left.

3.   Click Add load profile configuration. Select one of the load profile types that were added previously to the device type.

4.   The OBIS code is automatically generated based on the load profile type you selected. You can overrule this value.

5.   Click Add

3

Note

Load profile configurations can only be added when the device configuration is inactive. This is to prevent any possible effects on the data collection process.

On the overview page of the load profile the applied validation rules are shown. From here you can go to the validation rules page, for more information see “Data validation”.

Edit load profile configuration

5

Steps

1.   Go to Administration > Device management > click Device type > select a device type > click Configurations > select a configuration > click Load profile configurations in the menu on the left.

2.   Click the actions icon  and select Edit.

3.   Change the load profile configuration parameters.

4.   Click Save.

Remove load profile configuration

5

Steps

1.   Go to Administration > Device management > click Device type > select a device type > click Configurations > select a configuration > click Load profile configurations in the menu on the left.

2.   Click the actions icon  and select Remove.

3.   Click Remove. If the device configuration is active, load profile configuration can not be removed.

Add channels

5

Steps

1.   Click a load profile configuration.

2.   Click Add channel configuration. Complete the following fields:

Parameter

Description

Register type

Select the data of which you want to see interval readings. Only reading types that have not been used for other channels are available

 OBIS code

OBIS code linked to register type. Overrule the OBIS code if the OBIS code of the configuration would be different that the one of the device type.

Overflow value

The overflow value defines the maximum value that can be stored.

Number of fraction digits

Amount of digits after the decimal.

Use Multiplier

Indicate whether a multiplier should be used. Only use when the consumption data needs to be multiplied a specific value to get the actual consumption rate.

The value of the multiplier can be set on the device.

Collected reading type

The reading type of the data that is collected from the device.

Calculated reading type

The reading type of the calculated data, this reading type is used when values need to be calculated as consequence of a multiplier or to calculate delta values when bulk values were collected.

3.   Click Add.
