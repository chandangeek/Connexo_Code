Configure device configurations

View device configurations

5

Steps

1.   Go to Administration > Device management > Device type.

2.   Select a device type by clicking the device type name or create a new one as described in “Add a standard device type”.

3.   Click Configurations in the menu on the left.

You can see all device configurations of the device type in the overview table. Select a configuration to see its details in the details pane below. Here can see the device configurations parameters such as its data logger functionality. You can also see the status of the configuration together with links to its data sources.

Add device configurations

5

Steps

1.   Go to Administration > Device management > Device type.

2.   Select a device type by clicking the device type name or create a new one as described in “Add a standard device type”.

3.   Click Configurations in the menu on the left.

4.   Click Add device configuration. Complete the following fields:

Parameter

Description

Name

A unique name.

Description of the configuration

Non mandatory description of the device configuration.

Directly addressable

If the device cannot communicate directly with the communication server, e.g. in the case of a slave device, this option is disabled

Device is as a gateway

If the device doesn’t have gateway functionalities this option is disabled.

Type of gateway

Select the type of gateway:

oHome Area Network gateway: connects to a small number of slave devices in a home environment.

oLocal Area Network gateway: connects to a large number of slave device in a local (larger) environment.

Dependent of the type of gateway that you select the look & feel of the landing page will be different.

Data logger functionality

Select if the device can act as a data logger and share its channels and registers with data logger slaves.

If you select this option you will be able to link slave devices or create new slave devices on this data logger. See “Link data logger slave to data logger” for more information.

This option will not be available for devices with the data logger slave device type.

Multi-element functionality

Select if you want to measure multiple loads, such as main electricity, boilers, solar energy and other, using a single physical device.

If you select this option you will be able to link slave devices or create new ones on this device per load.

Validate data on storage

Select if you want to validate the data from the moment it is available in the system.

If you select yes, then data will be validated when a device transitions from one state to another where the auto-action “activate validation” is selected.

If this option is active, then this is visible in the validation status on the device.

5.   Click Add.

6.   Click the actions icon  to activate the device configuration. Or click the actions button on the device configuration overview page.

7.   Add and configure register, load profile and logbook configurations.

3

Note

When the device configuration is activated it can be used to create devices. Note however that once a device uses a specific device configuration, this configuration can no longer be deactivated.

Edit device configuration

5

Steps

1.   Go to Administration > Device management > Device type > select a device type > Configurations > Configurations.

2.   Click the actions icon  and select Edit.

3.   Change the device configuration parameters. If the configuration is active, then only the name and description can be altered.

4.   Click Save.

Remove device configuration

5

Steps

1.   Go to Administration > Device management > Device type > select a device type > Configurations > Configurations.

2.   Click the actions icon  and select Remove.

3.   Click Remove. Active configurations can not be removed.

Clone device configurations

5

Steps

1.   Go to Administration > Device management > Device type > click a device type name > Configurations.

2.   Click the actions icon  next to the configurations you want to clone and select Clone.

3.   Give a unique name to the new device configuration and click Clone.

The cloned device configuration has all the same features as the original device configuration, but its status is Inactive.
