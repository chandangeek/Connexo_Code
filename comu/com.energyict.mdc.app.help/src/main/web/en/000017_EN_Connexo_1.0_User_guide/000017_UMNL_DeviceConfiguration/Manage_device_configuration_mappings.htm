Manage device configuration mappings

The meter operator can define what happens with conflicting connection methods and security settings for all possible device configurations on a device type.

Generate conflicting device mappings

There are five different cases that can create conflicting device mappings, these are the following:

  • When you activate a device configuration.

  • When you add new security settings to an active device configuration.

  • When you edit security settings on an active device configuration.

  • When you add a new connection method on an active device configuration.

  • When you edit a connection method on an active device configuration.

In these cases MultiSense  will check if the connection methods and security settings of the current configuration conflict when mapping to all other device configurations configured on the device type and when you map all the other configuration to this configuration.

Figure: Use case when connection methods/security settings from the different configurations do not have conflicting attributes. In this case connection method A is mapped to connection method A on configuration B and connection method A’ is added.

 
Figure: Use case when two configurations contain connection methods/security settings that have conflicting attributes (for example different name). In this case the user will have to select which scenario will be executed when changing the device configuration. He can choose to map connection method A to connection method A’ or A’’ thereby using the attribute values of connection method A and adding a new connection method A’’ or A’. The third option is to remove the connection method from configuration A and adding the connection methods A’ and A’’.

 

View conflicting device mappings

5

Steps

1.   Go to Administration > Device management > Device types > select a device type > Configurations > Conflicting mappings.

The menu item Conflicting mappings shows the total amount of conflicting mappings.

By default all unsolved conflicting mappings are shown in a table.

Solve conflicting device mappings

5

Steps

1.   Go to Administration > Device management > Device types > select a device type > Configurations > Conflicting mappings.

2.   Click the actions icon  next to the conflicting mapping and select Solve.

This screen shows all conflicting connection methods and security settings when you go from one configuration (A) to another (B). MultiSense  offers two options to solve the conflicts:

Option

Description

Remove

Remove the connection method or security settings from the original device configuration. The connection method or security settings from the second configuration will be added.

When you select to remove the connection method/security settings, all these attributes will be removed from the device.

Map

The second option is to map the connection method/security settings from configuration A to configuration B. However this is only possible for connection methods that have the same connection type and security settings that have the same authentication and encryption level.

The attributes of the connection method/security setting of configuration A will be mapped to those of configuration B.

Only one connection method/security setting can be mapped to one connection method/security setting.

3.   Select a mapping option for every connection method and security setting.

4.   Click Save.

Edit conflicting device mappings

5

Steps

1.   Go to Administration > Device management > Device types > select a device type > Configurations > Conflicting mappings.

2.   Click All mappings in the upper right corner to see all mappings (unsolved + solved).

3.   Click the actions icon  and select Edit.

4.   Change the conflicting mappings solution.

5.   Click Save.
