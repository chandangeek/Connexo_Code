Configure validation rules

Add validation rule sets

5

Steps

1.   Go to Administration > Validation > Validation rule sets.

2.   Click Add validation rule set to create a new set. Complete the following fields:

Parameter

Description

Name

A unique name.

Description

A description of the rule set.

3.    Click Add.

Edit validation rule sets

5

Steps

1.   Go to Administration > Validation > Validation rule sets.

2.   Click the actions icon  and select Edit.

3.   Change the validation rule set parameters.

4.   Click Save.

Remove validation rule sets

5

Steps

1.   Go to Administration > Validation > Validation rule sets.

2.   Click the actions icon  and select Remove.

3.   Click Remove.

Add validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > click Validation rules in the menu on the left.

2.   Click Add validation rule. Fill in the following fields:

Parameter

Description

Name

A unique name

Validator

Select how to perform validation. Depending on the validator selected, extra parameters may need to be defined.

Reading type(s)

Select the data that will be validated. Multiple reading types can be added as described in “Add reading type”.

Data quality level

Suspect: the data is invalid and needs attention. This value will be subject to estimation. These values are indicated in red on the graphical representation of the data.

Informative: the data is valid, but could be an indication that something is going wrong. This value will not be estimated. These values are indicated in yellow on the graphical representation of the data.

3.   Click Add.

4.   Click the actions icon to activate or inactivate the rule.

3

Note

The following validators are available by default:

Validator

Description

Check missing values

Check for missing data. This data can be provided in a later stage by estimation and editing.

Reading quality

Reading qualities are comments added to values to indicate that something occurred at the time of that reading, for example a power outage.

The reading quality validator checks interval data and register readings if they contain one or more of the specified reading qualities. Note that for the system and index code of a reading qualities you can use the wildcard “*”. In this case, a reading with a reading quality with the correct category will be marked as suspect, regardless of its system or index code.

Register increase

Check consumption over time. If the value of the register is lower or equal to the previous register value then this data is marked as suspicious.

Check Fail equal data if you want to mark equal values as suspicious.

Threshold violation

Checks each interval value against a minimum and maximum value. Values outside these boundaries are marked as suspicious.

Edit validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > Validation rules.

2.   Click the actions icon  and select Edit.

3.   Change the validation rule parameters.

4.   Click Save.

Remove validation rules

5

Steps

1.   Go to Administration > Validation > Validation rule sets > click a validation rule set > Validation rules.

2.   Click the actions icon  and select Edit.

3.   Click Remove.

Add reading type

5

Steps

1.   Click Add reading types.

2.   Complete the filter fields to find the correct reading types.

3.   Click Apply.

Deselect items in the drop down menu to remove single filters or click Clear all to remove all applied filters.

4.   Select one or multiple reading types from the table.

You can see the reading type details by clicking the information icon .

5.   Click Add.
