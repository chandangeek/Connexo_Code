Default device life cycle states

Default device states

MultiSense offers different default states that cover all the basic device states needed to operate the system. Depending on the state you will be able to view data and/or perform specific actions. You can use these states to search for devices. All the default states are described below.

In stock

A device will be assigned the state In stock when the device has been installed on its location, but nothing has been configured yet. If a device is in stock you cannot activate validation, estimation and issue management but you are able to connect to this device. Devices in stock are not taken into account when calculating KPI’s except when indicated otherwise.

Commissioning

This is a state where a small batch of devices can be tested to monitor their performance. A device with this state is able to communicate with Connexo. Furthermore you can validate, estimate and edit its data. Typically you will keep devices for a limited period of time in this state.

Active

This is the state the device will reside in most of its time. When a device resides in this state it is installed and fully functional, meaning that the device will collect data and you are able to validate, estimate and edit this data. You need to define an usage point before you can transition to the active state.

Inactive

A device can be deactivated when there is no need for its use. The inactive device is still able to communicate and if needed data can be validated. Validation can be activated to detect possible theft. No data is removed and the device can simply be reactivated to be used again.

Decommissioned

This is the state when a device is taken down from its place of installation and will no longer be used. All data will remain available on the device in Connexo but communication with the physical device is no longer possible. When you decommission a device you are still able to view everything on this device, but you cannot edit the data. Decommissioned devices are not taken into account when calculating KPI’s except when indicated otherwise.

Removed

The state Removed indicates that the device and all its data has been removed from the database. You can remove a device if there was an error while creating the device or when the SOX compliant period has ended. Once you remove a device all its data is irreversibly removed.
