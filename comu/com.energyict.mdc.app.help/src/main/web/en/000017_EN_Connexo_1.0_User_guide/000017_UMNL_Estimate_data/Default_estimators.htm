Default estimators

Estimate using average with samples

1

Important

You need delta values for this estimation rule.

With this rule every suspect channel delta value is estimated by calculating the average of reading values from this channel or other reading types from the same time and day in the past. For this rule you need to configure the following parameters:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated with this rule, but can be estimated with the next estimation rule in the set or manually edited or estimated.

Allow negative values

Indicates that negative values can be used to estimate data.

Use advance readings

Select this when you want to scale the estimated values using the values that are adjacent to the estimation interval. You can choose to use the channel’s own values or use another reading type.

o   Bulk reading: use the channel’s bulk reading for estimation.

  • Reading type: select another reading type for estimation. This reading type needs to have readings at exactly the same day and time as the suspect data and the same type of data.

Minimum samples

The minimum amount of sample needed for estimation.

Maximum samples

The maximum amount of samples needed for estimation.

Relative period

Select the period in which Connexo will search for sample.

o   All: search in all past readings.

  • Period: select a relative period, as configured in Admin. This period will define how many samples you will find and be able to use for estimation.

Estimation with equal distribution

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule.

To estimate delta values using the equal distribution rule MultiSense will calculate the difference between the bulk or advance reading after the estimation interval and the bulk or advance reading before the estimation interval. It will then divide this difference by the amount of samples in the interval plus one. MultiSense will then use this value as delta value for every suspect value in the interval. For this rule you need to configure the following parameters:

Estimate with linear interpolation

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule. This rule only works with bulk readings.

With this rule MultiSense will estimate the suspect bulk readings. The delta values are then calculated based on these bulk readings (the difference between two consecutive bulk readings). To estimate the bulk readings MultiSense calculates the difference between the bulk reading after the estimation interval and the bulk reading before the interval and divides this difference by the amount of samples in the interval plus one.

In the second step MultiSense adds this difference to the bulk reading before the estimation interval to calculate the first suspect reading. The second estimated suspect reading will be the previous estimated reading plus the calculated difference from the first step. MultiSense repeats these steps until all suspect readings are estimated.

You need to configure the following parameters for this rule:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be estimated with the next estimation rule in the set or manually edited or estimated.

Estimate with power gap fill

1

Important

Bulk readings are required to be able to estimate suspect values with this estimation rule.

Using the power gap fill rule MultiSense changes every suspect reading in the estimation interval except the last reading by zero. The last equals the difference between the bulk reading after the interval and the bulk reading preceding the interval. The following parameters need to be configured:

Parameter

Description

Reading types

The data that will be estimated.

Max number of consecutive suspects

The maximum number of consecutive suspects that is allowed. If this amount is exceeded data is not estimated, but can be estimated with the next estimation rule in the set or manually edited or estimated.
