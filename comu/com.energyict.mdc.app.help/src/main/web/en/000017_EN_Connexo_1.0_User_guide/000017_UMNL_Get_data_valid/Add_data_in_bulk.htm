Import readings

1

Important

This importer only allows the import of daily readings.

To be able to import device readings, you need a CSV file, the first line of this file has to be the header and fields are separated using  "," or ";". The following information has to be included in this file:

Field

Required

Data format

Device identifier

Yes

String

Reading date

Yes

Date. As defined in import service. Default format is “dd/MM/yyyy HH:mm”

Reading type

Yes

String

Value

Yes

Number. Should match the configuration settings (overflow, number of digits, number of fraction digits).

Device identifier can be the device MRID or name. The importer will first check for MRID and if it cannot retrieve this it will search for devices with matching names.

It is possible to add different readings from the same device, create different rows with the same Name, but with different values. All values should match the configuration settings, being the overflow defined as well as the number of digits and number of fraction digits.

Example:

  • Device identifier;Reading date;Reading type MRID;Reading value;;;;
    SPE010000010080;25/06/2015  12:00:00;0.0.0.1.1.1.12.0.0.0.0.0.0.0.0.3.72.0;1,335,010,534.12;;;;
    SPE010000010080;25/06/2015  12:00:00;0.0.0.1.19.1.12.0.0.0.0.0.0.0.0.3.72.0;497,899,785;;;;
    SPE010000010081;25/06/2015  12:00:00;0.0.0.1.1.1.12.0.0.0.0.0.0.0.0.3.72.0;1,335,010,542;0.0.0.1.19.1.12.0.0.0.0.0.0.0.0.3.72.0;1,335,010,542;0.0.0.1.1.1.12.0.0.0.0.0.0.0.0.0.72.0;497,899,785

5

Steps

1.   Put the CSV file containing the readings in the import folder.

2.   Once import is finished successfully the original CSV is moved to the success folder. The defined values are now add to the devices in MultiSense .

Add channel readings to device without readings

In MultiSense  you can manually add readings to a device that has never been read. To be able to do this you need to follow the actions described below. This action is also available for registers, but here you will have to add one value at a time.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Channels in the menu on the left.

3.   Select a channel.

4.   In the filter on top of the page select Today as measurement time. Click Apply.

An info message appears showing that there a no readings available.

5.   Click Add reading.

6.   Click the calculated value in the table and type the value or copy/paste multiple values from another source.

7.   Click Save changes.

Copy readings from reference

In MultiSense  you have the option to copy the readings of a different device with the same or a different reading type.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Channels in the menu on the left.

3.   Click the Bulk action button and select Copy from reference.

4.   Complete the following fields:

Parameter

Description

Check device

Select the device to copy the data from.

Check reading type

Select the reading type of the data to copy.

Start date/time

Select which date/time to start the interval of data to copy.

Allow suspect data

When checked also suspect data will be copied.

Complete period

When checked no data will be copied if there are missing values in the collected data of the check device in the selected period.

Estimation comment

Select the comment to add to the reading. By default “Keep original comment” is selected. You can remove this by click the button next to the drop down menu.

5.   Click Copy.

6.   Click Save changes.
