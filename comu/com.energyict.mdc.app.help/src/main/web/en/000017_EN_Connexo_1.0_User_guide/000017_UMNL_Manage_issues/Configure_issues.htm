Understand issue creation rules

MultiSense offers the possibility the configure issue creation rules. This option limits the time you spend managing issues because you create rules to define which events will automatically trigger an issue. You can also define when the issue needs to be solved as well as what needs to be done with the issue such as assigning it to a specific user.

Different parameters are required to complete the configuration of a creation rule. First you will have to indicate the type of issue, this can be for example a data collection issue. Next you need to select a rule template to define the parameters of the rule.

A rule template is a collection of parameters that are specific for a certain rule. The exact values of the parameters are defined in the rule itself. An example of such a template is the template to create an issue when a slope is detected. All parameters will be shown, but then you will have to complete the reading type, the trend period and the threshold.

 

Available rule templates

In MultiSense different rule templates are available by default, below you can find a description of these rule templates and when to use them:

Rule template

Description

Data collection issues

 

Events from meters of concentrators

This rule template is designed to create an issue when multiple events that are related occur.

Create issue when specific event occurs

Is a rule template designed to create issues for simple events such as a lost connection or a communication failure.

Data validation issues

 

Create issue when suspects cannot be estimated

An issue is created when the data cannot be estimated by the estimation rules used by the configured estimation task.

Add data collection issue creation rules

5

Steps

1.   Go to Administration > Issue management > Issue creation rules.

2.   Click Add rule. Complete the following fields:

Parameter

Description

Name

A unique name

Issue type

Select Data collection

Rule template

Select the template containing all parameters to configure the rule.

Auto resolution

Close issues automatically when the connection or communication problem is solved.

Note that this option is only available for specific rule templates.

Issue reason

What is the cause of the issue

Due date

When does the issue need to be solved.

Comment

Add additional information to the issue.

Actions

Define what needs to be done when the issue is created. See “Add actions to issue creation rules” how to configure these actions.

Add actions to issue creation rules

5

Steps

1.   Go to Administration > Issue management > Issue creation rules.

2.   To add actions to a new creation rule: click Create rule.

To add actions to an existing creation rule: click the actions icon  and select Edit.

3.   Click Add action.

4.   Define when to perform the action: when the issue was created or when the issue has passed its due date.

5.   Select action: assign a user, send a notification or send someone to inspect the issue.

6.   Click Add.

Edit issue creation rules

5

Steps

1.   Go to Administration > Issue management > Issue creation rules.

2.   Click the actions icon  and select Edit.

3.   Change the issue creation rule parameters.

4.   Click Save.

Remove issue creation rules

5

Steps

1.   Go to Administration > Issue management > Issue creation rules.

2.   Click the actions icon  and select Remove.

3.   Click Remove.
