Issue management

It is often important that users are notified when something goes wrong during data collection and validation. Instead of having to manually verify all events that originate from different tasks, Connexo MultiSense  offers the possibility to configure rules to create issues when certain events occur that will warn the user or trigger a specific workflow.

This chapter covers all aspects of issue management ranging from configuring issue rules to fixing issues.

Connexo MultiSense  offers the possibility to easily handle and manage problems that can occur during the data collection process. This is achieved trough the generation of issues. You can configure when issues are created, how they are handled and even choose to close issues automatically.

Handle data collection issues

1. First Add data collection issue creation rules so that an issue is created when there is a connection failure.

2. Check whether if there are issues, there are different options to view issues:

     a. Go to All data will be exported to a CSV file. to see the most urgent issues that are assigned to you

     b. Check the meter/gateway landing page.

     c. Go to Workspace > Issues.

3. Click the issue name for more information.

4. In case of a connection error view the Connection history to identify the problem.

5. Configure connection methods on a device to solve the connection problem.

6. Fix issues by retrying the connection.
