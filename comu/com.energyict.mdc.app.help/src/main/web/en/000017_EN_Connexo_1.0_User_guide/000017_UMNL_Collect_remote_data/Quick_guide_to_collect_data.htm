Quick guide to collect data

Introduction

Connexo MultiSense is developed in a such way that it allows a logical workflow to go from setting up and configuring devices to finally collecting the data and analysing the data collection process together with the collected data. Many properties are defined at a higher level, called device types. This allows you to reuse settings for multiple devices. Device specific features are then specified in the configuration of the device itself.

 Collect remote data workflow

 

The first step is to configure the master data, this will form the basis for creating device types. A device type defines the physical capabilities of the device, which data can be read and what communication protocol has to be used.

 

In the next step the communication infrastructure is set up, allowing connection and communication with physical devices. Configure communication servers and ports as well as tasks such as reading registers and the schedules when to perform these tasks.

 

Next the device type is configured. This starts by adding master data to the device type. Then multiple configurations can be added to one device type. A configuration is used to customise the functionality according to a specific device.

 

Then the device itself is added. This can be done manually or be imported from a CSV file. To complete the configuration of these devices, device specific parameters will need to be specified (e.g. connection methods, protocol dialects).

 

Additionally configure rules to validate and create issues for data collection the collected data and the collection process. Validation is done to check the quality of the collection process and the collected data. Issues are used to monitor processes, record what went wrong and solve the problem.
