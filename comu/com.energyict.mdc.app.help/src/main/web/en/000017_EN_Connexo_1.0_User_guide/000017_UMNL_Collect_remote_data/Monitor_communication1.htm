Monitor communication

Communication overview

5

Steps

1.   Go to Workspace > Data communication.

2.   Click Communication overview.

The connections overview page is divided in four parts showing different types of connection related data. In the upper right conner the time stamp is displayed as well as the option the refresh the view (E). You have the option to filter the results on a certain device group (A), see “View communication overview per device group” for more information.

More information on the displayed data is given below.

Section

Options

Description

Summary

•   Communication summary (B)

Summary of all communication, showing the amount of successful communication tasks, ongoing and failed communication tasks. Also shows the KPI when you select a device group.

•   Communication servers (C)

Shows an overview of all communication servers and which are down and could be causing communication problems.

Server status is shown when hovering over the server.

•   Quick links (D)

Links to view all communications and the connection overview and different reports.

Communication over time (F)

•   Device groups

A graphical representation of the successful, ongoing and failed communications tasks relative to the defined target (KPI).

Communication tasks can be omitted or added to the graph by clicking the communication task group above the graph.

Overview (G)

•   Per current state

Detailed overview of the current status of all communications.

•   Per latest result

Detailed overview of the last status of all communications.

Breakdown (H)

•   Per communication schedule

Breakdown of the number of failed, successful and pending communications per communication schedule, communication task and device type. This allows identification of problems related to a specific component.

Hovering over the bar shows the amount of pending, successful and failed communications.

•   Per communication task

•   Per device type

Heatmap (I)

•   Device type

Allows you to see the correlation between the latest result of a communication and a device type.

Horizontal values represent the latest results while the vertical values are the different device types available in Connexo MultiSense.

Extra information is displayed when hovering over the different areas of the heatmap.
