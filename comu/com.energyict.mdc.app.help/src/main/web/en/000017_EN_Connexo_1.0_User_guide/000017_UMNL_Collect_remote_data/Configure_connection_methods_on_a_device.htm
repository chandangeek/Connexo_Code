Configure connection methods on a device

Understand connection methods on a device

Device connection methods are inherited from the device configuration. All parameters and connection details are the same, however in Connexo MultiSense it is possible to change these settings on a device level. This can be interesting when only a couple of devices have connection details that differ from the device configuration.

Configure an outbound connection method

1

Important

All parameters of the device connection method are inherited from the device configuration. Make sure that connection methods are configured in the device configuration.

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Connection methods in the menu on the left.

3.   Edit or remove connection methods via the actions icon .

4.   Click Add outbound connection method or Add connection method > Add outbound connection method. Complete following fields:

Parameter

Description

Configuration connection method

Device configurations available on to the used device type. Can only be used once per device. After saving, this parameters can no longer be changed.

Communication port pool

Select communication port pools available on the selected configuration.

Connection strategy

As soon as possible: connect whenever a communication task needs a connection.

Minimize connections: only connect with the device at the scheduled time you specified.

Connection schedule

Only available when “Minimize connections” is selected. The schedule when to set up a connection.

Connection window

Is the time window during which it is allowed to set up a connection and communication tasks can be executed.

Allow simultaneous connection

Allows multiple connections at the same time. Not possible when “Minimize connections” is selected.

Active

Activate the connection method.

5.   Complete or change inherited connection details.

6.   Click Add.

7.   Click the actions icon to activate the connection method and set as default.

3

Note

It is possible to save an incomplete connection method when not all acquired fields are filled in. In this case a message is shown to ask whether you are certain to save the incomplete connection method. The connection method will appear in the overview table, but will have the status incomplete.
