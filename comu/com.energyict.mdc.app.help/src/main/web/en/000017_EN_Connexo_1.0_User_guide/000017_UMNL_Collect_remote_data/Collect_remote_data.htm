Collect remote data

Overview

With Connexo MultiSense you can easily add and configure multiple devices while limiting the amount of parameters that need configuration. This chapter covers the features to schedule data collection and collect it on request. We will also explain how to monitor your collected data.

What would you like to do?

SET up communication infrastructure

1. Follow the Quick guide to collect data.

2. Understand communication in Connexo MultiSense.

3. Edit communication protocols if necessary.

4. Configure communication servers

5. Configure communication ports of the server.

6. Configure communication port pools.

7. Configure communication tasks (Administration) to define the tasks that need be ex­ecuted.

Solve connection failures

In MultiSense there are different options to detect and monitor connection failures. The first option is to use the dashboard.

1. Go to the All data will be exported to a CSV file. to monitor connection failures.

2. Go to Connection overview to see more details.

3. Filter connections to only see the failed connections.

4. Retry connection to solve the failure.

5. Go back to the Connection overview to see the result.

 

A second option to detect problems with connections is to monitor issues that are creat­ed when connection failures occur. For more information see Issue management.

 

The third option is to monitor the connection of a specific device via the meter/gateway landing page of that device

1. Search for devices to find the device.

2. On the bottom of the overview page of that device all connections are shown. There is a possibility to perform different actions here: Run now, (De)Activate and View his­tory.

Disconnect/Connect a meter

First you need to configure the device command:

1. Add communication tasks and add the contractor command category to the commu­nication task.

2. Add communication task configuration on device configuration to be able to add commands.

3. Change privileges of commands and (De)Activate commands so that these can be used on a device.

 

In the next steps you will add the command to the device and trigger its communication:

1. Search for devices to find to the device that needs to be (dis)connected.

2. Add commands to a device and select the open or close command from the contactor category.

3. Trigger communication for a command or wait until the communication task is picked up by a connection.

Define view/edit rights to specific security settings and commands

View and edit right are determined by privileges. All privileges in Connexo MultiSense are configured in the system administration application Admin.

In the MultiSense application you can ensure that only specific user can access sensitive data or perform dangerous actions. This is accomplished when you Add/remove privileg­es to security settings and Change privileges of commands.
