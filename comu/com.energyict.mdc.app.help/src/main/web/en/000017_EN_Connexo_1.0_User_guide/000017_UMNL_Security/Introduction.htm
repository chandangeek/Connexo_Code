Understand security in MultiSense

Why do you need security

In order to communicate, a trust chain is needed to validate a certificate's user. The trust chain of a certificate chain is an ordered list of certificates, containing an end-user subscriber certificate and intermediate certificates. This enables the receiver to verify that the sender and all intermediates certificates are trustworthy. If the trust chain is not complete, communication will be denied.

Keys are used to encrypt and decrypt content. Depending on the type of security different key types are available.

Certificates and trust stores are managed in Admin  (see chapter 7 Security in the Admin  user guide for more information). In MultiSense  trust stores and certificates are linked to device types. These items can then be further specified on protocol related features such as the connection method on the device configuration. On the device itself you can specify the key values.
