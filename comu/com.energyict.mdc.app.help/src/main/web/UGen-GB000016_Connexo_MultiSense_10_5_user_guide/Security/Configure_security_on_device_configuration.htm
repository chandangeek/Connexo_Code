Configure security on device configuration

Understand security on a device configuration

Depending on the protocol used by the device type, some general attributes, connection methods and security sets require keys or certificates. These attributes know what type of key/certificate they need and this is why you can choose from a list of security accessors available on the device type with the same type.

The actual values are determined on the device level.

Add security sets

5

Steps

1.   Go to Administration > Device management > Device type > Select a device type > Configurations > select a configuration > Communication > Security sets.

2.   Click Add security set. Complete the following fields:

Parameter

Description

Name

A unique name.

Client

Select one of the clients available on the meter. This option is only available if the protocol supports this.

Security suite

Select the DLMS security suite.

Only available if the protocol supports this.

Authentication level

The options are dependent of the device protocol used.

Encryption level

The options are dependent of the device protocol used.

Request security level

Select a level of security for sending a request.

The availability of this option depends on the selected security suite.

Response security level

Select a level of security for the response.

The availability of this option depends on the selected security suite.

Extra parameters

Extra security parameters may need to be specified depending on the used protocol, the selected authentication and encryption level and the response and request security level. This could for example be the selection of the security accessor to define the authentication and encryption keys.

3.   Click Add.

Edit security set

5

Steps

1.   Go to Administration > Device management > Device type > select a device type > Configurations > select a configuration > Communication > Security sets.

2.   Click the actions icon  and select Edit.

3.   Change the security setting parameters.

4.   Click Save.

Remove security sets

5

Steps

1.   Go to Administration > Device management > Device type > select a device type > Configurations > select a configuration > Communication > Security sets.

2.   Click the actions icon  and select Remove.

3.   Confirm by clicking Remove.

Configure security accessors on connection method

Depending on the communication protocol used some security accessors may need to be specified in the connection details of the connection. See “Add outbound connection methods” and “Add inbound connection methods” for more information.

Configure security accessors on communication protocol attributes

Depending on the communication protocol used different security accessors may need to be specified in the general attributes of the device configuration. See “Edit general attributes on a device configuration” for more information.
