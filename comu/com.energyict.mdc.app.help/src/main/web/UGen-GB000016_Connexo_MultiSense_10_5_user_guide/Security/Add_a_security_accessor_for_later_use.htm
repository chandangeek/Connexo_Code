Add a security accessor for later use

You can add a new security key or certificate type to use it in a security setting, connection method or as a general attribute.

5

Steps

1.   Go to Administration > Security > Security accessors.

2.   Click the button Add security accessor.

3.   On the Add security accessor page, add the following fields:

Parameter

Description

Name

Type a unique name for the security accessor.

Description

Type an optional description to provide some context about the accessor for later reference.

Accessor type

Select the type of accessor. It can be a key or a certificate. Your selection defines the other parameters that need to be specified.

If you select the accessor type key, then:

Key type

Select one of the key types that are available in Connexo.

Storage method

Select one of the available storage methods to store the keys securely. If the storage method is Wordline ESS, then the field Label is shown.

 

Note

Wordline ESS is an optional energy security solution module that manages the hardware security module (HSM) back-end servers that are linked with the Worldline ESS. Connexo will connect with this ESS using the value in the field Label.

Label

This mandatory text field stores the HSM key label that is associated with the current security accessor. This label refers to the profile in the ESS that determines the activities that can be done with the concerned key. Each security accessor has one label that is referring to one profile. Multiple security accessors can use the same label. Security accessors (and corresponding labels) are used in firmware verification, protocol management, CSR importers, shipment files, etc. This list isn’t limitative. The value isn’t validated towards the HSM.

 

Note

On the device you can see the Label field and overrule it if needed. See “Edit security accessors on device”.

Validity period

Define how long the key is valid.

If you select the accessor type certificate

Certificate type

Select one of the certificate types that are available in Connexo.

Trust store

Select one of the trust stores that have been defined in Connexo Admin. Connexo uses this trust store to validate the trust chain of the certificate.

Storage method

Only available if the selected certificate type is a client certificate type. Select one of the available storage methods.

Manage centrally

Check to manage the accessor centrally and specify active and passive certificate on the administration level.

Active certificate

Only shown if Manage centrally is checked.Select one of the available certificates. The list depends on the selected certificate type, trust store and storage method.

Passive certificate

Only shown if Manage centrally is checked. Select one of the available certificates. The list depends on the selected certificate type, trust store and storage method. This field is optional.

4.   Click Add.

3

Note

If the accessor type is a key, then it will get level 1 and 2 view and edit privileges by default. To change these see “Change the privileges of a key security accessor” for more information.
