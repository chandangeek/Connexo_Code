Understand communication

How do devices communicate with the data collection system? In Connexo MultiSense communication from and to a device goes trough a communication port pool. Communication from a device to Connexo MultiSense uses an inbound communication port pool while communication to a device uses an outbound port pool. A communication port pool consists out of different communication ports. These ports can belong to different communication servers to allow load balancing and redundancy.

To be able to communicate, Connexo MultiSense supports multiple communication servers. Each communication server controls different communication ports. A communication port can be a physical modem or a logical connection. A communication server can collect data, write data to and perform actions on devices. This is done using communication tasks you configure. These are then assigned to a communication port pool to be executed. To be able to perform communication tasks a connection is needed. The connection will allow communication between Connexo and the device while the communication task defines what happens during this connection.

You can have multiple communication servers. This allows to divide the communication tasks. Because communication port pools consist out of communication ports belonging to different servers this guarantees that another port of the same pool can take over if an outage occurs. If an outage of the communication server occurs than another server, that is part of the same port pool, will take over.
