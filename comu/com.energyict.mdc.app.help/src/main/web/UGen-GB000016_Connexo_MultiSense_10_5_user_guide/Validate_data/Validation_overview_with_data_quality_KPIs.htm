Validation overview with data quality KPIs

To see the validation results on all devices of a specific device group Connexo MultiSense uses data quality KPIs. These KPIs periodically calculate the amount of suspects of a device group as defined in their configuration. The actual calculation formula of the data quality KPIs is determined by the Connexo MultiSense software but you can configure the device group on which the KPI should be calculated as well as the calculation frequency. The calculated results of the data quality KPIs can be seen on Workspace > Data validation > Data quality.

View the available data quality KPIs

You can see a complete overview of all data quality KPIs that are currently available in Connexo MultiSense.

5

Steps

1.   Go to Administration > KPIs > Data quality KPIs.

The table shows all configured data quality KPIs by the device group they apply to. You also see how often the data quality KPI is calculated and when it was last calculated.

Add a data quality KPI

You can add a data quality KPI to see the data quality of devices in a particular device group.

5

Steps

1.   Go to Administration > KPIs > Data quality KPIs.

2.   Click Add data quality KPI.

3.   Complete the following fields:

Parameter

Description

Device group

Select the device group that contains the devices for which the data quality KPI must be calculated. You can only create one data quality KPI per device group. Device groups that are already associated with a data quality KPI aren’t listed in the drop-down list.

Calculation frequency

Select how often the KPI must be recalculated.

4.   Click Add.

The new data quality KPI is now listed in the overview table.

Remove a data quality KPI

You can remove a data quality KPI if it is no longer relevant.

5

Steps

1.   Go to Administration > KPIs > Data quality KPIs.

2.   Click the remove icon next to the data quality KPI that you want to remove.

3.   Click Remove to confirm.

The data quality KPI is no longer available from the overview table.

View the calculated results of the data quality KPI

The data quality KPI calculates and stores reading qualities to see the data quality in a group of devices. The calculation range is three months and the reading qualities are calculated every day.

The data quality KPI calculates the amounts of readings for each of the following reading qualities:

Flag

Description

 

Suspect readings that are generated using a particular validator.

 

Confirmed readings.

 

Estimated readings that are generated using a particular estimator.

 

Readings with the Informative reading quality.

 

Edited readings that were added, changed or removed.

The details panel under the table shows for each device a breakdown of the data quality information such as:

  • Number of suspect readings in registers and channels.

  • Number of suspect readings per validator.

  • Number of added, removed or changed meter readings.

  • Number of estimated readings per estimator.

 

5

Steps

1.   Go to Workspace > Data validation > Data quality.

2.   Use the filter bar on top to see the data quality KPI results that are relevant to you. Don’t forget to click Apply after you specified the filter criteria. The available device groups in this filter are those that have been defined in the configuration of the data quality KPI.

The table shows all devices that answer the entered filter criteria. For every device you can see its name, serial number, device type, configuration and the corresponding data quality KPI results.

The results in the overview table will be refreshed in line with the calculation frequency that is specified in the data quality KPI. See “Add a data quality KPI”.

3.   Click a device name to open it and edit the suspect values as described in “Edit data”.

4.   Click a device type name to open it and see its data sources and device configurations.
