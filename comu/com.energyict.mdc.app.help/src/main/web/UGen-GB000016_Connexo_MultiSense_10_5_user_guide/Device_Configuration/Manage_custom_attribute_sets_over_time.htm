Manage custom attribute sets over time

Understand custom attribute set versions

In Connexo MultiSense you have the option to add multiple versions to specific custom attribute sets. This will allow you to create a new version of the attribute set that will be active starting from a specific date with specific values whenever there is a change in the device, registers or channels. When adding a new version you are still able to see the previous versions together with the values used for that attribute sets during that period.

Privileges that were set in Admin define which users are allowed to add and edit version, as well as what they can see. If a custom attribute set is able to have different versions, is defined in the code of the attribute set.

View custom attribute set version on a device

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Device attributes in the menu on the left and then click Versions next to the custom attributes set or click History in the menu on the left and select the tab corresponding to the custom attribute set.

On the Device attribute page the values of the current version of the custom attribute sets is shown as well as the period the version is valid.

The history page of the custom device attribute shows all versions: the period they are active and the values they contain.

View custom attribute set version on the register of a device

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Registers in the menu on the left.

3.   Click a register name and go to the Specifications tab or select a register in table and click Versions.

On the Specifications tab or the details pane of the register overview page only the current version of the custom attribute set is shown.

The history page of the custom device attribute shows all versions: the period they are active and the values they contain.

View custom attribute set version on the channel of a device

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Click Channels in the menu on the left.

3.   Click a channel name and go to the Specifications tab or select a channel in table and click Versions.

On the Specifications tab or the details pane of the channel overview page only the current version of the custom attribute set is shown.

The history page of the custom device attribute shows all versions: the period they are active and the values they contain.

Add custom attribute set versions

5

Steps

1.   Select a device using Search, see “Advanced search”.

2.   Go to the custom attributes of the device, register or channel as described in “View custom attribute set version on a device”, “View custom attribute set version on the register of a device” and “View custom attribute set version on the channel of a device”.

3.   Click Add version and complete the following fields:

Parameter

Description

Start

Start date of the version, from this day and time the version will become active.

You can also choose not to specify a specific date.

End

End date of the version. The version will be active until this day and time.

You can also choose not to specify a specific date.

Custom attributes

Enter the custom attribute values for this version.

4.   Click Add.

If the selected time window introduces gaps or overlaps with the other configured versions, then an error message will be shown. Close the message.

A new table will appear showing the conflicts and some additional information. You can change the date and time to remove the error or click the actions icon and select Align left or Align right. Which options you see are dependent of the versions available and the date selected.

Options

Description

Align left

Start date of the new version will be set equal to the end date of the previous version.

Align right

The end date of the new version will be set equal to the start date of the next version.

You can also choose not the fix the error yourself but click Add and then the time and time will be adjusted automatically to remove any inconsistencies.

Clone custom attribute set version

5

Steps

1.   Go to the custom attribute versions page.

2.   Click the actions icon  and select Clone. Depending on your privileges this action may not be possible.

3.   Edit the custom attribute parameters.

4.   Click Clone.

If there are conflicts in the configured time window of the cloned version, these can be handled as described in step 4 of “Add custom attribute set versions”.

Edit custom attribute set version

5

Steps

1.   Go to the custom attribute versions page.

2.   Click the actions icon  and select Edit. Depending on your privileges this action may not be possible.

If there are conflicts in the configured time window of the cloned version, these can be handled as described in step 4 of “Add custom attribute set versions”.
