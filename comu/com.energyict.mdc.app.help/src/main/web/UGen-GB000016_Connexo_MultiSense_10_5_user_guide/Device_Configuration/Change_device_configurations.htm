Understand device configuration management

When to change device configurations

In some cases you might wish to change the device configuration of one or more devices, for example:

  • when you install a device but you do not know which contract will be used or what type of user will use the meter

  • upgrade of a customer

  • change of functionality

If you change the configuration of a device, you need to be certain there are no conflicts between the original configuration and the new configuration.

How to change device configurations in Connexo MultiSense

In Connexo MultiSense changing a device configuration is managed on two different levels: on the device type and on the device. The actual changing of a device configuration is executed on a device level. On the device type you manage what happens with conflicts when you change one configuration into another.

When a device type has multiple active configurations Connexo MultiSense will check if there are conflicts in all possible options for changing device configurations. These conflicts can then be solved allowing the change of one configuration to another on a device to go without errors.

When mapping two configurations Connexo MultiSense maps every data source, communication feature, rule set from the original configuration to the new configuration. This mapping will not cause any conflicts because it is not possible to make two identical registers or communication tasks. The following cases are possible and will be handled as such:

Use case

Solution

Data source / Communication feature / rule set is available in both configurations

The data source / communication feature / rule set from the original configuration will be mapped to the new configuration together with all its attributes.

Data source / Communication feature / rule set is only available in the original configurations

The data source / communication feature / rule set from the original configuration will be removed together with all its attributes. Consequently all these values will be lost.

Data source / Communication feature / rule set is only available in the new configurations

The data source / communication feature / rule set will be added and the attribute values will remain empty.
Figure: Use case when registers and other data sources are mapped from one configuration to another.

 

The connection methods and security settings of device configurations can cause conflicts. This can be explained as follows, device configurations can have connection methods with the same connection type or security settings with the same authentication and encryption levels. However the attributes such as port number, host and passwords can be different. What will you do with these attributes when changing device configurations? This is what you will configure on the device type, you will define what happens in case conflicts between configurations arise. It is also possible that the connection methods in the two device configurations use the same connection type, but the connection methods have different names.

In Connexo MultiSense you are able to change device configurations of a single device or of multiple devices using the device search.
