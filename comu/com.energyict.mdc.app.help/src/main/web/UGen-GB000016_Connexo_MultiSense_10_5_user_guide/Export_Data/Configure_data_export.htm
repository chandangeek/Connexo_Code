Add export task for device readings

1

Important

First configure Device groups as described in “Manage device groups”.

5

Steps

1.   Go to Administration > Data exchange > Data export tasks.

2.   Click Add export task.

3.   Give a unique name to the export task and define the Log level of the executed export tasks. Options are Error, Warning, Information, Debug and Trace. Where the level error only create loggings if errors occur and trace gives a detailed description of all executions. Warning is the default log level.

4.   Complete the other fields that are divided in different sections:

Schedule:

Parameter

Description

Recurrence

Indicate the frequency of the task. Select None when you only want to run the task ad hoc.

Start on

Define the start date of the task.

Data selection

Parameter

Description

Data selector

Select the Device readings data selector.

Device group

Select the device group to export data of. Click Add device group to add a new group.

Reading types

Add one or more reading types as described in “Add reading type” to indicate which data will be exported.

Export window

Select one of the predefined export periods. The possible options are the relative periods configured in Admin.

Indicate whether to export data starting from the export period or starting from when the last export ended.

Updated data

Select this option if you want to export data that has been changed since a previous export.

Select the time window of the updated data and whether you want to export adjacent data as well.

Keep in mind to specify the update identifier to tag the updated data!

Missing data

Select to export data of which missing intervals are excluded (data with gaps) or choose not to export the export period when it contains missing data.

Validated data

Select to export all data, (Export all data (including suspect/not validated data)) only the valid data leaving out all suspects and not validated values in the export window (Skip intervals with suspect/not validated data) or choose not to export the export period when it contains suspect or not validated data (Skip export window for reading types with suspect/not validated data).

Output format

Different formatters may be available, this can affect which parameters to configure are displayed.

Parameter

Description

Data formatter

Select the formatter to define the format of the exported file.

Identifier

Custom tag to add to the file name of all exported files.

Update identifier

Custom tag to show that this concerns updated data. Define a value different from the identifier described above to create separate export files containing updated data.

Identical values for Identifier and Update identifier will result in updates being included in the regular export file.

Separator

Select which separator to use to separate the values in the exported file.

Destinations

a. Click Add destination. It is possible to add multiple destination to one data export task.

b. Depending on the Method you select, different extra parameters have to be defined..

Method

Parameters

Save file

  • File name: the name of the file. Add tags to give more information to the file. If you are exporting updated data add the tag <identifier>, then separate files will be created based on the Update identifier and Identifier specified in the output format parameters.

  • File extension: the extension of the file eg. CSV.

  • File location: specify the location to store the exported data. Tags can be added. If the <identifier> tag is added, updated data will be stored in a different folder with the name defined by the file location parameter and the Update identifier defined in the output format parameters.

Mail

  • To: email addresses of recipients, divided by semicolons.

  • Subject

  • Attachment name: name of the file. Add tags to give more information to the file. If you are exporting updated data add the tag <identifier>, then separate files will be created based on the Update identifier and Identifier specified in the output format parameters

  • Attachment extension: extension of the file.

FTP

  • FTP details: Details of the file server.

  • File name: Name of the file. Add tags to give more information to the file. If you are exporting updated data add the tag <identifier>, then separate files will be created based on the Update identifier and Identifier specified in the output format parameters.

  • File extension

  • File location: Location to store the exported data. Tags can be added. If the <identifier> tag is added, updated data will be stored in a different folder with the name defined by the file location parameter and the Update identifier defined in the output format parameters.

FTPS

Same parameters as FTP.

SFTP

  • Hostname

  • Port: Default port is 21.

  • User

  • Password

  • File name: Multiple tags can be used in the file name. They will be replaced during the export. The available tags are shown by hovering over the info iconnext to the field.

  • File extension

  • File location: Multiple tags can be used in the file location name. They will be replaced during the export. The available tags are shown by hovering over the info iconnext to the field.

c. Click Add.

5.   Click Add.

 

1

Important

The following characters can’t be used in file names, file extensions or file locations unless they are part of one of the available tags.

Output result

Each created file will contain the following fields:

  • Interval time stamp: is in ISO format. It is the complete date plus hours, minutes, seconds and a decimal fraction of a second (if different from 0). The format is YYYY-MM-DDThh:mm:ss.sTZD.

  • Device MRID: is a text field.

  • Device name: is a text field. The name cannot contain the selected separator (“,” or “;”).

  • Reading type: the reading types are grouped per device and it is the device group that determines the order. Within a reading type, the values are ordered chronologically on the interval time stamp: the oldest interval time stamp first.

  • Meter value: the value is a number. New data will be marked with the defined Identifier tag and updated data will be marked with the Update identifier tag.

  • Validation result: can have the following results:

  • If the data was not validated an empty string is shown.

  • If the data was validated and is valid the string “valid” is shown.

  • If the data is missing the string “missing” is shown.

  • If the data is present and suspect the string “suspect” is shown.

Example:

  • 2015-06-01T00:00:00+02:00;3d2d2c2b-6c4d-45d0-9285-46ef25d4d87c;EDR513PX468003;0.0.2.1.1.1.12.0.0.0.0.0.0.0.0.0.72.0;3771825;valid

3

Note

It is recommended to always add the <identifier> tag to either the file name or file location and it is required to do so when exporting updates to a different file or folder.

5.   Click Add.

 

1

Important

The following characters can’t be used in file names, file extensions or file locations unless they are part of one of the available tags.

Output result

Each created file will contain the following fields:

  • Event date: the timestamp of the event occurrence, not the collection date. The format of the event date is defined by ISO 8601. It is the complete date plus hours, minutes, seconds and a decimal fraction of a second (if different from 0). The format is YYYY-MM-DDThh:mm:ss.sTZD.

  • Event type: the 4 digit ID. All events are grouped per device. Events are sorted chronologically based on the event date with the newest events on the bottom of the list.

  • Device MRID: is a text field.

  • Device name: is a text field. The name cannot contain the selected separator (“,” or “;”).

Example:

  • 2015-07-15T08:22:20.500+01:00,0.36.114.24,3d2d2c2b-6c4d-45d0-9285-46ef25d4d87c,SPE001

Add reading type

5

Steps

1.   Click Add reading types.

2.   Complete the filter fields to find the correct reading types.

3.   Click Apply.

Deselect items in the drop down menu to remove single filters or click Clear all to remove all applied filters.

4.   Select one or more reading types from the table.

You can see the reading type details by clicking the information icon .

5.   Click Add.
