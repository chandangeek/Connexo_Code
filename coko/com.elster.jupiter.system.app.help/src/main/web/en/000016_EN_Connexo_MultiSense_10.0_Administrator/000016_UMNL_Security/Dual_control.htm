Understand dual control

Dual control in Connexo

In Connexo you can configure most objects using parameters. Generally you can trust one person to make the changes. However in certain cases changing a parameter can have a major impact on Connexo, as well as the utility / environment that is managed with Connexo.

To minimize this risk, Connexo is equipped with the dual control mechanism that is based on the four-eyes principle. This means that certain changes must be approved by at least two people. The two people are the one person that made the change and then a second (or third) person that agrees with the alteration and approves it. The change is only effective when the approver agrees.

Figure: How dual control works in Connexo. Meter operator can apply changes in the application, but when dual control is available all approvers , assigned by the dual control administrator, must approve the change before it can become effective (situation A).

 

Example

A smart energy meter often has a physical breaker that in some cases you can control remotely using the deviceâ€™s communication. Sending a disconnect command from Connexo to this device will result in a breaker status switch. Consequently no energy can be consumed anymore. Switching the breaker of an energy meter is required to support use cases as move in / move out. When a user does not pay its energy bill, you can remotely disconnect the meter and prevent energy theft.

However this introduces a huge risk. If you can remotely disconnect one meter it is also possible to do a bulk operation and disconnect the whole meter landscape of a utility. This can result in a major power change on the net which can cause imbalance of the grid and results in consumers not being able to consume energy anymore.

To prevent this scenario from happening, Connexo offers the possibility to limit the amount of disconnect commands that are sent remotely per day. This prevents a user from disconnecting all devices at once.

If you allow that one person can change this parameter and the change is effective immediately you still have the same risk. As this one person is still the weak link in the chain. If he changes the upper limit to 1 million, executes all disconnect commands and reverts the parameter to the old state we will still have the same problems.

With the dual control mechanism you can specify that only three people can accept the change of the maximum amount of device commands and that all three of them have to accept this before it can be effective. In this way the utility is protected against one person that wants to harm the utility.
