Import time of use calendar

Time of use calendars are imported into Connexo with an XML file which is validated by the calendar xsd file. The existing import framework is used for this purpose. See “Import services”. After importing the calendar, it becomes available for viewing (see “View a time of use calendar”) and assigning to a device (refer to the Connexo MultiSense user guide).

3

Note

Some delay may occur when first using the new calendar because time series for an entire year will be calculated. Afterwards, new series will be calculated via a background process.

events

Occurrence that you want to keep track of. This can be the start of an off-peak period or of a holiday period. An event always has an ID, a name and a code. The code corresponds to the twelfth field of the CIM reading type.

Yes

day-type

A day type splits up each day in a period by means of ranges and associates these ranges with events. So, the day type will allow you to apply different tariffs to different moments of the day. Each range only has a start time. It lasts from its start time until the start of another range of that day. For example, if a range A starts at 23:00 and a range B starts at 02:00, then the period for range A is from 23:00 until 23:59 and from 0:00 until 2:00. Each moment of the day is covered by a day type. Apart from one or more ranges, each day type has an ID and a name.

Yes

periods

A period corresponds to a time range within a year, i.e mostly a few months. It always has a name and a week template. The week template indicates for each day the valid day type.

Periods only have start dates. Each period ends when it transitions into the next one. These transitions are setup either on fixed dates (day, month, year) or on recurring dates (day, month). Periods can overlap years. For example, if period 1 starts on 1 December and period 2 starts on 1 March, then December, January and February are part of period 1.

The whole year must be covered by periods, including the time before the start of the very first period. When using recurring periods you only have to look at the last started period of the preceding year to know what period applies to the time before the start of the first period in the current year. In case of fixed periods, there is no way to know what period applies to the time before the start of your first period in the start year. Therefore, your very first period must start on 1 January. If needed, you have the possibility to let it start earlier because the period will wrap around the year change anyway. You just need to make sure that the period spans the entire year.

Yes

exceptions

Special cases where the regular rules do not apply such as holidays, special days that are linked to an event, etc. This exception can be on a fixed date (day, month, year) or it can be a recurring date (day, month).
