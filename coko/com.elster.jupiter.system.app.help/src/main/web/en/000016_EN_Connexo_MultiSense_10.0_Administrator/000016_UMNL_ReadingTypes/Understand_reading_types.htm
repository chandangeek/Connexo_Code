Understand reading types

Reading types provide a detailed description of the data that is collected. This parameter is based on CIM standards, making data interchangeable with other systems. A reading type consists out of 18 key attributes, each corresponding to a different feature:

Attribute

Name

Description

1

Time period of interest

Reflects how the reading is viewed or captured over a period of time.

2

Data qualifier

Describes a salient (~main or important) attribute of the data.

May be used to define a mathematical operation carried out over the period of interest as defined in attribute #1.

3

Time

Describes the way the value was originally measured and not the frequency at which it is reported or presented.

4

Accumulation

Indicates how the value is represented to accumulate over time.

5

Direction of flow

Indicates the direction of flow for readings where this is important (e.g. anything involving current).

6

Commodity

Commodity (~service category) being measured.

7

Kind

Helps to identify what is being measured. Further refines the commodity by providing a reading category.

8

Interharmonic numerator

Indicates the harmonic or interhamonic basis of the measurement

9

Interharmonic denominator

See 8.

10

Argument numerator

Used to introduce numbers into the unit of measure.

11

Argument denominator

See 10.

12

Time of use

Describes any attribution of the value to a specific time of use bucket. See “Time of use calendars”.

13

Critical peak period

Describes any attribution of the value to a specific critical peak period bucket.

14

Consumption tier

Refers to the method of billing in which a certain "block" of energy is purchased (or sold) at one price, after which the next block of energy is purchased at another price, and so on, all throughout a defined period.

15

Phase

Metering phase

16

Multiplier

Describes a power-of-ten multiplier against the data value.

17

Unit of measure

Metering unit

18

Currency

Currency

Reading types in Connexo

Reading types are used throughout the different Connexo applications. Connexo uses them to identify register, channel (load profile) and usage point data. With these reading types different data types can be linked for example to link data of meters with usage points.

In admin you can add all the reading types you need. You can add one by one or add them in bulk. It is possible to deactivate reading types, as a result, the reading types are no longer available in the applications for configuration of different features. However data will still be collected for these reading types. The reading types are not deleted and you can reactivate them when they are needed.
