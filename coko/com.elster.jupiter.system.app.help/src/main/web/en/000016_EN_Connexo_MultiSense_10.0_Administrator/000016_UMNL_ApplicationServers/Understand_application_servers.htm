Understand application servers

What are application servers

An application server is a server that runs different services in the background which allows processes that run for a long period of time not to hinder the user. Examples of such processes are data export and import and the calculation of KPIs.

The services that run on the application server are called message services because the server handles messages. These message services can be configured to reduce the load on the system. This can be achieved by creating different application servers for different services. For example, two application servers can be created and configured for data export. This will divide the export messages over the two servers allowing them to be executed at the same time.

You can also scale the threads of an application server by increasing the number of threads of the message service. This will determine the amount of services such as export tasks that can be performed in parallel on a single server.

Using multiple servers also creates a more stable system because when one server is down due to error or maintenance, the second server can take over.

Application servers in Connexo

Application servers in Connexo can easily be added and managed using the Admin application. When you add a new application server three parameters need to be defined: a unique name, the export and import path and the message and import services.

The export path sets the default path where files and data will be exported to. However this path can be overruled when you configure an export task. Beware that if you create multiple application servers for data export tasks that belong together, the export path of the different applications servers is the same. Otherwise your data will be scattered over different locations. The same applies to import services.

By default all message services and import services are added to the application server, but these can easily be removed to create the application server that meets your needs.
