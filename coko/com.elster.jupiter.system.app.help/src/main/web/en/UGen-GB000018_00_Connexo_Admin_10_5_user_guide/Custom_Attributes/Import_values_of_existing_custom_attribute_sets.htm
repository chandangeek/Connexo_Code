Import values of existing custom attribute sets

You can specify values for existing custom attributes for multiple devices in an external file and then import this file into Connexo. For this purpose the Device custom attributes importer is available to create the appropriate import services. This method is a lot faster than editing the custom attributes device per device.

Import file with custom attribute sets

You can easily upload an external file using the “Upload files for import” feature in Connexo MultiSense. The maximum file size is 100 Mb.

Features of the Device custom attributes importer

  • Specify multiple custom attribute sets in one file and update values and versions of multiple custom attribute sets on multiple devices, device registers and device channels simultaneously.

  • The following actions can be done:

  • Update values of a non-versioned attribute set.

  • Update values of a versioned attribute set.

  • Create a new version on a versioned attribute set.

  • Update start and end time of a versioned attribute set.

  • Errors are handled properly and clear messages are written into the import service logging. See “View import service history” (Connexo Admin) and “View import service history” (Connexo MultiSense).

  • The importer supports the following data types:

  • Number.

  • String.

  • Boolean.

  • Quantity.

The quantity field format is ‘value:multiplier:unit’. For instance 23,84:3:V = 23,84V).

  • Enum.

Resolve conflicts using the Device custom attributes importer

When updating or adding new versions, eventual conflicts are automatically resolved.

Case

Automatic resolution

The new version partially overlaps an existing version, i.e. the start date of the new version is within an existing version.

The end date of the existing version is updated and equals the start date of the new version.

The new version partially overlaps an existing version, i.e. the end date of the new version is within an existing version.

The start date of the existing version is updated and equals the end date of the new version.

There is a gap between the new and the existing version where the new version comes after the existing one.

The start date of the new version is updated.

There is a gap between the new and the existing version where the new version comes before the existing one.

The end date of the new version is updated.

The new version completely overlaps an existing version.

The existing version is removed.

Expected file format for the Device custom attributes importer

  • The order of the fields that are related to the custom attributes isn’t defined. It can be random.

  • Values in columns with the Boolean data type that are left empty are imported as false.

  • The import file must contain the defined header titles.
