Understand message queues

A message queue contains all messages that are waiting to be picked up and handled by the message services. When the message service is active it will pick up a message from the queue (in the order they were added) and try to execute it. If the execution fails the message service will try again after a defined period of time. If after a defined amount of tries, the execution still fails, then the message will be put on the error queue.

It is possible to link multiple message services to one message queue.

View message queues

You can see an overview of all existing queues with for each queue corresponding details and operational information.

5

Steps

1.   Go to Administration > Application server > Message queues.

2.   Make sure that the link Overview in the left-hand menu is selected.

This page shows all types of both default and custom queues with the applicable number of retries and the time before the next retry. The default message queues are indicated with  in the first column of the table.

3.   You can remove added message queues by clicking the cross at the end of the corresponding row. Following, click Remove in the confirmation dialogue.

Add message queues

You can add new message queues next to the default ones in order to manage scalability. You will be able to assign tasks to the created message queues.

5

Steps

1.   Go to Administration > Application server > Message queues.

2.   On the Overview page, click the button Add message queue.

3.   On the Add message queue page, enter a unique name and specify the task type for the new message queue.

4.   Click Add.

The new message queue can now be seen in the table on the overview page.

Monitor message queues

You can monitor all message queues in Connexo and see number of messages that they contain as well as the errors that occurred in each queue.

5

Steps

1.   Go to Administration > Application server > Message queues.

2.   Click Monitor in the menu on the left.

This page shows an overview of all message queues in the system. The table shows per message queue the amount of messages and the amount of errors.

3.   Click the actions icon  and select Clear error queue to reset the error counter to zero.

Edit message queues

5

Steps

1.   Go to Administration > Application server > Message queues.

2.   The following parameters of the message queues can be adjusted:

Parameter

Description

Retries

Define the number of times a message service should retry to execute the messages of this message queue.

Time before retry

Define how long the message service should wait before it retries to execute the messages of this message queue.

3.   Click the values to edit them.

4.   Click Save settings or Undo to cancel the changes.

Understand recurrent tasks

Recurrent tasks are all the tasks that are scheduled in the different Connexo applications. The task overview allows the system administrator to monitor the system and see how many tasks are scheduled, when they are scheduled and what their current status is. Based on this information you can manage your system and improve performance for example by adding additional application servers for data export.

Monitor recurrent tasks

5

Steps

1.   Go to Administration > Application server > Task overview.

The overview table shows all recurrent tasks in all Connexo applications. For every task the application, used queue and scheduled run are displayed. More details are available in the details pane below.

2.   Filter the tasks by using the filter on top of the page. Select on or more criteria and click Apply.

3.   Remove filters by deselecting items in the drop down menu or click Clear. Remove all applied filters by clicking Clear all.

Assign a task to a queue

You can assign data export, estimation or validation task to a different queue if this is needed to better manage the server load.

5

Steps

1.   Go to Administration > Application server > Task overview.

2.   Select task and click the actions icon at the bottom of its line.

3.   Select Set queue.

4.   In the popup dialogue, select the needed queue.

Only those queues that are compatible with the selected task type are shown.

5.   Click Save.

The new queue is now shown on the Task overview page. You may need to refresh the page.
