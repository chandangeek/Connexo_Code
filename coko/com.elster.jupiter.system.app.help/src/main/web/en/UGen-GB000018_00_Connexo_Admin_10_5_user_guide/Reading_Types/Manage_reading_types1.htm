Manage reading types

Add reading types

You can add a reading type set with one or more reading types. You can either create a basic set of reading types based on a limited set of parameters or you can go for the extended set of reading types using all 18 parameters of the CIM code. The created reading type set may contain multiple reading types depending on the values that were entered in the parameters.

3

Note

A number of reading type sets are already available in Connexo by default.

 

5

Steps

1.   Go to Administration > Reading types management > Reading types.

2.   Click Add reading type.

3.   Complete the following fields:

Parameter

Description

Reading type set

Name of the reading type set that is used to group the underlying reading types. It is the common name for all reading types that are added based the basic or extended CIM attributes of the reading type set.

Specify by

Select how to specify the reading type(s):

oCIM code: Enter the 18 attributes of a CIM code separated by a dot.

  • Form: Use the Basic and Extended tabs with drop-down boxes containing the value and description for each CIM attribute.

a. The Basic tab contains by default six drop-downs, i.e. Commodity (#6), Kind (#7), Direction of flow (#5), Unit (#17), Period (#1) and Aggregate (#2). In these drop-downs you can select only one value per CIM attribute. Depending on the selected value in Commodity (#6), you will be able to make certain selections in Kind (#7) and Direction of flow (#5). The available values in Unit (#17) depend on the selection in Kind (#7). The selection in Commodity (#6) also determines the attributes in the section Additional parameters. These attributes aren’t mandatory and multiple values can be selected simultaneously.

b. The Extended tab contains all 18 attributes of the CIM code with all available values in drop-down lists. You can select multiple values from each drop-down list.

At the bottom of the Basic and Extended tab you can see in red the total amount of reading types that will be added with a maximum of 1000 reading types that can be added simultaneously.

4.   Click Add.

5.   Click Add in the pop-up window to confirm adding the specified number of reading types.

3

Note

The created reading types are activated and ready for use.

Activate/Deactivate single reading types

You can activate or deactivate a reading type to make it available for selection in Connexo.

5

Steps

1.   Go to Administration > Reading types management > Reading types.

2.   Click a reading type set name to see its details.

3.   Click Reading types in the vertical navigation column at the left.

4.   Click the actions icon  next to the needed reading type and select Activate or Deactivate.

Alternatively, click the Actions button in the details pane of the selected reading type and select Activate or Deactivate.

When you deactivate the reading types, they will no longer be available for selection in the configuration of different features in the other Connexo applications. You are still able to collect and save data from reading types that you deactivate.

Activate/Deactivate reading types in bulk

To save time you can activate or deactivate multiple reading types simultaneously in a bulk action.

5

Steps

1.   Go to Administration > Reading types management > Reading types.

2.   You can prefilter the needed reading types as described in “View reading type sets and reading types”.

3.   Click Bulk action.

4.   Select All reading types to apply the change on all (filtered) reading types or use Selected reading types to narrow down your selection and hand-pick the needed reading types.

5.    Click Next.

6.   Select Activate or Deactivate.

When you deactivate the reading types, they will no longer be available for selection in the configuration of different features in the other Connexo applications. You are still able to collect and save data from reading types that you deactivate.

7.   Click Next.

8.   Click Confirm.

The change is now queued for execution.

9.   Click Finish.

Add a reading type to an existing reading type set

You can add one or more extra reading types to an existing reading type set to make it more representative. In order to make sure that the newly added reading type is compatible with the existing reading type set, some properties can’t be edited. These read-only properties are:

  • Commodity.

  • Kind.

  • Direction of flow.

  • Unit.

 

5

Steps

1.   Go to Administration > Reading types management > Reading types.

2.   Open a particular reading type set.

3.   Click Reading types in the vertical navigation bar on the left.

4.   Click the button Add reading type on top of the reading types table.

5.   Complete the necessary fields as explained in “Add reading types”.

6.   Click Add.

The added reading types are now in the reading types overview.

Edit a reading type set

You can edit the alias of a set of reading types to correct eventual mistakes.

5

Steps

1.   Go to Administration > Reading types management > Reading types.

2.   Select Edit from the actions icon next to the needed reading type set.

3.   In the pop-up, enter the new alias. This will be your new reading type set name.

4.   Click Save.

3

Note

  • The alias must be unique.

  • The alias change will affect all reading types that are associated with the edited reading type set.
