Configure web service endpoints

Manage web service endpoint

Add web service endpoints

5

Steps

1.   Go to Administration > Web services > Web service endpoints.

2.   Click Add web service endpoint. Complete the following fields:

Parameter

Description

Name

Give a unique name

Webservice

Select a web service. Depending on this selection different parameters need to be defined.

Url path

Required for incoming web services. Provide a unique path for the URL of the following format https://<host>:<port>/<path>.

Url

Required for outgoing web services. Provide a unique url in the following format https://<host>:<port>/<path>.

Log level

Select the log level of the endpoint. If you select a high log level this will result in a detailed logging resulting in high database usage.

Trace requests

If you select this option, Connexo will create a file with the name you specified containing all request details.

HTTP compression

Compress outbound HTTP data and accept inbound compressed HTTP data.

Scheme validation

Validate the XML schemes.

Authentication

Add authentication level, if you select basic authentication then you will need to specify user name and password. For inbound services, you can specify a user role.

3.   Click Add.

Activate/deactivate web service endpoints

5

Steps

1.   Go to Administration > Web services > Web service endpoints.

2.   Click the actions icon next to the endpoint that you want to activate/deactivate.

3.   Select Activate/Deactivate.

Edit web service endpoints

5

Steps

1.   Go to Administration > Web services > Web service endpoints.

2.   Click the actions icon  next to the endpoint that you want to edit and select Edit.

3.   Change the web service endpoint parameters. Note that not all parameters can be edited when the endpoint is active.

4.   Click Save.

Remove web service endpoints

5

Steps

1.   Go to Administration > Web services > Web service endpoints.

2.   Click the actions icon  next to the endpoint that you want to remove and select Remove.

3.   Confirm by clicking Remove.

3

Note

Requests which are being executed during the removal of the endpoint will be finalised with the settings of the endpoint at the time of the request. Requests after the endpoint has been removed, will result in an error.
