Configure application servers

Add an application server

5

Steps

1.   Go to Administration > Application server > Application servers.

2.   Click Add application server and complete the following fields:

Parameter

Description

Name

Name of the server that the application runs on.

Export path

Default path to store exported files and data. The path can be overruled in the export task configuration.

Import path

Default path to import data from. The path can be overruled in the import service configuration.

Message services

Select the message services that need to be executed by the application server by removing those that do not need to be executed. This can be done by clicking the remove icon .

Change the number of threads or the status of the service by clicking the value.

Message services can be added when you click Add message services.

Import services

By default all configured import services are added to the application server. Remove import services by clicking the remove icon , add import services by clicking Add import services.

3.   Click Add.

4.   Click the actions icon  and select Activate.

3

Note

When you activate an application server this server becomes available in the application instead of being activated as such.

Edit application server

5

Steps

1.   Go to Administration > Application server > Application servers.

2.   Click the actions icon  next to the application server you want to edit and select Edit.

3.   Change the application server parameters

4.   Add, edit or remove message and import services.

5.   Click Save.

Edit message services on application server

5

Steps

1.   Go to Administration > Application server > Application servers > click application server name > select Message services in the menu on the left.

2.   Click the status or the number of threads to change the value.

3.   Remove message services by clicking the remove icon  or add services by clicking Add message service.

4.   Click Save changes to save or click Undo to cancel all changes made.

Edit import services on application server

5

Steps

1.   Go to Administration > Application server > Application servers > click application server name > select Import services in the menu on the left.

2.   Remove import services by clicking the remove icon  or add services by clicking Add import service.

3.   Click Save changes to save or click Undo to cancel all changes made.

Remove application servers

5

Steps

1.   Go to Administration > Application server > Application servers.

2.   Click the actions icon  and select Remove to delete the application server. Click Remove in the dialog that appears.

Default message services

In Connexo different message services are available by default. The table below gives an overview of these services together with a short description.

Message service

Description

Calculate connection and communication KPIs

Calculates the communication and connection KPIs on the time points specified in the configuration.

Create data collection issues

Picks up events created by the data collection engine and creates issues based on the configured issue creation rules.

Create data validation issues

Picks up events created by the data validation engine and creates issues based on the configured issue creation rules.

Handle bulk actions for device configuration changes

Executes the change of device configuration of the devices selected using the device search and using the Bulk action option.

Handle communication schedule addition/removal to/from device

Performs the bulk action of adding or removing the selected shared communication schedule to every single device defined by the selected filters on the device search page.

Handle communication task rescheduling

Performs the bulk action “Run” for every single item of the selection made on Workspace > Data communication > Communications.

Handle connection attribute updates

Performs the bulk action of changing the connection attributes for every single item selected on Workspace > Data communication > Connections.

Handle connection rescheduling

Performs the bulk action “Run” or “Run now” for every single item of the selection made on Workspace > Data communication > Connections.

Handle connection updates on communication task configuration

When changing the connection method of a communication task configuration on a device configuration, this service will change the connection method of every device using this device configuration.

Handle Connexo Facts search

Required for the integration of Facts with the other Connexo applications.

Handle Connexo Flow

Required for the integration of Flow with the other Connexo applications.

Handle data estimation

Executes data estimation tasks.

Handle data export

Executes data export tasks, triggered manually or automatically.

Handle data import

Handles all events from the import services and thus executing the actual import action. Usage point importer events are not handled by this message service.

Handle data purge

Executes data purging in the background.

Handle Data Validation

Executes data validation tasks.

Handle device state change

When altering the device life cycle of a device type, this service will change the device state of all devices of this device type into the corresponding state of the new device life cycle.

Handle firmware campaign

This service will which devices of the selected device group have the specified device type and will subsequently upload the desired firmware. Because it can take some time to calculate the amount of devices in the firmware campaign, the firmware campaign progress is not always shown immediately on the firmware campaign page.

Handle obsolete communication schedules

This service will handle inactivating all communication schedules of devices that use a communication task configuration of the device configuration that has been set on inactive.

Handle overdue issues

Checks whether issues are overdue and if they are, perform the action specified in the issue creation rule.

Handle priority updates on communication task configuration

When changing the urgency of a communication task configuration on a device configuration, this service will change the urgency of every device using this device configuration.

Handle status updates on communication task configuration

When changing the status of a communication task configuration on a device configuration, this service will change the status of the communication task on every device using this device configuration.

Handle usage point import

Handles all events from the usage point importer.

Itemize communication schedule addition/removal to/from device

Converts specified filter criteria on the device search page into single items and puts them on the queue to be handled by “Handle addition/removal of a communication schedule” message service.

Itemize communications from filter

Converts specified filter criteria on the communications page (Workspace > Data communication > Communications) into single items and puts them on the queue to be handled by the “Reschedule communication rescheduling” message service.

Itemize connection attribute updates from filter

Converts specified filter criteria on the connections page (Workspace > Data communication > Connections) into single items and puts them on the queue to be handled by the “Handle connection property updates” message service.

Itemize connection rescheduling from filter

Converts specified filter criteria on the connections page (Workspace > Data communication > Connections) into single items and .

Recalculate communication schedules

When the schedule of a shared communication schedule is adjusted, this service will calculate the next run of the communication tasks of all devices that use this shared communication schedule.

Revalidate connection methods after property removal on configuration level

When removing one of connection details of the connection method configuration on a device configuration, this service will change the status of every device that inherited these connection details to “Incomplete”
