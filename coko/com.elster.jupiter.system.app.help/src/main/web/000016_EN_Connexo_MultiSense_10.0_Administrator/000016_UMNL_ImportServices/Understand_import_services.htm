Understand import services

Import services are configured in Admin  and used in the other Connexo applications. When creating new import services Connexo offers different default file importers that allow the import of different types of data to perform different actions. During configuration you need to specify the location of the files that need to be imported. Next to the import folder you will also specify the folder for in progress files and files of successful and failed imports. All this information is also visible in the applications using the import services when you look at the import service details.

The import service will take the file that needs to be imported and move it to the In progress folder and subsequently put it on the queue. The actual import is handled by a message service. This means that for import to be successful import service as well as message service need to be configured and added to the application server.

Default file imports

Connexo offers different standard importers that are available by default upon installation of Connexo. These standard importers are marked by the tag [STD]. These importers are described in the MultiSense  and Insight  user guides together with the description and an example of CSV files needed for import.
