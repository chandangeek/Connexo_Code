Serviceanrufe, Serviceanruf-Typen und Lebenszyklus

Serviceanrufe verstehen

Was ist ein Serviceanruf?

Serviceanrufe werden verwendet, wenn die Verantwortung für die Ausführung einer Handlung einem externen System wie einem Rechnungssystem oder einem MDM übertragen wurde. Damit können Sie den Status und Fortschritt von Handlungen verfolgen, die von Systemen der Drittanbieter über Webdienste, API, Importer, Prozesse usw. an verschiedenen Objekten in Connexo wie Geräten und Verwendungspunkten angefordert werden.

Anhand der Serviceanrufe kann der Operator sehen, welche Änderungen an einem Objekt vorgenommen wurden und woher die Handlung stammt. All diese Informationen ermöglichen dem Operator zu untersuchen, was passiert und bei Bedarf einzuschreiten.

Wichtig

Serviceanrufe sind ein Kontrollmechanismus und führen selbst keine Handlungen aus. Der aktuelle Service wird vom Anfordernden ausgeführt, d.i. das System des Drittanbieters. Der Serviceanruf funktioniert als Tag.

Serviceanruftypen verstehen

In Connexo erzeugen Systeme von Drittanbietern über den Serviceanruf-Rahmen Serviceanrufe. Dieser Rahmen erfordert von Serviceanruf-Typen, dass eingehende Anfragen kontrolliert werden können. Serviceanrufe hängen immer mit einem Serviceanruf-Typen zusammen.

Jeder Serviceanruf-Typ hat einige obligatorische Eigenschaften wie einen einzigartigen Typennamen, eine Version und ein Log Level. Außerdem hat jeder Serviceanruf-Typ einen zugehörigen Lebenszyklus für diesen Serviceanruf-Typ (Siehe “Lebenszyklen von Serviceanruftypen verstehen”). Derselbe Lebenszyklus des Serviceanruf-Typen kann mit mehreren Serviceanruf-Typen verbunden sein. Ein Serviceanruf hat immer einen bestimmten Zustand.

Die Eigenschaften eines Serviceanrufs werden für jeden Serviceanruf definiert. So wird beispielsweise ein Serviceanruf, der von einem Webservice zum Abkoppeln eines Messgeräts erstellt wird, andere Eigenschaften benötigen, als ein Serviceanruf, der von einem Webservice zum Erfassen von Messgerätdaten erzeugt wird. Der Serviceanruf-Typ verbindet den Serviceanruf-Anwender mit den korrekten benutzerdefinierten Eigenschaften.

Serviceanruf-Typen werden von Ihrem Service- und Lieferteam installiert und können nach Erstellung nicht geändert werden.

Serviceanruf-Typen können in Connexo-Admin angezeigt werden.

Lebenszyklen von Serviceanruftypen verstehen

Ein Serviceanruf befindet sich immer in einem bestimmten Zustand. Das ist der aktuelle Zustand. Zustände werden über einen endlichen Automaten verwaltet. Also kann jeder Serviceanruf sich nur in einem Zustand gleichzeitig befinden. Er kann von einem in einen anderen Zustand wechseln, wenn das von einem auslösenden Ereignis oder einer Kondition ausgelöst wird. Das nennt man einen Übergang. Der Lebenszyklus des Serviceanrufs-Typs wird durch eine Liste seiner Zustände und der auslösenden Bedingungen für jeden Übergang definiert.

Der Lebenszyklus des Serviceanrufs-Typs wird im Serviceanruf-Typ benannt und definiert, der von Ihrem Service- und Lieferteam installiert wurde. Um bestehende Lebenszyklendiagramme zu beschränken, können Zustände und Übergänge aus dem Lebenszyklus gelöscht, aber nicht hinzugefügt werden.
