Warum benötigen Sie Zertifikate

Für die Kommunikation ist eine Vertrauenskette notwendig, um den Anwender eines Zertifikats zu validieren. Die Vertrauenskette einer Zertifikatkette ist eine geordnete Liste von Zertifikaten, mit einem Endverbraucher-Abonnementzertifikat und Zwischenzertifikate (die das Zwischenzertifikat repräsentieren). Damit kann der Empfänger verifizieren, dass der Sender und alle Zwischenzertifikate vertrauenswürdig sind. Wenn die Vertrauenskette nicht vollständig ist, wird die Kommunikation abgelehnt.

In Connexo benötigen Kommunikationsserver, die mit Geräten kommunizieren, manchmal den Nachweis, dass sie wirklich mit diesem Gerätetypen kommunizieren. Das funktioniert auch in der umgekehrten Reihenfolge. Beispielsweise benötigt Beacon 3100 den Nachweis vom Server, dass er wirklich ein Kommunikationsserver ist.

Dieser Nachweis erfolgt in der Form eines Zertifikats. Das Zertifikat des Geräts wird vom Hersteller bereitgestellt. Das Zertifikat des Kommunikationsservers unterstützt TLS (Transport Layer Security) Protokolle. Je nach verwendeter DLMS-Sicherheit (Suite 0, 1 oder 2) werden während der Kommunikation verschiedene Verschlüsselungs- und Authentifikationsebenen verwendet. Nur wenn Sie DLMS Sicherheitssuite 1 oder 2 erfordern, benötigen Sie Zertifikate.

Für mehrere Kommunikationsserver wird nur ein Zertifikat verwendet. Der Grund dafür ist, dass Geräte nur eine beschränkte Speicherkapazität haben und Sie also normalerweise Kommunikationsserver zur Kommunikation verwenden.

Abbildung: Sicherheit während der Kommunikation in Connexo.

 

Damit man in Connexo kommunizieren kann, ist es wichtig, dass der Trust Store sowie die Zertifikate konfiguriert und verfügbar sind. Wir empfehlen zuerst den Trust Store einzurichten und dann die Zertifikate zu importieren.

Zertifikatmanagement in  Connexo Admin

In Connexo Admin wird Sicherheit in zwei verschiedenen Ebenen konfiguriert:

  • Trust Stores: Ein Key Store, der verwendet wird, wenn Entscheidungen darüber gemacht werden, was vertrauenswürdig ist. Der Trust Store in Connexo Admin enthält eine Sammlung von Zertifikaten zur Unterstützung der Trust Chain. Der Trust Store wird als Key Store Datei importiert.

  • Zertifikate: in Connexo Admin werden die Zertifikate der niedrigsten Stufe, die einen der verfügbaren Trust Stores verwenden, hier gespeichert. Sie können Zertifikate importieren oder einen CSR (Zertifikatsregistrierungsanforderung) erstellen und das erstellte Zertifikat dann auf der Grundlage dieses CSR importieren.
