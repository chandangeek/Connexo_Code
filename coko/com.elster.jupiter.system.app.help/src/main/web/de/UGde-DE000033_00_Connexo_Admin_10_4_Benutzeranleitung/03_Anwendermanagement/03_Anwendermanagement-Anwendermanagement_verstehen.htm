Anwendermanagement verstehen

Anwendermanagement ist ein kritischer Teil von Connexo MultiSense. Es ermöglicht Ihnen ein sicheres System zu erhalten, indem die Anwendungen definiert werden, auf die Anwender mittels ihrer Rechte innerhalb dieser Anwendungen zugreifen können. Das Anwendermanagementsystem basiert auf Anwendern, Anwenderrollen und Rechten.

Benutzer

Jede Person, die die Connexo MultiSense Anwendung verwendet, muss als Anwender definiert werden. Jeder hat sein eigenes einzigartiges Profil. Anwendern können Rollen zugewiesen werden, wenn sie erstellt werden, aber sie können auch anschließend noch verändert werden. Anwender können nur durch Synchronisierung mit einem aktiven Verzeichnis erstellt werden. Wenn Ihnen keines zur Verfügung steht, können Sie immer einen lokalen aktiven Verzeichnisserver mittels WSO2-Identitätsserver einrichten.

Über LDAP-Integration ist einmaliges Anmelden möglich. Das bedeutet, dass Sie sich nur einmal anmelden müssen, um Zugang zu allen Anwendungen zu bekommen. Externe Anwender können ebenfalls definiert werden und Zugang zu verschiedenen Anwendungen erhalten.

Anwenderrollen

Anwenderrollen bestimmen, welche Anwender Zugang zu welcher Anwendung haben und was sie in dieser Anwendung tun können und was nicht. Anwenderrollen werden verwendet, um Rechte zu gruppieren und verschiedenen Anwendern zuweisen zu können. Jeder Anwender, dem eine bestimmte Anwenderrolle zugewiesen wurde, hat alle Rechte dieser Rolle. Wenn einem Anwender mehreren Rollen zugewiesen wurden, hat der Anwender alle Rechte jeder Rolle.

Arbeitsgruppen

Gruppiert Anwender auf der Grundlage der Aufgaben, die sie ausführen. Arbeitsgruppen ermöglichen Ihnen Arbeit zwischen Personen zu verteilen, die dieselben Aufgaben erbringen. Wenn ein Objekt einer Arbeitsgruppe zugewiesen wurde, können Sie Objekte je nach Verfügbarkeit der Anwender dieser Arbeitsgruppe einfach zuweisen und aberkennen. Arbeitsgruppen sind in Connexo für die Funktionsalarme, Probleme und Aufgaben verfügbar.

Rechte

Um die Rechte innerhalb von Connexo zu verstehen, muss erst die allgemeine Struktur besprochen werden. Connexo besteht aus verschiedenen Anwendungen wie Connexo Admin, MultiSense, Fakten, usw. Jede Anwendung hat verschiedene Ressourcen. Eine Ressource ist eine Funktion oder eine Kombination von Funktionen innerhalb der Anwendung, z.B. bestimmt die Ressource “Kommunikationsinfrastruktur” alle Funktionen zur Verwaltung der Kommunikationsinfrastruktur wie Kommunikationsserver und Ports. Pro Ressource können Sie Rechte definieren.

Rechte bestimmen, welche Rechte ein Anwender hat, um bestimmte Handlungen bei einer bestimmten Ressource auszuführen. Das kann beispielsweise Ansehen, Planen, Kommunikation und Feinabstimmung der Validierung eines Geräts sein. Da Anwendern mehrere Rollen zugewiesen werden können, sind alle mit diesen Rollen verbundenen Rechte gruppiert. Das bedeutet, dass es nicht notwendig ist, jedes einzelne Recht zu jeder einzelnen Rolle hinzuzufügen.

Rechte können weiterhin in Ebenen verteilt werden. Ebenen ermöglichen die Gruppierung von Anwenderrollen und die Zuweisung zu einem bestimmten Recht. Ebenen werden nur für die Definition von Rechten von einigen Ressourcen wie Gerätesicherheit und Gerätekommandos verwendet. Sie können beispielsweise alle Anwenderrollen, die mit Stromzähler arbeiten, Rechte der Ebene 1 zuweisen und allen Anwenderrollen, die mit Gaszählern arbeiten Ebene 2. In MultiSense können Sie dann die Rechte und die Ebenen mit den Geräten verbinden.
