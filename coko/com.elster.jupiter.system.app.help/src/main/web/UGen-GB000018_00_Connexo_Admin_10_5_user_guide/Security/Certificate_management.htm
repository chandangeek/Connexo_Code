Certificate management

Understand certificate management

Why do you need certificates

In order to communicate, a trust chain is needed to validate a certificate's user. The trust chain of a certificate chain is an ordered list of certificates, containing an end-user subscriber certificate and intermediate certificates (that represents the intermediate CA). This enables the receiver to verify that the sender and all intermediates certificates are trustworthy. If the trust chain is not complete, communication will be denied.

In Connexo communication servers that communicate with devices sometimes require proof that they are really communicating with that device type. This also works in the reverse order for example the Beacon 3100 needs proof from the server that this really is a communication server.

This proof is provided in the form of a certificate. The certificate of the device is provided by the manufacturer. The certificate of the communication server supports TLS (Transport Layer Security) protocols. Depending on the DLMS security used (suite 0, 1 or 2) different levels of encryption and authentication are used during communication. Only when you require DLMS security suite 1 or 2 you need certificates.

Only one certificate is used for multiple communication servers. The reason for this is that devices only have a limited storage capacity and they typically use different communication servers to communicate.

To be able to communicate in Connexo it is important that the trust store as well as the certificates are configured and available. We recommend to first set up the trust store and then import the certificates.

Certificate management in Connexo Admin

In Connexo Admin security is configured on two different levels:

  • Trust stores: a key store that is used when making decisions about what to trust. The trust store in Connexo Admin contains a collection of certificates to support the trust chain. The trust store is imported as a key store file.

  • Certificates: in Connexo Admin the lowest level certificates that use one of the available trust stores are stored here. You can import certificates or create a CSR (Certificate Signing Request) and then import the certificate created based on this CSR.
