Understand user management

User management is a critical part of Connexo MultiSense. It allows you to maintain a secure system by defining the applications that users can access their privileges within these applications. The user management system is based on users, user roles and privileges.

Users

Every person that uses the Connexo MultiSense application must be defined as a user. Each of them has its own unique profile. Roles can be assigned to users when they are created, but they can also be changed after creation. Users can only be created by means of synchronisation with an active directory. If you don’t have one available, you can always set up a local active directory server using WSO2 identity server.

Via LDAP integration single sign-on is possible. This means that you only have to log in once to access all applications. External users can also be defined and access different applications.

User roles

User roles determine which users have access to which application and what they can and cannot do in this application. User roles are used to group privileges into sets that can be assigned to different users. Every user assigned with a specific user role has all of the privileges included in that role. When a user is assigned multiple roles, then the user will have all of the privileges of each role.

Workgroups

Group users based on the tasks they perform. Workgroups allow you to distribute work between people that perform the same tasks. When an item is assigned to a workgroup, you can easily assign and unassign items depending on the availability of the users of that workgroup. Workgroups are available in Connexo for the features alarms, issues and tasks.

Privileges

In order to understand privileges within Connexo, first the overall structure must be discussed. Connexo consists out of different applications such as Connexo Admin, MultiSense, Facts, etc. Every application has different resources. A resource is a feature or a combination of features within the application, e.g. the resource “Communication infrastructure” designates all features to manage the communication infrastructure such as the communication servers and ports. Per resources you are able to define privileges.

Privileges designate what rights a user has to perform certain actions on a specific resource. This can be for example the viewing, scheduling, communication and fine-tuning of the validation on a device. Because users can have multiple roles assigned to them, all privileges linked to these roles are grouped. This means it is not necessary to add every single privileges to every single role.

Privileges can be further divided into levels. Levels allow grouping of user roles and assigning them to a specific privilege. Levels are only used for defining privileges of some resources such as device security and device commands. For example you can assign all user roles working with electricity meters to level 1 privileges and all user roles working with gas meters to level 2. In MultiSense you can then link the privileges and the levels to the devices.
